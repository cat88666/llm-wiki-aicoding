# coding agent 选型对照表

> ⚠️ 讲师个人实战体感，非评测数据。工具迭代极快，结论随时会变（截至 2026-05）。
> **选型第一轴 = 对 spec 的遵从度**（这正是 SDD 这门课押注的东西），再叠加其余维度。

| 维度 | Codex | Claude Code 原厂 | Claude Code + DeepSeek | GitHub Copilot |
|---|---|---|---|---|
| 复杂任务 / **spec 强遵从** | **最好** | 强 | 可能有偏差 | 不推荐 |
| 自主探索能力 | 中 | **最强** | 中 | 弱 |
| UI 审美 / 文科类任务 | 中 | **好** | 中 | — |
| 中文 / 文案语言顺畅度 | 中 | 中 | **好很多** | — |
| 成本 | 较高 | 较高 | **便宜** | — |
| 速度 | 快 | 中 | **耗时长** | — |
| 整体交付质量 | 高 | 高 | 可以（够用） | 低端平替 |

## 一句话定位
- **Codex**：复杂任务、强 spec 遵从场景的首选体感。
- **Claude Code 原厂**：自主探索强、UI 审美好、文科类任务好。
- **CC + DeepSeek**：spec 遵从略有偏差、耗时长，但整体交付够用、**便宜**、**中文文案明显更顺**——性价比路线。
- **GitHub Copilot**：早期低端平替，**现不推荐**——Pro 对大陆新用户有限制、试用政策反复，对用户不够稳定。

## 怎么用这张表
按你自己的场景给维度加权重：强 spec/复杂任务的押 Codex；中文文案多、预算紧的押 CC+DS；要自主探索和审美的押 CC 原厂。

> 政策类信息（Copilot）最易过期，**用前自己再确认一次**。
