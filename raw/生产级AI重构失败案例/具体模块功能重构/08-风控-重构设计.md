# 风控重构设计文档

> 对应分析文档：`docs/Junk-游戏准备重构/08-风控.md`
> 设计日期：2026-06-04
> 工程：dx-game-frame
> 分支：20260615/junk
> 前置依赖：19 风控数据上报（已完成，提供 `UserRiskCache` 带 `(tableId, userId)` 维度的稳定缓存）

## 1. 背景与目标

当前风控逻辑集中在 `BaseTableRiskService`（全静态门面）：

- **P0**：`checkRiskData` 用单个 `try/catch` 包住 5 条规则计算，任何一条 NPE 都会让整次结果变成 **空 map**，外部根本无法区分"无风险"和"计算异常"。入座、实时轮询、关系图（1041）、强制站起都直接消费这个 map，**异常 = 放行**，安全风险高。
- **P0**：`realtimeCheckRisk` 在一次执行里分别从 `UserRiskCache.getTableRiskData` / `BaseTableUserCache.fetchTableUserIds` / `BaseTableHandCache.isStartIng` / `BaseTableUserCache.fetchAllUidList` 多次独立读取，**没有一致快照**。中途用户站起、入座、退出都会让"算到的风险用户"和"实际通知接收人"对不上。
- **P1**：风险优先级、`KickerReasonEnum` 映射、`RiskTypeEnum.seatError/seatDesc`、`RiskBootstrapService.riskTypeEnumToDesc` 分散四处，新增风险类型容易漏配；`REALTIME_VOICE_RISK` 已在枚举里，但 `RISK_PRIORITY_ORDER` 没收。
- **P1**：`realtimeCheckRisk` 直接调用 `UserLeaveBootstrapService.userStand`、`RiskStandLeaveUserVisitor` 生成 `SpecifyLeaveUserDto`、`GdUserLeaveBootstrapService.checkSitDownUserRealCanLeave` 单点检查 —— 三处幂等与优先级不清。
- **P2**：`info` 级日志整段 `GsonUtils.toJson(tableInfo/riskData/riskMap)`，含 IP/GPS，高峰 CPU 与隐私风险。

目标：消除两条 P0，集中优先级/Reason/提示语策略，全链路日志脱敏；保持 14038 协议字段、入座抛错码、强制站起 visitor 接口 100% 兼容。**不**在本期处理 `RiskActionPlanner`（资金/带出联动太大，与 04 站起、18 预站起一起做更合适）。

## 2. 决策记录

| 决策项 | 结论 |
|--------|------|
| 主导方案 | Frame 主导，全部改造落在 `dx-game-frame`；掼蛋无新增类 |
| `checkRiskData` 是否破坏性改返回类型 | 不破坏。新增 `checkRiskDataResult(tableId, riskData) → RiskCheckResult`；旧 `checkRiskData(...) → Map<...>` 委托新方法 `.getRiskMap()` 并打 `@Deprecated`，老调用点行为不变 |
| `RiskCheckResult` 字段 | `riskMap`、`success`、`failedRules: List<RuleFailure>`（每条失败记录规则名+异常 message） |
| 入座失败策略 | `success=false` 时按"保守拦截"抛 `BusinessExceptionTypeEnums.TABLE_RISK_ERROR`（已有错误码），不放行；理由：宁可误拦，不可漏拦 |
| `RiskTableSnapshot` 范围 | 包含 `tableId / tableInfo / riskData / playerUserIds / watchUserIds / handStarted / capturedAt`；`realtimeCheckRisk` 整次执行用同一份快照 |
| 风险优先级集中 | 新增 `RiskPolicyRegistry`（`@Component`，纯查询）暴露 `getPriorityOrder() / getKickerReason() / getStandType() / getSeatError()`；旧静态门面 `getRiskReason` 内部委托 |
| `REALTIME_VOICE_RISK` 是否纳入 PRIORITY | 暂不动。设计层只新增挂载点（`RiskPolicyRegistry.getPriorityOrder()` 返回 unmodifiable list），是否纳入由产品/合规决定；当前枚举里它的 `seatError` 是 `TABLE_RISK_VOICE_ERROR`，但 `RISK_PRIORITY_ORDER` 不含 —— 保持现状作为待确认项 |
| 灰度开关 | 不做开关；新逻辑通过新方法 + 旧适配双轨：调用点是否切到新方法即可灰度 |
| 14038 通知协议 | `UserRiskNoticeDto` 不动；含义不变 |
| `RiskActionPlanner` | 本期不做。`realtimeCheckRisk` 仍直接调 `userStand`；`RiskStandLeaveUserVisitor` 仍直接生成 `SpecifyLeaveUserDto`；本期只解决数据正确性 |

## 3. 整体架构

### 改造前

```
BaseTableRiskService (全静态门面)
  ├─ checkRiskSitDown(tableId,userId)
  │    ├─ UserRiskCache.getSitDownRiskData
  │    ├─ checkRiskData → Map<...>   ← 异常吞掉，返回空 map
  │    └─ getUserRiskType + throw BusinessException
  ├─ checkRiskData(tableId, riskData) → Map<...>
  │    └─ try { 5 条规则 } catch { return riskMap }  ← P0 异常掩盖
  ├─ realtimeCheckRisk(BaseTable)
  │    ├─ table.setLastRiskCheckTime
  │    ├─ checkRiskData
  │    ├─ getNeetStandUserIds              ← 重新从 cache 读
  │    ├─ userStand 站起
  │    ├─ checkRiskData (再算一次)
  │    └─ sendCheckRiskNotice              ← 再次从 cache 读 allUidList
  ├─ getNeetStandUserIds (private)
  ├─ sendCheckRiskNotice (private)
  └─ getRiskReason(RiskTypeEnum) → KickerReasonEnum

RISK_PRIORITY_ORDER 私有 list / RiskBootstrapService.riskTypeEnumToDesc 独立门面
```

### 改造后

```
BaseTableRiskService (兼容门面，保留所有 public 静态方法签名)
  ├─ checkRiskSitDown(tableId,userId)
  │    ├─ RiskCheckResult result = checkRiskDataResult(...)
  │    └─ result.success=false → 抛 TABLE_RISK_ERROR (保守拦截)
  │       result.success=true & 命中风险 → 抛风险错误码
  ├─ checkRiskDataResult(tableId, riskData) → RiskCheckResult   ← 新增 P0 修复
  │    ├─ 每条规则独立 try/catch → RuleFailure
  │    └─ 任意失败 → success=false
  ├─ checkRiskData(tableId, riskData) → Map<...>   @Deprecated  ← 旧消费者过渡
  │    └─ return checkRiskDataResult(...).getRiskMap()
  ├─ realtimeCheckRisk(BaseTable)
  │    ├─ RiskTableSnapshot snapshot = RiskTableSnapshot.capture(table)
  │    ├─ checkRiskDataResult(snapshot)
  │    ├─ getNeedStandUserIds(snapshot, riskUserIds)            ← 用 snapshot
  │    ├─ userStand 站起
  │    ├─ checkRiskDataResult(snapshot) (二次确认)
  │    └─ sendCheckRiskNotice(snapshot, ...)                    ← 用 snapshot
  └─ getRiskReason(...) → riskPolicyRegistry.getKickerReason(...)

RiskCheckResult (新, model)
  ├─ Map<RiskTypeEnum, Set<Long>> riskMap
  ├─ boolean success
  ├─ List<RuleFailure> failedRules  (规则名+异常message)
  └─ getUserRiskType(userId) 工具方法

RiskTableSnapshot (新, model)
  ├─ tableId / tableInfo / riskData
  ├─ playerUserIds / watchUserIds / allUserIds
  ├─ handStarted
  ├─ capturedAt
  └─ capture(BaseTable) 静态工厂

RiskPolicyRegistry (新, @Component)
  ├─ List<RiskTypeEnum> getPriorityOrder()
  ├─ KickerReasonEnum getKickerReason(RiskTypeEnum)
  ├─ UserStandTypeEnum getStandType(RiskTypeEnum)
  └─ BusinessExceptionTypeEnums getSeatError(RiskTypeEnum)
```

**兼容性原则**：
- `BaseTableRiskService` 所有 public static 方法签名、返回类型不变；包括 `checkRiskData / checkRiskSitDown / realtimeCheckRisk / getUserRiskType / getRiskReason`。
- 现有 4 个调用方完全无感：`BaseUserSitDownCheckService` / `TableLoopTask` / `RiskStandLeaveUserVisitor` / `GdUserCheckCommandBootstrap.shouldAllowRealTimeVideoRiskLeave` / `GdUserLeaveBootstrapService.checkSitDownUserRealCanLeave` / `RiskGraphAssembler`（19 引入）。
- 14038 `UserRiskNoticeDto` 字段不动。
- `BaseTable.lastRiskCheckTime` / `isRiskControl` 字段语义不变。

```java
public static RiskCheckResult checkRiskDataResult(Long tableId, List<RiskDataCollectionDto> riskData) {
    Map<RiskTypeEnum, Set<Long>> riskMap = new EnumMap<>(RiskTypeEnum.class);
    List<RuleFailure> failures = new ArrayList<>();

    safeRule(RiskTypeEnum.REALTIME_VIDEO_RISK, () -> UserRiskCache.getRealTimeVideoRiskData(tableId, riskData), riskMap, failures);
    safeRule(RiskTypeEnum.DEVICE_RISK,         () -> UserRiskCache.getDeviceRiskData(tableId, riskData),         riskMap, failures);
    safeRule(RiskTypeEnum.GPS_NOT_OPEN,        () -> UserRiskCache.getGpsSwitchRiskData(tableId, riskData),      riskMap, failures);
    safeRule(RiskTypeEnum.GPS_EQUAL,           () -> UserRiskCache.getGpsRiskData(tableId, riskData),            riskMap, failures);
    safeRule(RiskTypeEnum.IP_EQUAL,            () -> UserRiskCache.getIPRiskData(tableId, riskData),             riskMap, failures);

    boolean success = failures.isEmpty();
    log.debug("通用风控检测 tableId:{} success:{} riskTypes:{} failures:{}",
            tableId, success, riskMap.keySet(), failures);
    if (!success) {
        log.warn("通用风控检测部分规则失败 tableId:{} failures:{}", tableId, failures);
    }
    return new RiskCheckResult(riskMap, success, failures);
}

private static void safeRule(RiskTypeEnum rule, Supplier<Set<Long>> supplier,
                             Map<RiskTypeEnum, Set<Long>> sink, List<RuleFailure> failures) {
    try {
        Set<Long> hit = supplier.get();
        if (CollectionUtils.isNotEmpty(hit)) {
            sink.put(rule, hit);
        }
    } catch (Exception e) {
        failures.add(new RuleFailure(rule, e.getMessage()));
    }
}
```

`checkRiskSitDown` 改造，对失败采用保守策略：

```java
public static void checkRiskSitDown(Long tableId, Long userId) {
    List<RiskDataCollectionDto> sitDownRiskData = UserRiskCache.getSitDownRiskData(tableId, userId);
    RiskCheckResult result = checkRiskDataResult(tableId, sitDownRiskData);
    if (!result.isSuccess()) {
        log.warn("入座风控规则部分失败,保守拦截 tableId:{} userId:{} failures:{}",
                tableId, userId, result.getFailedRules());
        throw BusinessException.createException(BusinessExceptionTypeEnums.TABLE_RISK_ERROR);
    }
    RiskTypeEnum riskTypeEnum = getUserRiskType(userId, result.getRiskMap());
    if (riskTypeEnum != RiskTypeEnum.CHECK_NO_RISK) {
        log.warn("入座风控检测失败 tableId:{} userId:{} riskTypeEnum:{}", tableId, userId, riskTypeEnum);
        throw BusinessException.createMessageException(riskTypeEnum.getSeatError(), riskTypeEnum.getSeatDesc());
    }
}
```

`RiskTableSnapshot`：

```java
@Getter
public class RiskTableSnapshot {
    private final Long tableId;
    private final BaseTableInfo tableInfo;
    private final List<RiskDataCollectionDto> riskData;
    private final Set<Long> playerUserIds;
    private final Set<Long> activePlayerUserIds;   // PLAYER_USER 中 DEFAULT 状态用户（开局中已参与本手）
    private final Set<Long> allUserIds;            // PLAYER + WATCH + 等待入座
    private final boolean handStarted;
    private final long capturedAt;

    private RiskTableSnapshot(...) { ... }

    public static RiskTableSnapshot capture(BaseTable table) {
        Long tableId = table.getTableId();
        BaseTableInfo tableInfo = BaseTableInfoCache.getTableInfo(tableId);
        List<RiskDataCollectionDto> riskData = UserRiskCache.getTableRiskData(tableId);
        Set<Long> playerUserIds = new HashSet<>(BaseTableUserCache.fetchTableUserIds(tableId, TableUserCacheEnum.PLAYER_USER));
        Set<Long> activePlayerUserIds = new HashSet<>(BaseTableUserCache.fetchTablePlayerUserIds(tableId, UserStatusEnum.DEFAULT));
        Set<Long> allUserIds = new HashSet<>(BaseTableUserCache.fetchAllUidList(tableId));
        boolean handStarted = BaseTableHandCache.isStartIng(tableId);
        return new RiskTableSnapshot(tableId, tableInfo, riskData,
                playerUserIds, activePlayerUserIds, allUserIds, handStarted, System.currentTimeMillis());
    }
}
```

`realtimeCheckRisk` 改造（关键节点）：

```java
public static void realtimeCheckRisk(BaseTable table) {
    Long tableId = table.getTableId();
    try {
        table.setLastRiskCheckTime(System.currentTimeMillis());
        boolean isOldRisked = table.getIsRiskControl();

        RiskTableSnapshot snapshot = RiskTableSnapshot.capture(table);
        RiskCheckResult result = checkRiskDataResult(tableId, snapshot.getRiskData());
        Map<RiskTypeEnum, Set<Long>> riskUserMap = result.getRiskMap();
        table.setIsRiskControl(result.hasRisk());

        if (!isOldRisked && !result.hasRisk() && !result.hasFailure()) {
            return;
        }

        Set<Long> riskUserIds = riskUserMap.values().stream().flatMap(Set::stream).collect(Collectors.toSet());
        List<Long> needStandUsers = computeNeedStandUserIds(snapshot, riskUserIds);

        Map<RiskTypeEnum, Set<Long>> lastRiskUserMap;
        if (!CollectionUtils.isEmpty(needStandUsers)) {
            for (Long standUserId : needStandUsers) {
                RiskTypeEnum riskTypeEnum = getUserRiskType(standUserId, riskUserMap);
                UserStandTypeEnum standTypeEnum = riskPolicyRegistry().getStandType(riskTypeEnum);
                KickerReasonEnum kickerReasonEnum = riskPolicyRegistry().getKickerReason(riskTypeEnum);
                UserLeaveBootstrapService.userStand(standUserId, tableId, standTypeEnum, kickerReasonEnum);
                log.debug("牌桌风控实时检测,风控站起 tableId:{} userId:{} standType:{} reason:{}",
                        tableId, standUserId, standTypeEnum, kickerReasonEnum);
            }
            // 二次确认仍基于 snapshot.riskData（已经拷贝），不重读 cache
            lastRiskUserMap = checkRiskDataResult(tableId, snapshot.getRiskData()).getRiskMap();
            table.setIsRiskControl(MapUtils.isNotEmpty(lastRiskUserMap));
            Set<Long> lastRiskUserIds = lastRiskUserMap.values().stream().flatMap(Set::stream).collect(Collectors.toSet());
            Set<Long> standAndNoRiskUsers = needStandUsers.stream().filter(uid -> !lastRiskUserIds.contains(uid)).collect(Collectors.toSet());
            riskUserMap.forEach((riskType, ids) -> {
                Set<Long> filtered = ids.stream().filter(standAndNoRiskUsers::contains).collect(Collectors.toSet());
                if (!filtered.isEmpty()) {
                    lastRiskUserMap.computeIfAbsent(riskType, k -> new HashSet<>()).addAll(filtered);
                }
            });
        } else {
            lastRiskUserMap = riskUserMap;
        }

        sendCheckRiskNotice(snapshot, table.getIsRiskControl(), needStandUsers, lastRiskUserMap);
    } catch (Exception e) {
        log.error("检测风控发生异常 tableId:{}", tableId, e);
    }
}

private static List<Long> computeNeedStandUserIds(RiskTableSnapshot snapshot, Set<Long> riskUserIds) {
    if (CollectionUtils.isEmpty(riskUserIds)) {
        return new ArrayList<>();
    }
    Set<Long> candidates = new HashSet<>(snapshot.getPlayerUserIds());
    if (snapshot.isHandStarted()) {
        candidates.removeAll(snapshot.getActivePlayerUserIds());
    }
    return candidates.stream().filter(riskUserIds::contains).distinct().collect(Collectors.toList());
}

private static void sendCheckRiskNotice(RiskTableSnapshot snapshot, Boolean isRiskControl,
                                        List<Long> standUserIdList, Map<RiskTypeEnum, Set<Long>> riskUserMap) {
    Long tableId = snapshot.getTableId();
    Set<Integer> tableRiskTypes = MapUtils.isNotEmpty(riskUserMap)
            ? riskUserMap.keySet().stream().map(RiskTypeEnum::getCode).collect(Collectors.toCollection(HashSet::new))
            : Collections.emptySet();

    Set<Long> recipients = new HashSet<>(snapshot.getAllUserIds());
    recipients.addAll(standUserIdList);
    for (Long userId : recipients) {
        RiskTypeEnum typeEnum = getUserRiskType(userId, riskUserMap);
        boolean isStand = standUserIdList.contains(userId);
        String riskTypeDesc = RiskBootstrapService.riskTypeEnumToDesc(tableId, isStand, typeEnum);
        UserRiskNoticeDto noticeDto = new UserRiskNoticeDto();
        noticeDto.setUserId(userId);
        noticeDto.setRiskType(typeEnum.getCode());
        noticeDto.setRiskTypeDesc(riskTypeDesc);
        noticeDto.setTableHasRisk(isRiskControl);
        noticeDto.setTableRiskTypes(new HashSet<>(tableRiskTypes));
        noticeDto.setUserHasRisk(!RiskTypeEnum.CHECK_NO_RISK.equals(typeEnum));
        NetResponseBody netResponseBody = NetResponseBuilder.buildNotice(CommonProto.RISK_CHANGE_NOTICE, noticeDto, tableId);
        SendUtil.sendNotice(userId, netResponseBody);
    }
    log.debug("风控检测通知发送结束 tableId:{} isRiskControl:{} recipientCount:{} standCount:{} riskTypes:{}",
            tableId, isRiskControl, recipients.size(), standUserIdList.size(), riskUserMap.keySet());
}
```

### R-03：`RiskPolicyRegistry` 集中优先级 / Reason / StandType（P1）

**文件**：新增 `RiskPolicyRegistry.java`；修改 `BaseTableRiskService.getRiskReason` 内部委托

```java
@Component
public class RiskPolicyRegistry {

    private static final List<RiskTypeEnum> PRIORITY_ORDER = List.of(
            RiskTypeEnum.REALTIME_VIDEO_RISK,
            RiskTypeEnum.DEVICE_RISK,
            RiskTypeEnum.GPS_NOT_OPEN,
            RiskTypeEnum.GPS_EQUAL,
            RiskTypeEnum.IP_EQUAL
    );

    public List<RiskTypeEnum> getPriorityOrder() { return PRIORITY_ORDER; }

    public KickerReasonEnum getKickerReason(RiskTypeEnum type) {
        if (type == null) return KickerReasonEnum.RISK_STAND;
        switch (type) {
            case REALTIME_VOICE_RISK:
            case REALTIME_VIDEO_RISK: return KickerReasonEnum.RISK_VIDEO_STAND;
            case DEVICE_RISK:         return KickerReasonEnum.RISK_IOS_STAND;
            case GPS_EQUAL:
            case GPS_NOT_OPEN:        return KickerReasonEnum.RISK_GPS_STAND;
            case IP_EQUAL:            return KickerReasonEnum.RISK_IP_STAND;
            default:                  return KickerReasonEnum.RISK_STAND;
        }
    }

    public UserStandTypeEnum getStandType(RiskTypeEnum type) {
        if (type == null) return UserStandTypeEnum.RISK_STAND;
        return Objects.equals(RiskTypeEnum.REALTIME_VIDEO_RISK, type)
                ? UserStandTypeEnum.RISK_VIDEO_STAND
                : UserStandTypeEnum.RISK_STAND;
    }

    public BusinessExceptionTypeEnums getSeatError(RiskTypeEnum type) {
        return type == null ? BusinessExceptionTypeEnums.TABLE_RISK_ERROR : type.getSeatError();
    }
}
```

`BaseTableRiskService` 改造：
- `RISK_PRIORITY_ORDER` 私有常量 → 委托 `RiskPolicyRegistry.getPriorityOrder()`（通过静态 SpringContextUtils 取 bean，类似 `UserRiskCache.baseSystemParamsConfig` 的模式）
- `getRiskReason(RiskTypeEnum)` 内部委托 registry，方法签名保持
- `getUserRiskType(userId, riskResults)` 使用 registry 的 priorityOrder

## 5. 新增组件详细设计

### 5.1 `RiskCheckResult`（frame model）

**包**：`com.dx.game.frame.model.dto.risk`

封装一次风控规则计算结果。`success=false` 表示**至少有一条规则抛异常**，调用方应采取保守策略。

### 5.2 `RuleFailure`（frame model）

**包**：`com.dx.game.frame.model.dto.risk`

单条规则失败信息：规则枚举 + 异常 message。本期不带 throwable，避免日志膨胀。

### 5.3 `RiskTableSnapshot`（frame model）

**包**：`com.dx.game.frame.model.dto.risk`

`realtimeCheckRisk` 一次执行的快照。`capture(BaseTable)` 静态工厂内一次性读全所有依赖 cache，后续算法不重读。

> 注：snapshot 内 `riskData` 是 `UserRiskCache.getTableRiskData` 返回的 `List<RiskDataCollectionDto>`，本身是对 cache 内对象的引用而非深拷贝（19 已校验 cache value 是 mutable shared）。本期不强制深拷贝，因为：1) `realtimeCheckRisk` 在 TableLoopTask 线程串行执行，不会自我冲突；2) 1040 写入虽然异步，但只 set 字段；3) 二次确认（站起后）使用同一份 snapshot.riskData 是有意为之，避免"刚站起 → 重新拉 cache → 看到新上报"导致重复处理。

### 5.4 `RiskPolicyRegistry`（frame service）

**包**：`com.dx.game.frame.service.biz.risk`

集中管理风险优先级、KickerReason、StandType、SeatError。`@Component` Spring bean；静态门面 `BaseTableRiskService` 通过 `SpringContextUtils.getBean(RiskPolicyRegistry.class)` 拿到实例（lazy static 初始化，与 `UserRiskCache.baseSystemParamsConfig` 同模式）。

## 6. 不在本期处理的范围（明确遗留）

- `RiskActionPlanner` 统一动作规划（NOTICE_ONLY / STAND_NOW / STAND_AFTER_HAND）：本期 `realtimeCheckRisk` 仍直接调 `userStand`，`RiskStandLeaveUserVisitor` 仍直接生成 `SpecifyLeaveUserDto`。这块需要和 04 站起 / 18 预站起 / 17 放弃补码联动，单独做。
- `REALTIME_VOICE_RISK` 是否纳入优先级：留作待确认项，等产品/合规决策。
- 风控失败上报指标 / 告警：本期只在日志里 `warn` 输出 `failures`，不接入指标系统（统一在监控基建里做）。

## 10. 验收关键场景

1. **GPS 计算抛 NPE 时不再静默放行**：构造一条 `getGpsRiskData` 抛 NPE 的场景，入座请求被 `TABLE_RISK_ERROR` 拦截；实时轮询日志能看到 `failedRules: [GPS_EQUAL]`。
2. **一致快照**：实时检测过程中模拟一名 player 同时退出，`sendCheckRiskNotice` 不会向已退出用户发 14038；`needStandUsers` 不会包含中途新入座的 player。
3. **优先级集中**：删除 `BaseTableRiskService.RISK_PRIORITY_ORDER` 私有常量后，`getUserRiskType` 行为仍与改造前一致（用 `RiskPolicyRegistry.getPriorityOrder()`）。
4. **`getRiskReason` 行为一致**：5 种 RiskTypeEnum + null + UNKNOWN 输入分别得到与改造前一致的 `KickerReasonEnum`。
5. **日志体积**：1 次 `realtimeCheckRisk` 在无风险桌产生 0 条 info 日志；有风险桌产生 ≤ 3 条 info（开始、站起、通知发送结束），所有 IP/GPS payload 在 debug 级。
6. **`RiskGraphAssembler`（19 引入）**：1041 协议响应字段无变化；GPS 缺失场景仍正常返回。
7. **`checkRiskSitDown` 错误码**：规则成功+用户命中 IP_EQUAL → `TABLE_RISK_ERROR`（旧行为）；规则失败 → `TABLE_RISK_ERROR`（保守拦截，新行为）；规则成功+无风险 → 通过。
