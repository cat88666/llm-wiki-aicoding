# 入座重构设计文档

> 对应分析文档：`docs/Junk-游戏准备重构/02-入座.md`
> 设计日期：2026-06-04
> 工程：dx-game-frame + dx-game-guandan
> 分支：20260615/junk
> 前置依赖：`01-进入房间-重构设计.md`（复用其 ValidatorChain / StateService / 表级锁框架）

## 1. 背景与目标

当前入座流程编排散落在 `ClubUserSitDownDefault.handle()` / `PrivateUserSitDownDefault.handle()`，校验、限流、筹码、状态迁移、后置混在一起，存在：

- P0：占座转入座 `handle()` 直接读 `TableUserWalletCache.getWallet().getNextBringInChipScore()` 无空保护（`UserSitDownDefault.java:157`），钱包缓存被清时 NPE，玩家卡占座
- P0：WATCH_USER→PLAYER_USER 迁移非原子，`OccupyToSitDownTask` 由 EventManager 调度时可能与主动入座并发（`BaseUserSitDownDefault.java:101-112`）
- P1：HALL/RACE 的 `handle()` 是空方法，客户端误发请求会静默成功无响应（`HallUserSitDownDefault.java:17-19`、`RaceUserSitDownDefault.java:17-19`）
- P1：`occupyAddChipNotice` 整体被 try-catch 吞异常，且超时任务注册在 try 块内，注册失败玩家永远卡 OCCUPY（`BaseOccupyUserService.java:145-237`）
- P1：限流 `checkSentinel` 在 `checkChipScore` 之前消耗令牌，余额不足→充值→重试被限流拦截（`ClubUserSitDownDefault.java:26-30`）
- P2：`BaseUserSitDownHandler` 与 `BaseUserSitDownDefault` 三个方法高度重复（S-09）；大量 `GsonUtils.toJson` info 日志（S-10）；`onlyCheck` 死字段（S-11）

目标：消除 P0/P1 问题，结构性拆分编排、校验、状态写入三个关注点，复用 01-进入房间已建的 ValidatorChain / StateService 框架，保持协议和 DTO 完全兼容。

## 2. 决策记录

| 决策项 | 结论 |
|--------|------|
| 主导方案 | Frame 主导：`SitDownFlowService` / `SitDownValidatorChain` / `SitDownStateService` 放 frame，guandan 提供 contributor / stateHandler 扩展。**与 02-入座.md 落地设计中"FlowService 放 guandan"的初稿不同**——改放 frame，理由：与 `01-进入房间` 决策一致，且 R-08 明确要求复用进房已建的校验链框架 |
| 校验/限流顺序 | `checkSentinel` 从 `checkChipScore` 之前移到之后（R-04），余额不足不再消耗限流令牌 |
| HALL/RACE 空 handler | 显式抛 `NOT_SUPPORT` 业务异常（R-02），不做开关直接替换 |
| 占座超时任务注册 | 从 `occupyAddChipNotice` 的 try-catch 中剥离，**先注册超时任务再发通知**（R-03），保证通知失败也不会卡 OCCUPY |
| 线程模型 | 保留 TABLE 线程入口；`SitDownStateService` 内用表级 `ReentrantLock` 串行化 WATCH→PLAYER 迁移与占座转入座，覆盖 `OccupyToSitDownTask` 由 EventManager 调度的并发窗口（R-09 / S-01 / S-03） |
| `onlyCheck` 字段 | `@Deprecated` 标注保留，不删（协议兼容） |
| `BaseUserSitDownHandler` | `@Deprecated` 标注；Handler 直接委托 `SitDownFlowService`，消除与 `BaseUserSitDownDefault` 的重复（R-06） |
| 灰度开关 | 不做开关，直接替换，依赖测试覆盖（与 01 一致） |
| `UserItem.status` FSM | 复用 01-进入房间决策的状态语义，入座只新增 `OCCUPY → DEFAULT/SIT_DOWN_WAIT_ONE_HAND` 合法转换，不重定义状态枚举 |

## 3. 整体架构

### 改造前

```
UserSitDownCommand (TABLE线程, dx-game-frame-cmd)
  └─ GameHandleFactory.getHandle(spaceType)
       ├─ HallUserSitDownHandler  → HallUserSitDownDefault.handle()    — 空方法(S-04)
       ├─ RaceUserSitDownHandler  → RaceUserSitDownDefault.handle()    — 空方法(S-04)
       ├─ ClubUserSitDownHandler  → ClubUserSitDownDefault.handle()
       └─ PrivateUserSitDownHandler → PrivateUserSitDownDefault.handle()
            ├─ UserSitDownBootstrapService.init()  — 静态门面(S-08)
            ├─ checkSitDown()        — 踢出/座位/权限/风控
            ├─ checkSentinel()       — 三级限流（顺序在 chip 之前, S-06）
            ├─ checkChipScore()      — 私人房跳过/余额校验
            ├─ handler()             — WATCH→PLAYER 非原子迁移(S-01)，占座 wallet 无空保护(S-02)
            └─ after()               — 事件+响应+通知+占座补码（吞异常 S-05）

BaseUserSitDownHandler(抽象) — checkChipScore/handler/after 与 BaseUserSitDownDefault 重复(S-09)
```

### 改造后

```
UserSitDownCommand (TABLE线程, 不变)
  └─ GameHandleFactory.getHandle(spaceType)
       ├─ Hall/RaceUserSitDownDefault.handle() — 显式抛 NOT_SUPPORT(R-02)
       └─ Club/PrivateUserSitDownDefault.handle() — 委托 SitDownFlowService.sitDown(context)

SitDownFlowService (frame, Spring Bean)
  ├─ init                       — UserSitDownBootstrapService.init（私人审核房由 GdUserSitDownDefault.init 覆盖）
  ├─ SitDownValidatorChain      — 踢出/座位/权限/风控 + ISitDownValidatorContributor 扩展
  ├─ checkChipScore             — 私人房跳过/余额校验
  ├─ checkSentinel              — 移到 chipScore 之后(R-04)
  ├─ SitDownStateService        — 表级锁 + WATCH→PLAYER + 占座状态机(R-09)
  └─ after                      — 事件/响应/通知；占座补码先注册超时任务(R-03)

dx-game-guandan 扩展：
  ├─ GdSitDownValidatorContributor — 掼蛋特有入座校验扩展点
  └─ GdSitDownStateService         — 结算重坐补码恢复、ensureOccupyUserSitDown（实现 IOccupyResitHandler）
```

- 协议号 1001、伪协议 1016、`UserSitDownReqDto`、`UserSitDownRespDto`、通知 14001/14022 字段不变
- `UserSitDownCommand`、`HandlerSpaceService` 注册、`@ThreadModel(TABLE)` 不变
- `GdUserSitDownDefault.init()`（私人审核房 PRIVATE_SCORE）逻辑保留不动
- `BaseUserSitDownHandler` 标记 `@Deprecated` 但不删除（麻将等其他游戏可能仍继承）

## 4. 安全改造（R-01 ~ R-05）

### R-01：占座入座 wallet 空值保护（S-02）

**文件**：`UserSitDownDefault.java`（`biz.bootstrap.def`），`handle()` 中 L156-157

改法：

```java
TableUserWallet tableUserWallet = TableUserWalletCache.getWallet(tableId, userId, WalletConstant.walletType);
if (tableUserWallet == null) {
    log.warn("占座入座-钱包缓存为空，跳过本次占座转入座 tableId:{} userId:{}", tableId, userId);
    return;   // 兜底：保持 OCCUPY，等待下次带入或占座超时，而非 NPE 卡死
}
BigDecimal nextBringInChipScore = tableUserWallet.getNextBringInChipScore();
```

> 兜底策略选"保持 OCCUPY 等待重试"，而非抛业务异常——占座转入座由任务触发，无客户端等待响应，直接返回更安全。

### R-02：HALL/RACE 空 handler 显式拒绝（S-04）

**文件**：`HallUserSitDownDefault.java`、`RaceUserSitDownDefault.java`

```java
public void handle(UserSitDownReqDto req) {
    log.warn("大厅/比赛桌不支持手动入座 tableId:{} userId:{}", req.getTableId(), req.getUserId());
    throw BusinessException.createException(BusinessExceptionTypeEnums.NOT_SUPPORT_OPERATION);
```

### R-03：占座补码通知超时任务先注册（S-05）

**文件**：`BaseOccupyUserService.occupyAddChipNotice()`（L145-237）

把 `OccupyTimeOutTask` 注册从 try 块**末尾**提到 try 块**最前面**（紧跟占座校验之后），保证即使后续通知组装/发送抛异常，超时兜底任务已注册：

```java
public void occupyAddChipNotice(AddChipTypeEnum addChipTypeEnum, Long tableId, Long userId, BigDecimal amt) {
    if (!BaseTableUserCache.hasOccupyUser(tableId, userId)) {
        log.error("不是占座玩家无法占座处理 tableId:{} userId:{} addChipTypeEnum:{}", tableId, userId, addChipTypeEnum);
        return;
    }
    // R-03: 超时任务先注册，与通知解耦，注册失败显式抛出
    int countdown = resolveOccupyCountdown(tableId, userId, addChipTypeEnum);
    String eventIdKey = EventUtils.getKey(OnceEventTypeEnum.OCCUPY_TIMEOUT_TASK, tableId, userId);
    eventManager.publish(new OccupyTimeOutTask(tableId, userId, eventIdKey, countdown * 1000));

    try {
        // 组装并广播 14022 占座补码通知
        // publish UserSitOccupyFinishEvent
    } catch (Exception e) {
        log.error("占座补码通知发送异常（超时任务已注册，玩家不会卡死） tableId:{} userId:{}", tableId, userId, e);
    }

```

### R-04：限流顺序调整（S-06）

不修改 `checkSentinel` 本身，只在新的 `SitDownFlowService` 中把它放到 `checkChipScore` **之后**：

```
旧：checkSitDown → checkSentinel → checkChipScore → handler
新：validate(链) → checkChipScore → checkSentinel → applyState
```

余额不足在 `checkChipScore` 即抛出，限流令牌不被消耗，充值后可立即重试。

### R-05：日志收敛（S-10）

`UserSitDownCommand`、`BaseUserSitDownCheckService`、`BaseUserSitDownDefault`、`UserSitDownDefault` 中所有 `GsonUtils.toJson(context/userItem/playUserList)` 的 info 日志降为 debug。

主链路只保留两条 info：

```java

log.info("玩家入座成功 tableId:{} userId:{} status:{}", tableId, userId, status);
```

## 5. 新增组件详细设计

### 5.1 `SitDownFlowService`（frame）

**包**：`com.dx.game.frame.service.biz.sitdown`

```java
@Slf4j
@Component
public class SitDownFlowService {

    @Resource
    private SitDownValidatorChain validatorChain;
    @Resource
    private SitDownStateService   stateService;

    public void sitDown(UserSitDownContext context) {
        Long tableId = context.getTableId();
        Long userId  = context.getRequest().getUserId();

        // 1. 初始化（私人审核房 PRIVATE_SCORE 由 GdUserSitDownDefault.init 覆盖）
        UserSitDownBootstrapService.init(context);

        // 2. 校验链：踢出 / 座位 / 权限 / 风控 + 游戏侧扩展
        validatorChain.validate(context);

        // 3. 筹码校验（私人房跳过余额校验）
        BaseUserSitDownDefault dataBoot = GameBaseBootstrapFactory.getDataBoot(BaseUserSitDownDefault.class);
        dataBoot.checkChipScore(context);

        // 4. R-04：限流移到筹码校验之后，余额不足不消耗令牌
        BaseUserSitDownCheckService.checkSentinel(tableId, userId);

        // 5. 状态迁移：表级锁 WATCH→PLAYER + 占座（R-09 / S-01 / S-03）
        stateService.applyState(context, dataBoot);

        // 6. 后置：事件 / 响应 / 通知 / 占座补码兜底（guandan 侧 after 覆盖保留）
        dataBoot.after(context);
        log.info("玩家入座成功 tableId:{} userId:{} chairId:{}", tableId, userId, context.getRequest().getChairId());
    }
}
```

> `init / checkChipScore / after` 仍走原有静态门面与多态 `dataBoot`（guandan `UserSitDownBootstrap` 覆盖 `checkChipScore/after` 不变），FlowService 只负责重排顺序和插入校验链/状态服务，最小化对既有逻辑的改动。

### 5.3 `SitDownStateService`（frame）

**包**：`com.dx.game.frame.service.biz.sitdown`

```java
@Slf4j
@Component
public class SitDownStateService {

    /** 游戏侧可选：占座转入座/结算重坐恢复（清补码标记、唤醒开局）。 */
    @Resource(required = false)
    private IOccupyResitHandler occupyResitHandler;

    private final ConcurrentHashMap<Long, ReentrantLock> tableLocks = new ConcurrentHashMap<>();

    /** 主动入座：WATCH→PLAYER 迁移 + 占座/入座，表级锁串行化。 */
    public void applyState(UserSitDownContext context, BaseUserSitDownDefault dataBoot) {
        Long tableId = context.getTableId();
        ReentrantLock lock = tableLocks.computeIfAbsent(tableId, k -> new ReentrantLock());
        lock.lock();
        try {
            dataBoot.handler(context);   // 内含 WATCH→PLAYER 迁移、userSitDown / userSitDownOccupy
        } finally {
            lock.unlock();
        }
    }

    /**
     * 占座转入座：供 OccupyToSitDownTask / 结算重坐走同一把表级锁，
     * 避免 EventManager 调度的占座转入座与玩家主动入座并发（S-01 / S-03）。
     */
    public void occupyToSitDown(Long tableId, Long userId) {
        ReentrantLock lock = tableLocks.computeIfAbsent(tableId, k -> new ReentrantLock());
        lock.lock();
        try {
            if (occupyResitHandler != null) {
                occupyResitHandler.onOccupyToSitDown(tableId, userId);
            }
        } finally {
            lock.unlock();
        }
    }

    /** 桌解散时调用，释放锁对象防止内存泄漏。 */
    public void releaseTableLock(Long tableId) {
        tableLocks.remove(tableId);
    }
}
```

**接口**（frame）：

```java
public interface IOccupyResitHandler {
    /** 占座转入座/结算重坐后由游戏侧恢复状态（如清补码标记、唤醒开局）。 */
    void onOccupyToSitDown(Long tableId, Long userId);
}
```

## 10. 验收关键场景

1. **占座转入座钱包缺失**：占座状态下钱包缓存被清，`OccupyToSitDownTask` 触发后不抛 NPE，保持 OCCUPY 等待重试/超时（R-01）
2. **HALL/RACE 入座**：向大厅/比赛桌发 1001，返回 `NOT_SUPPORT_OPERATION` 业务异常，不再静默无响应（R-02）
3. **占座通知失败不卡死**：模拟 `occupyAddChipNotice` 通知组装抛异常，超时任务仍已注册，玩家到期被站起（R-03）
4. **余额不足后充值重试**：非私人房余额不足被 `checkChipScore` 拦截，限流令牌未消耗，充值后立即重试可入座（R-04）
5. **并发占座转入座**：`OccupyToSitDownTask`（EventManager）与玩家主动入座并发，表级锁保证同桌状态迁移线性化，不产生双 PLAYER_USER（R-09 / S-01）
6. **私人审核房入座**：`GdUserSitDownDefault.init` 的 PRIVATE_SCORE 覆盖与房主 `autoBringLimit` 逻辑不变，回归通过
7. **入座日志**：成功入座只有两条 info（入座、入座成功），无 `GsonUtils.toJson` 大对象输出（R-05）
