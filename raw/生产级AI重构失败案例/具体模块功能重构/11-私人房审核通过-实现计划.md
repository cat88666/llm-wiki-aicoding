## R-04：审核消息集中校验（P1-2，先行）
- **新增**：`AuditMessageValidator`（frame-service）
- **步骤**：`dealBringInReviewDto` 入口集中校验 审核单状态(INIT/PROCESSING)/金额匹配/桌子存在未解散/holder 一致/申请人存在/未过期；脏消息早拒绝。
- **测试**：错桌/错 holder/过期/非 INIT → 拒绝。
- **回滚**：移除校验器，恢复 validateAuditMessage。

## R-01：Redis CAS 审核幂等（P0-1，复用 03 R-03）
- **新增/复用**：`AuditIdempotencyService`（与 03 R-03 同一 Redis CAS 基础）
- **步骤**：审核单 ID 幂等键，CAS INIT→PROCESSING→PASS_APPLIED（REJECT/TIMEOUT 终态同理）；钱包 RPC eventId 关联审核单 ID。本地 `CACHE`/`AUDIT_CACHE`/`AUDIT_PASS_CACHE` 降为性能优化。
- **测试**：PASS/REJECT/TIMEOUT 重复消息 CAS 拦截不重复上分；多节点/重启回放幂等。
- **回滚**：恢复本地 cache 幂等。
- **⚠ 待确认**：审核单是否允许新增 PROCESSING/PASS_APPLIED 中间态；否则用钱包 eventId 幂等（设计 §10）。

## R-02：资金/状态可补偿 + 占座转入座表级锁（P0-2，复用 03 R-02）
- **新增**：`AuditApplyPlan`(frame-model) + `AuditWalletApplyService` + `AuditStateApplyService`
- **步骤**：PASS 钱包上分(eventId=auditId)→本地钱包刷新→状态推进→通知，分步状态(quotaApplied/walletApplied/localStateApplied/noticeSent)可幂等重试，失败进补偿队列；占座转入座 `new OccupyToSitDownTask(...).run()` 改 `sitDownStateService.occupyToSitDown`(02 表级锁)；`auditPassUserNotExistsTable` 策略显式化。
- **测试**：PASS 各步失败可定位重试；占座用户 PASS 入座(表级锁)；用户已退出 PASS 策略。
- **回滚**：恢复 processGuandanAuditTransferIn 直链 + .run。
- **⚠ 跨功能**：复用 02 SitDownStateService、03 ResultApplier；回归占座转入座 + 结算继续。

## R-03：REJECT/TIMEOUT 结算补码迁出（P1-1）
- **新增**：`SettlementAuditRejectHandler`（guandan）
- **步骤**：把 `handleSettleComplementAuditReject` 的 移除站起/占座任务 + `SettleChipComplementService`/`LevelUpBonusHandler`/`userStand` 处理迁到 handler；带入服务只转发审核结果。
- **测试**：REJECT 不残留占座转入座任务，结算补码恢复/站起正确；TIMEOUT 状态一致。
- **回滚**：处理迁回 UserBringInBootstrap。
