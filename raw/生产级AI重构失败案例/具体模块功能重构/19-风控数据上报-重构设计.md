# 风控数据上报重构设计文档

> 对应分析文档：`docs/Junk-游戏准备重构/19-风控数据上报.md`
> 设计日期：2026-06-04
> 工程：dx-game-frame
> 分支：20260615/junk
> 涉及协议：1040 `RISK_DATA_COLLECTION`、1041 `RISK_DATA_GRAPH`

## 1. 背景与目标

当前 1040/1041 实现集中在 `RiskDataCollectionCommandDefault` 和 `RiskDataGraphCommandDefault`，配合 `UserRiskCache` 本地 Guava 缓存：

- **P0**：`UserRiskCache.USER_RISK_DATA_CACHE` key 仅有 `userId`，跨桌或换桌时 IP/GPS/设备数据互相串用。
- **P0**：1040 直接读取 `tableInfo.getRoomCode()`、`riskData.getIp()` 等字段，缺空值保护；Redis 同步与主响应耦合，任一异常导致 1040 系统错误。
- **P1**：1041 关系图直接 `riskData.getLatitude().doubleValue()`，GPS 缺失易 NPE；只算 IP/GPS 三种关系，未覆盖 08-风控 的设备/视频规则，UI 展示与实时风控站起结果不一致。
- **P1**：1040/1041 大量 `info` 级 `GsonUtils.toJson(...)` 打印 IP、GPS、整桌风险数据，CPU 与隐私双重风险。

目标：消除所有 P0 风险（缓存维度、主响应解耦、空值），统一 1041 关系图与 08-风控 规则结果，结构性拆分编排/缓存/Redis同步/关系图组装，保持协议和 DTO 完全兼容。

## 2. 决策记录

| 决策项 | 结论 |
|--------|------|
| 主导方案 | Frame 主导：所有改造落在 `dx-game-frame`，掼蛋无需新增类（1040/1041 当前实现全部在 frame） |
| 缓存 key 重构方式 | 直接替换 cache 内部 key 类型为 `RiskDataCacheKey(tableId, userId)`；对外方法签名（`tableId, userId`）保持不变 |
| 旧 `Long` key 缓存兼容 | 不做双写；3 小时 TTL 自然淘汰，验收测试覆盖跨桌污染消除即可 |
| Redis 同步隔离 | 抽出 `RiskDataRedisSyncService`，try/catch 包裹，失败仅 warn 不影响 1040 响应 |
| 1041 风险类型统一 | 关系图复用 `BaseTableRiskService.checkRiskData(tableId, riskData)` 的结果。新增可选字段 `riskTypeList` 在 `UserRiskInfoDto` 中表示用户级风险（设备/视频/GPS未开），保留 `riskGraphList` 表示用户间关系（IP相同/GPS相近） |
| 灰度开关 | 不做开关，直接替换，依赖测试与日志覆盖 |
| 日志收敛 | IP/GPS/整桌风险数据从 `info` 降为 `debug`；info 仅保留 `tableId/userId/respUserCount/riskTypeCount` 等可观测元数据 |
| 关系图缓存维度变更对 08-风控 影响 | 08-风控 通过 `UserRiskCache` 同一组方法读取，方法签名不变；R-01 对 08 透明 |

## 3. 整体架构

### 改造前

```
RiskDataCollectionCommand (PLAYER线程, 1040)
  └─ RiskDataCollectionCommandDefault.handleCommand(req)
       ├─ UserRiskCache.refreshRiskData(req)           — userId-only cache
       ├─ UserRiskCache.getUserRiskData(tableId,userId) — userId-only cache
       ├─ syncRiskDataToRedis(tableId, userId)         — 直接抛异常会落到外层 catch
       │    ├─ tableInfo.getRoomCode()  无空保护
       │    └─ riskData.getIp()/getLatitude()/getLongitude() 无空保护
       └─ SendUtil.send(...)

RiskDataGraphCommand (TABLE线程, 1041)
  └─ RiskDataGraphCommandDefault.handleCommand(req)
       ├─ UserRiskCache.getTableRiskData(tableId)
       ├─ getUserRiskGraph(...)                        — 内置 IP/GPS 三规则
       │    └─ riskData.getLatitude().doubleValue()    GPS缺失 NPE，外层 catch 静默吞
       └─ SendUtil.send(...)

UserRiskCache (Guava LoadingCache<Long, RiskDataCollectionDto>)
  ├─ key = userId 跨桌串用
  └─ 3h TTL
```

### 改造后

```
RiskDataCollectionCommand (PLAYER线程, 1040)
  └─ RiskDataCollectionCommandDefault.handleCommand(req)
       ├─ UserRiskCache.refreshRiskData(req)           — (tableId,userId) cache
       ├─ UserRiskCache.getUserRiskData(tableId,userId)
       ├─ riskDataRedisSyncService.syncSafely(tableId, userId, riskData)
       │    └─ try/catch warn — 失败不影响主响应
       └─ SendUtil.send(...)

RiskDataGraphCommand (TABLE线程, 1041)
  └─ RiskDataGraphCommandDefault.handleCommand(req)
       └─ riskGraphAssembler.assemble(tableId, currentUserId)
            ├─ UserRiskCache.getTableRiskData(tableId)
            ├─ BaseTableRiskService.checkRiskData(tableId, riskData)  — 5规则统一
            ├─ buildPairwiseEdges(riskData, tableInfo)                — IP相同/GPS相近，GPS空值短路
            └─ 装配 RiskDataGraphRespDto
                 ├─ riskGraphList     — 已有 pairwise 关系
                 └─ riskTypeList      — 新增可选 user-level 风险类型

UserRiskCache (Guava LoadingCache<RiskDataCacheKey, RiskDataCollectionDto>)
  ├─ key = (tableId, userId)
  └─ 3h TTL

RiskDataRedisSyncService (新)
  └─ syncSafely(tableId, userId, riskData)
       ├─ tableInfo/riskData 空值短路并 warn
       ├─ IP/GPS 字段缺省值兜底
       └─ try/catch warn — 异常不上抛

RiskGraphAssembler (新)
  ├─ assemble(tableId, currentUserId) → RiskDataGraphRespDto
  ├─ 复用 BaseTableRiskService.checkRiskData
  └─ buildPairwiseEdges GPS 空值短路
```

**兼容性原则**：
- 协议号 1040/1041 不变；`RiskDataCollectionReqDto`、`RiskDataCollectionDto`、`RiskDataGraphReqDto`、`RiskDataGraphRespDto` 字段不删不改；`UserRiskInfoDto.riskTypeList` 为新增 **可选** 字段，老客户端忽略不破坏。
- `UserRiskCache.initRiskDataEnterRoom/refreshRiskData/getUserRiskData/getTableRiskData/getSitDownRiskData/getRealTimeVideoRiskData/getDeviceRiskData/getGpsSwitchRiskData/getGpsRiskData/getIPRiskData` 方法签名保持不变（参数仍是 `tableId, userId` / `tableId`）。
- 改造对 `BaseTableRiskService`、`GdUserCheckCommandBootstrap`、`GdUserLeaveBootstrapService`、`SettleTableSummaryService`、`RiskStandLeaveUserVisitor`、`EnterRoomFlowService` 透明。

```java
public static RiskDataCollectionDto getUserRiskData(Long tableId, Long userId) {
    RiskDataCollectionDto riskData = USER_RISK_DATA_CACHE.getUnchecked(new RiskDataCacheKey(tableId, userId));
    BaseTableInfo tableInfo = BaseTableInfoCache.getTableInfo(tableId);
```

### R-02：1040 主响应与 Redis 同步解耦（P0）

**文件**：`RiskDataCollectionCommandDefault.java`、新增 `RiskDataRedisSyncService.java`

抽出 Redis 同步，主响应不受其异常影响：

```java
@Slf4j
@Component
public class RiskDataRedisSyncService {

    @Resource
    private HallTableUserSeatRedisService hallTableUserSeatRedisService;

    public void syncSafely(Long tableId, Long userId, RiskDataCollectionDto riskData) {
        if (riskData == null) {
            log.warn("syncRiskData-riskData为空，跳过同步 tableId:{} userId:{}", tableId, userId);
            return;
        }
        BaseTableInfo tableInfo = BaseTableInfoCache.getTableInfo(tableId);
        if (tableInfo == null) {
            log.warn("syncRiskData-tableInfo为空，跳过同步 tableId:{} userId:{}", tableId, userId);
            return;
        }
        try {
            RiskDataDto riskDataDto = new RiskDataDto();
            riskDataDto.setIp(riskData.getIp());
            riskDataDto.setLatitude(riskData.getLatitude());
            riskDataDto.setLongitude(riskData.getLongitude());
            String roomCode = GameUtils.getRoomCode(tableInfo.getRoomCode(), tableInfo.getTableCode(), GameConstant.topGameTypeEnum);
            hallTableUserSeatRedisService.syncRiskData(roomCode, tableId, userId, riskDataDto);
        } catch (Exception e) {
            log.warn("syncRiskData-Redis同步失败，不影响主响应 tableId:{} userId:{}", tableId, userId, e);
        }
    }
}
```

### R-03：1041 GPS 空值保护 + 风险类型统一（P1）

**文件**：`RiskDataGraphCommandDefault.java`（瘦身委托）、新增 `RiskGraphAssembler.java`

新增 `RiskGraphAssembler`：

```java
@Slf4j
@Component
public class RiskGraphAssembler {

    public RiskDataGraphRespDto assemble(Long tableId, Long currentUserId) {
        List<RiskDataCollectionDto> tableRiskData = UserRiskCache.getTableRiskData(tableId);
        if (tableRiskData == null) {
            throw BusinessException.createException(BusinessExceptionTypeEnums.TABLE_RISK_ERROR);
        }
        BaseTableInfo tableInfo = BaseTableInfoCache.getTableInfo(tableId);
        if (tableInfo == null) {
            log.warn("assembleRiskGraph-tableInfo缺失 tableId:{}", tableId);
            throw BusinessException.createException(BusinessExceptionTypeEnums.TABLE_RISK_ERROR);
        }

        // 1) 复用 08-风控 规则引擎：得到每用户的所有风险类型
        Map<RiskTypeEnum, Set<Long>> riskMap = BaseTableRiskService.checkRiskData(tableId, tableRiskData);

        // 2) 已有的 pairwise 关系（IP 相同 / GPS 接近），GPS 空值短路
        Map<Long, List<UserRiskInfoWrapperDto>> graphConnect = buildPairwiseEdges(tableRiskData, tableInfo);

        // 3) 装配响应
        Map<Long, RiskDataCollectionDto> riskDataMap = tableRiskData.stream()
                .collect(Collectors.toMap(RiskDataCollectionDto::getUserId, p -> p, (a, b) -> a));
        Map<Long, UserFriendRemarkDto> friendRemarkMap = UserFriendCache.getFriendRemarkCache(
                currentUserId,
                tableRiskData.stream().map(RiskDataCollectionDto::getUserId).collect(Collectors.toSet()));
        List<BaseUserItem> playUserList = BaseTableUserCache.fetchTableUsers(tableId, TableUserCacheEnum.PLAYER_USER);

        RiskDataGraphRespDto resp = new RiskDataGraphRespDto();
        resp.setIsRiskControl(!riskMap.isEmpty());
        List<UserRiskInfoDto> riskPlayerList = new ArrayList<>();
        for (BaseUserItem item : playUserList) {
            UserBaseInfo baseInfo = UserBaseInfoCache.getUserBaseInfo(item.getUserId());
            UserRiskInfoDto info = new UserRiskInfoDto();
            info.setUserId(item.getUserId());
            info.setNickName(baseInfo == null ? "" : baseInfo.getNickName());
            info.setChairId(BaseTableUserCache.fetchUserChairId(tableId, item.getUserId()));
            info.setRemarkName(friendRemarkMap.getOrDefault(item.getUserId(), new UserFriendRemarkDto()).getUserNameRemark());
            info.setTopImagesUrl(baseInfo == null ? "" : baseInfo.getTopImagesUrl());
            if (graphConnect.containsKey(item.getUserId())) {
                info.setRiskGraphList(graphConnect.get(item.getUserId()));
            }
            RiskDataCollectionDto riskData = riskDataMap.get(item.getUserId());
            if (riskData != null) {
                info.setGpsImposeSwitch(riskData.getGpsImposeSwitch());
            }
            // R-04: 新增用户级风险类型（设备/视频/GPS未开）
            info.setRiskTypeList(collectUserRiskTypes(item.getUserId(), riskMap));
            riskPlayerList.add(info);
        }
        resp.setRiskPlayerList(riskPlayerList.stream()
                .sorted(Comparator.comparing(UserRiskInfoDto::getChairId))
                .collect(Collectors.toList()));
        return resp;
    }

    private List<Integer> collectUserRiskTypes(Long userId, Map<RiskTypeEnum, Set<Long>> riskMap) {
        return riskMap.entrySet().stream()
                .filter(e -> e.getValue().contains(userId))
                .map(e -> e.getKey().getCode())
                .collect(Collectors.toList());
    }

    private Map<Long, List<UserRiskInfoWrapperDto>> buildPairwiseEdges(
            List<RiskDataCollectionDto> riskData, BaseTableInfo tableInfo) {
        Map<Long, List<UserRiskInfoWrapperDto>> adj = new HashMap<>(riskData.size());
        for (RiskDataCollectionDto a : riskData) {
            List<UserRiskInfoWrapperDto> resList = new ArrayList<>();
            for (RiskDataCollectionDto b : riskData) {
                if (b.getUserId().equals(a.getUserId())) {
                    continue;
                }
                UserRiskInfoWrapperDto edge = new UserRiskInfoWrapperDto();
                edge.setUserId(b.getUserId());
                edge.setChairId(BaseTableUserCache.fetchUserChairId(tableInfo.getTableId(), b.getUserId()));

                boolean gpsOn = tableInfo.getGpsImposeSwitch() != null && tableInfo.getGpsImposeSwitch() == 2;
                boolean gpsValid = isGpsValid(a) && isGpsValid(b);
                if (gpsOn && gpsValid) {
                    edge.setDistance(GeoCalculator.calculateDistance(
                            a.getLatitude().doubleValue(), a.getLongitude().doubleValue(),
                            b.getLatitude().doubleValue(), b.getLongitude().doubleValue()));
                }

                List<Integer> typeList = new ArrayList<>();
                if (tableInfo.getIpImposeSwitch() != null && tableInfo.getIpImposeSwitch() == 2
                        && a.getIp() != null && a.getIp().equals(b.getIp())) {
                    typeList.add(RiskTypeEnum.IP_EQUAL.getCode());
                }
                if (gpsOn && gpsValid
                        && isNearby(a, b, tableInfo.getLimitToLessThan())) {
                    typeList.add(RiskTypeEnum.GPS_EQUAL.getCode());
                }
                if (gpsOn && (a.getGpsImposeSwitch() == null || a.getGpsImposeSwitch() != 2
                        || b.getGpsImposeSwitch() == null || b.getGpsImposeSwitch() != 2)) {
                    typeList.add(RiskTypeEnum.GPS_NOT_OPEN.getCode());
                }
                if (!typeList.isEmpty()) {
                    edge.setTypeList(typeList);
                    resList.add(edge);
                }
            }
            adj.put(a.getUserId(), resList);
        }
        return adj;
    }

    private boolean isGpsValid(RiskDataCollectionDto data) {
        return data != null && data.getLatitude() != null && data.getLongitude() != null
                && data.getLatitude().compareTo(BigDecimal.ZERO) != 0
                && data.getLongitude().compareTo(BigDecimal.ZERO) != 0;
    }

    private boolean isNearby(RiskDataCollectionDto a, RiskDataCollectionDto b, Integer limitDistance) {
        if (limitDistance == null) {
            return false;
        }
        BigDecimal distance = GeoCalculator.calculateDistance(
                a.getLatitude().doubleValue(), a.getLongitude().doubleValue(),
                b.getLatitude().doubleValue(), b.getLongitude().doubleValue());
        return distance.compareTo(new BigDecimal(limitDistance)) < 0;
    }
}
```

## 5. 新增组件详细设计

### 5.1 `RiskDataCacheKey`（frame，`UserRiskCache` 嵌套静态类）

`tableId` + `userId` 不可变组合 key，`@EqualsAndHashCode` 基于两个字段。无单独文件，作为 `UserRiskCache` 内的 `public static final class` 暴露给同包测试即可。

### 5.2 `RiskDataRedisSyncService`（frame，`com.dx.game.frame.service.biz.risk`）

职责：将原 `RiskDataCollectionCommandDefault.syncRiskDataToRedis` 私有方法升级为独立 Spring Bean。
- 包含 `riskData`/`tableInfo` 空保护。
- 整体 `try/catch warn`，Redis 同步异常不抛给上游。
- 后续接入 metrics（埋点 hook 留 `// TODO 接入风控同步指标` 注释，本期不强行落地）。

### 5.3 `RiskGraphAssembler`（frame，`com.dx.game.frame.service.biz.risk`）

职责：组装 1041 响应。
- 复用 `BaseTableRiskService.checkRiskData` 得到全规则风险结果。
- 单独维护 pairwise 边（IP 相同 / GPS 接近）以兼容旧字段 `riskGraphList`。
- 新增 `riskTypeList` 字段反映用户级风险（设备 / 视频 / GPS未开 等所有 5 类）。
- 内部 GPS/IP 字段空值检查全面。

## 11. 验收关键场景

1. **跨桌防串用**：用户 A 在桌 T1 上报 IP=1.1.1.1，再进入桌 T2 上报 IP=2.2.2.2。`UserRiskCache.getUserRiskData(T1, A) -> 1.1.1.1`，`getUserRiskData(T2, A) -> 2.2.2.2`，两份独立。
2. **Redis 同步失败不影响主响应**：Mock `HallTableUserSeatRedisService.syncRiskData` 抛 RuntimeException，1040 仍返回正常响应，仅 warn 日志。
3. **GPS 缺失不 NPE**：构造缺 latitude/longitude 的玩家，请求 1041，返回不抛系统异常，距离字段为空，GPS_NOT_OPEN 类型仍正确判定。
4. **1041 设备风险展示**：桌开启 onlyIosDeviceSwitch=2，加入非 iOS 玩家，1041 该玩家 `riskTypeList` 包含 `DEVICE_RISK.code`。
5. **日志收敛**：1040/1041 info 级日志只含 tableId+userId 元数据，无 IP/GPS payload；debug 级保留全部信息便于排障。
6. **缓存自然淘汰**：旧 userId-only key 不再被写入；3h 后旧条目过期；过渡期不阻塞业务。
