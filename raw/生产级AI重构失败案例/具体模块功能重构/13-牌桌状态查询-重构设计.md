## 1. 背景与目标

1025 `CommonProto.TABLE_STATUS` 是 1000 入房 + 1032 入房前校验之外的轻量状态查询协议，由 `TableStatusCommand`（`@ThreadModel(PLAYER)`）委托 `TableStatusCommandDefault` 处理。当前实现存在：

**实际稳定性问题**：

- **P0**：`check()` 中 hall / private / club 校验分支**逻辑错误**——
  ```java
  if (tableInfo.isHallTable()) { Hall校验 }
  if (tableInfo.isPrivateTable()) { Private校验 } else { Club校验 }
  ```
  - HALL 桌：`isHallTable==true` 走 Hall 校验；`isPrivateTable==false` 又走 else 分支的 **俱乐部校验** → 大厅桌**重复**跑两份权限校验
  - RACE 桌：`isHallTable==false`、`isPrivateTable==false` → 错误走**俱乐部校验**
  - CLUB 桌：正确走 else 俱乐部校验
  - PRIVATE 桌：正确走 Private 分支
  影响：大厅桌权限校验做了两遍（性能 + 行为意外）、比赛桌错误地按俱乐部规则校验。
- **P0**：1025 对**解散桌**会抛 NPE。`initContext` 在 `BaseTableCache.checkDisbandTable(tableId)==true` 时直接 `return context`（**不 set tableStatus**），但下游 `after()` L114 `respDto.setStatus(context.getTableStatus().getCode())` 必然 NPE → 解散桌的 1025 查询变成系统异常而不是返回明确"已结束"。
- **P1**：`after()` 中 `BeanUtils.copyProperties(tableInfo, ...)` 对 null tableInfo NPE；`tableInfo.getTableId()` 也是 null NPE。`BaseTable table` 在 L102 拿到后未使用却引发依赖。
- **P2**：1025 链路三处大对象 toJson info：`handleCommand` / `initContext` / `after`。含设备/桌配置 payload。

**结构性问题（不在本期）**：

- `GameTableBootstrapService.init` 在查询路径里副作用初始化牌桌（设计文档 P0-1）—— 跨多个 service，需要与 01 入房 flow 联调，不在本期承诺
- `TableStatusEnum` 与 `GameStatusEnum` 来源不统一（设计文档 P1-2）—— 涉及客户端协议字段语义
- `refreshUser` 同步预热 `UserConfigCache`（设计文档 P1-3）—— 需要接限流框架

目标：消除 P0 逻辑 bug 与 NPE；为 disband 桌返回明确 `GAME_END` 状态而非 SYSTEM_EXCEPTION；加固 `tableInfo` 空值兜底；收敛 1025 链路含敏感 payload 的 info 日志。保持 1025 请求/响应 100% 协议兼容。

## 2. 决策记录

| 决策项 | 结论 |
|--------|------|
| 主导方案 | Frame 主导。改造完全位于 `dx-game-frame`；掼蛋无新增/无修改 |
| `check()` 分支重构方式 | 改为互斥 `if (isHallTable) ... else if (isPrivateTable) ... else ...`。HALL → Hall校验；PRIVATE → Private校验；其余（CLUB/RACE/兜底）→ Club校验。**保持改造前 CLUB 与 RACE 走俱乐部校验的行为**（仅修掉 HALL 重复 + 失语义） |
| `disband` 桌响应策略 | `initContext` 在 disband 路径下显式 `setTableStatus(GameStatusEnum.GAME_END)` 并 set `disbanded=true` 上下文标记；`after()` 对 disband 路径短路：只回 `status` + `isPassword=false`，**不**调用 `BeanUtils.copyProperties`（tableInfo 可能已清理），**不**生成 ASD（无意义） |
| 是否新增 disband 专用响应字段 | **不新增**。复用 `status=GAME_END(2)`；客户端通过 status 判断；不破坏协议兼容 |
| `check()` 是否在 disband 桌跳过 | **跳过**。解散桌权限校验无意义，直接 short-circuit 减少无效 RPC（与改造前比，原本 disband 桌会走 check 然后 after NPE，新行为是 disband 桌直接走 after 返回 GAME_END） |
| `refreshUser` 是否在 disband 桌跳过 | **跳过**。预热目的是为后续入房做准备，disband 桌没有后续入房可能 |
| `after()` tableInfo 空值兜底 | tableInfo 为 null → `tableSimpleCfgDto` 用 `new TableSimpleCfgDto()`（空配置）；不再 NPE。warn 日志能定位 |
| `GameStatusEnum` 与 `TableStatusEnum` 统一 | **不在本期**。涉及客户端协议字段语义对齐，转交 13 §13 待确认项 |
| `GameTableBootstrapService.init` 副作用 | **不动**。功能边界明确要求继续兼容；改造前如何，改造后如何 |
| 灰度开关 | 不做。P0 修复为纯防御，行为只在 disband 路径与 HALL/RACE 校验路径上改善 |

## 3. 整体架构

### 改造前

```
TableStatusCommand (PLAYER 线程, 1025)
  └─ TableStatusCommandDefault.handleCommand
       ├─ PC 设备拦截
       ├─ 排队拦截 (RetryLineUpRedisService)
       ├─ initContext()
       │   ├─ disbandTable=true → return context (tableStatus 不 set, P0 后续 NPE)
       │   ├─ table==null → GameTableBootstrapService.init
       │   └─ context.setTableStatus(GameStatusEnum.getByCode(table.getTableStatus()))
       ├─ check()
       │   ├─ if isHallTable → Hall 校验
       │   ├─ if isPrivateTable → Private 校验
       │   └─ else → Club 校验    ← P0 逻辑 bug: HALL/RACE 桌错误命中
       ├─ refreshUser()
       └─ after()
            ├─ BaseTableCache.getTable / BaseTableInfoCache.getTableInfo (无空保护)
            ├─ BeanUtils.copyProperties(tableInfo, ...)   ← P1 NPE 风险
            ├─ context.getTableStatus().getCode()         ← P0 disband 路径 NPE
            └─ SendUtil.send(TableStatusRespDto)
```

### 改造后

```
TableStatusCommand (PLAYER 线程, 不变)
  └─ TableStatusCommandDefault.handleCommand
       ├─ PC 设备拦截 (不变)
       ├─ 排队拦截 (不变)
       ├─ initContext()
       │   ├─ disbandTable=true → R-02 setTableStatus(GAME_END) + disbanded=true → return
       │   ├─ table==null → GameTableBootstrapService.init (不变)
       │   └─ context.setTableStatus(GameStatusEnum.getByCode(table.getTableStatus()))
       ├─ check()  ── R-02 disbanded 短路 + R-01 互斥分支
       │   ├─ disbanded → return
       │   ├─ if isHallTable     → Hall 校验
       │   ├─ else if isPrivateTable → Private 校验
       │   └─ else               → Club 校验  (CLUB/RACE 都走，保持改造前 CLUB 行为)
       ├─ refreshUser()  ── R-02 disbanded 短路
       └─ after()  ── R-02 disbanded 短路 + R-03 tableInfo 兜底 + R-04 日志收敛
            ├─ disbanded → 只回 status=GAME_END + isPassword=false → send
            ├─ tableInfo == null → tableSimpleCfgDto = new TableSimpleCfgDto() + warn
            └─ 正常路径同改造前
```

**兼容性原则**：

- 协议号 1025 / `TableStatusReqDto` / `TableStatusRespDto` 字段全部不变
- `TableStatusCommand`、`@ThreadModel(PLAYER)` 不变
- `TableStatusContext` 仅新增内部上下文字段 `disbanded`（不进协议）
- 解散桌行为变化：改造前抛 NPE → 改造后正常返回 `status=GAME_END`。客户端原本收到 SYSTEM_EXCEPTION，改造后收到正常响应；按"未知 → 错误"约定的客户端能正确处理 GAME_END
- HALL 桌权限校验从 "Hall+Club 重复跑" → 只跑 Hall。RACE 桌从 Club 校验 → Club 校验（不变，但需在变更记录中标注待产品确认 RACE 是否应走 Hall）

## 4. 安全改造

### R-01：`check()` 改互斥分支（P0 逻辑 bug 修复）

**文件**：`TableStatusCommandDefault.java`

```java
protected void check(TableStatusContext context) {
    if (context.isDisbanded()) {            // R-02 配套
        return;
    }
    Long tableId = context.getTableId();
    Long userId = context.getRequest().getUserId();
    Integer deviceType = context.getRequest().getDeviceType();
    boolean isInRoomUser = BaseUserCheckService.checkUserIsInRoom(tableId, userId);

    BaseTableInfo tableInfo = BaseTableInfoCache.getTableInfo(tableId);
    if (tableInfo == null) {
        log.warn("牌桌状态查询-tableInfo 缺失,跳过权限校验 tableId:{} userId:{}", tableId, userId);
        return;
    }
    if (tableInfo.isHallTable()) {
        BaseUserCheckAuthService.checkUserCanJoinRoomHall(tableId, userId, isInRoomUser, deviceType);
    } else if (tableInfo.isPrivateTable()) {
        BaseUserCheckAuthService.checkUserCanJoinRoomPrivate(tableId, userId, isInRoomUser, deviceType);
    } else {
        BaseUserCheckAuthService.checkUserCanJoinRoom(tableId, userId, isInRoomUser, deviceType);
    }
}
```

### R-02：disband 桌不再 NPE，返回 GAME_END（P0 NPE 修复）

**文件**：`TableStatusContext.java`（新增 `disbanded` 字段）、`TableStatusCommandDefault.java`

`TableStatusContext` 新增内部字段：
```java
private boolean disbanded = false;
```

`initContext()`：
```java
if (BaseTableCache.checkDisbandTable(tableId)) {
    context.setDisbanded(true);
    context.setTableStatus(GameStatusEnum.GAME_END);   // 兜底状态，避免 after() NPE
    log.info("牌桌状态查询-牌桌已解散 tableId:{}", tableId);
    return context;
}
```

`check()` / `refreshUser()` 见 R-01 / R-02 配套：disband 路径短路。

`after()`：
```java
protected void after(TableStatusContext context) {
    Long tableId = context.getTableId();
    Long userId = context.getRequest().getUserId();
    String encryptKey = context.getRequest().getEncryptKey();

    TableStatusRespDto respDto = new TableStatusRespDto();
    respDto.setStatus(context.getTableStatus().getCode());

    if (context.isDisbanded()) {
        respDto.setIsPassword(false);
        respDto.setTableSimpleCfgDto(new TableSimpleCfgDto());
        log.info("牌桌状态查询-返回解散响应 tableId:{} userId:{}", tableId, userId);
        SendUtil.send(NetResponseBuilder.buildResponse(CommonProto.TABLE_STATUS, respDto, context.getRequest()));
        return;
    }

    BaseTableInfo tableInfo = BaseTableInfoCache.getTableInfo(tableId);
    TableSimpleCfgDto tableSimpleCfgDto = new TableSimpleCfgDto();
    if (tableInfo != null) {
        BeanUtils.copyProperties(tableInfo, tableSimpleCfgDto);
        respDto.setIsPassword(BaseUserCheckAuthService.checkUserisNeedPwd(tableInfo.getTableId(), userId));
    } else {
        log.warn("牌桌状态查询-tableInfo 缺失,使用空配置兜底 tableId:{} userId:{}", tableId, userId);
        respDto.setIsPassword(false);
    }
    respDto.setTableSimpleCfgDto(tableSimpleCfgDto);

    if (encryptKey != null && !encryptKey.isEmpty()) {
        respDto.setAsdDto(AsdConvert.convert(userId, tableId, encryptKey));
    }

    log.info("牌桌状态查询-返回信息 tableId:{} userId:{} status:{}",
            tableId, userId, respDto.getStatus());
    log.debug("牌桌状态查询-返回明细 tableId:{} respDto:{}", tableId, GsonUtils.toJson(respDto));
    SendUtil.send(NetResponseBuilder.buildResponse(CommonProto.TABLE_STATUS, respDto, context.getRequest()));
}
```

## 10. 验收关键场景

1. **HALL 桌不再重复跑两份校验**：HALL 桌发 1025，只跑 `checkUserCanJoinRoomHall`，不再额外跑 `checkUserCanJoinRoom`
2. **PRIVATE 桌走 Private 分支**：私人房发 1025，只跑 `checkUserCanJoinRoomPrivate`
3. **CLUB 桌走 Club 分支**：行为不变
4. **RACE 桌走 Club 分支**：行为不变（标注为待确认）
5. **解散桌不抛 NPE**：mock `BaseTableCache.checkDisbandTable` 返回 true，1025 返回 `status=GAME_END(2)`、`isPassword=false`、空 `tableSimpleCfgDto`，不抛系统异常
6. **tableInfo 缺失场景**：mock `BaseTableInfoCache.getTableInfo` 返回 null，1025 返回 `status` + 空 `tableSimpleCfgDto`，warn 日志能定位
7. **日志体积**：1 次正常 1025 产生 ≤ 3 条 info（开始 / 返回 / 完成）；payload 全部降 debug
