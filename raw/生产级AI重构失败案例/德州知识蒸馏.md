# dx-game-texas 知识蒸馏

> 鱿鱼游戏玩法 + 局结算逻辑 + 多手积分聚合 + 保险系统 + 蘑菇堆叠

---

## 一、架构总览

### 1.1 技术栈
- Java / Spring Boot
- netbolt 自研 TCP 游戏框架（协议号路由 + 线程模型注解）
- Dubbo RPC（结算、钱包、玩家信息）
- RocketMQ（阶段信息、大厅状态同步）
- Redis（全部运行时状态缓存）
- AES 加密手牌 + native C 库（`libdz_game_c`）加解密

### 1.2 分层架构
```
Command 层 (proto 协议号路由)
  ├── @ThreadModel(PLAYER) — 按 userId 序列化
  ├── @ThreadModel(TABLE)  — 按 tableId 序列化（所有牌桌状态变更）
  └── @ThreadModel(NETTY)  — Netty IO 线程（仅心跳）

Context 层 (CommandContext<ReqDto>)
  └── 持有 request + 缓存模型对象

Handler 层 (TableSpaceService<T> 策略接口)
  ├── gameModel()     → GameModelEnum（操作类型）
  ├── gameSpaceType() → List<GameSpaceTypeEnums>（空间类型）
  └── handle(T) / handleR(T) → R

Service 层 (跨 handler 复用逻辑)

Event 层 (Spring ApplicationEvent 解耦后置动作)

Cache 层 (TableCache / TableUserCache / TableUserWalletCache)
  └── Redis key: dx::game::texas::table:{tableId}
```

### 1.3 空间类型多态
`DxGameServerFactory.getHandle(spaceType, gameModel)` 运行时路由到对应 handler：
- **CLUB** — 俱乐部桌
- **PRIVATE** — 私人桌
- **HALL** — 大厅桌
- **RACE** — 比赛桌（SNG/MTT）

## 二、核心数据模型

### 2.1 Table（牌桌）
```
extends BaseTable
├── holderId          — 房主 ID
├── reviewCounts      — 总手数（用于牌谱回放）
├── seatList          — 座位号列表
├── insurePot         — 保险池金额
├── handCardNum       — 每人手牌数
├── cryptoCardStr     — 加密牌组字符串（@LogIgnoreField）
├── status            — GameStatusEnum: GAME_WAIT(0) / GAME_STARTING(1) / GAME_END(2)
└── init(handCode)    — 每手开始前：handCounts++, 清除加密状态
```

### 2.2 TableHand（当前手牌数据）
```
extends BaseTableHand
├── betRound          — 当前轮次: 0=无, 1=preflop, 2=flop, 3=turn, 4=river
├── gameHandStatus    — 手牌生命周期: WAIT(1) → START(2) → SETTLEMENT(3) → END(4)
├── publicCards       — 公牌列表（明文整数）
├── publicCardList    — 加密公牌列表
├── mainPot           — 主池 PotItem
├── sidePots          — 边池列表
├── invalidSidePot    — 无效边池（需退还）
├── roundMap          — Map<Integer, TableHandRound>（每轮数据）
├── currentChairId    — 当前操作座位
├── currentUserBetInfo — 当前操作玩家信息
├── bbChairId / button — 大盲/庄家座位
├── sendCardChairIds  — 发牌顺序
├── settlementType    — 0=正常, 1=仅一个有效玩家（其余全弃牌）
├── insureDetails     — 保险购买记录
├── ownShowdownFlag   — 主动亮牌阶段标志
└── init()            — 下一手重置：清空池/牌/轮次/标志
```

### 2.3 TableHandRound（单轮下注数据）
```
├── betRound               — 轮次号
├── maxCallAmount          — 本轮最高下注
├── maxRaiseAmount         — 最高加注额
├── lastRaiseAmount        — 最后一次加注额
├── raiseCount             — 加注次数
├── tableHandRoundDetailList — 所有操作记录
└── getMinimumBet(bb)      — 最小加注 = 无下注时等于 BB; 否则 maxRaise - betAmount + maxCall
```

### 2.4 TableHandRoundDetail（单次操作记录）
```
├── userId / betAction / betAmount / chipScore
├── commitType        — 1=主动, 2=超时, 3=系统托管
├── raiseCount / minRaiseAmount / maxRaiseAmount
├── potTotalAmountBeforeAction — 操作前底池总额
└── handActionSer     — 操作序号
```

### 2.5 UserItem（玩家状态）
```
├── status            — UserStatusEnum: DEFAULT(0)/FOLD(6)/CALL(7)/CHECK(8)/RAISE(9)/ALLIN(10)/BET(11)/OCCUPY(15)/STRADDLE(16)
├── callScore         — 当前轮下注额
├── callTotalScore    — 本手总下注额
├── allinBetRound     — 全下时的轮次
├── userCardInfo      — UserCardInfo（加密手牌）
├── maxCardsType      — 最大牌型
├── startChipScore    — 本手开始时筹码
├── showdownStatus    — 亮牌状态
├── squidJoinType     — 鱿鱼参与类型
├── getChipScore()    — 从 TableUserWalletCache 读取（非本地存储）
└── initData()        — 新手重置：清注额/牌信息/标志位
```

### 2.6 PotItem（底池）
```
├── totalScore        — 底池总额
├── userIds           — 参与者列表
├── hasAllinUser      — 是否有全下玩家
├── sideNo            — 0=主池, 1+=边池编号
├── maxCardUserIdList — 赢家列表
└── potUserInfoList   — 池内玩家信息
```

### 2.7 TableInfo（牌桌规则配置）
```
├── sbBlindScore / bbBlindScore     — 盲注
├── seatsNumberLimit                — 座位数
├── insuranceMode                   — 0=关, 1=经典, 2=EV
├── anteSwitch / anteScore          — 前注
├── straddleSwitch                  — Straddle 开关
├── squidGameSwitch                 — 鱿鱼游戏开关
├── criticalHitPlaySwitch           — 暴击玩法开关
├── stackMushroomSwitch             — 蘑菇堆叠开关
└── 多种延时/保险/特殊规则配置
```

## 三、游戏生命周期

### 3.1 完整流程
```
进入房间 (ENTER_ROOM, proto=100, PLAYER线程)
  ├── tableInitServiceHandle.init()   — 首次初始化牌桌（RPC加载配置→缓存）
  ├── EnterRoomCheckHandler.handle()  — 密码/上限/维护检查
  ├── watchUsers.put(userId, userItem) — 新用户先进观战列表
  └── 返回完整牌桌快照 EnterRoomRespDto（50+字段，支持断线重连）

坐下 (USER_SIT_DOWN, TABLE线程)
  ├── checkCanSitDown()        — 频率/座位冲突/锁定检查
  ├── checkChipScore()         — 筹码充足性（6种不足类型）
  ├── Branch:
  │   ├── 筹码不足 → OCCUPY 状态 → OccupyTimeOutTask 倒计时
  │   └── 筹码充足 → userSitDownCommonDeal()
  │         ├── 个性化通知（每人AES加密不同牌面）
  │         └── seatChangeNotice 广播
  └── 鱿鱼参与类型处理（立即/下轮/默认）

开局 (GAME_START, TABLE线程)
  ├── check: gameHandStatus==WAIT, 筹码>0, 无OCCUPY暂停, 鱿鱼状态检查
  ├── doRoomGameStart():
  │   ├── betRound = PREFLOP
  │   ├── CardService.init()          — 初始化牌组 + AES密钥
  │   ├── selectButtonBbSbStraddle()  — 分配庄/盲/Straddle
  │   ├── buildSendCardChairIds()     — 确定发牌顺序
  │   ├── initTexasUsers()            — 座位编码
  │   ├── mushRoomGameService.addMushRoom() — 蘑菇堆叠
  │   ├── setButtonBlindInfo()        — 记录盲注金额
  │   ├── status = GAME_STARTING / gameHandStatus = GAME_HAND_START
  │   └── TableUserCache.initHandUsers() — 快照手牌玩家
  ├── gameStartNotice.handleNotice()  — 广播开局
  ├── collPoolAnimation()             — 前注动画延迟
  └── gameStartSendCard()             — 发手牌（每人AES加密独立发送）

下注轮 (USER_BET / USER_FOLD_CARD, TABLE线程)
  ├── UserBetHandler.handle():
  │   ├── 验证: 轮到该玩家, 金额合法, 符合最低加注
  │   ├── setUserStatus(): CHECK/CALL/RAISE/BET/ALLIN/FOLD
  │   ├── 扣减筹码: TableUserWalletCache.subChipScore()
  │   ├── 记录操作: tableHandRoundService.addDetail()
  │   └── 发布 UserBetActionEvent
  ├── UserFoldHandler.handle():
  │   ├── 设置 FOLD 状态, 记录弃牌轮次
  │   ├── 待带入筹码立即生效
  │   └── 发布 UserFoldActionEvent

事件驱动轮转:
  UserBetActionListener / UserFoldActionListener
    ├── 取消该玩家倒计时
    ├── tableHandService.handleWhenHandOrRoundOver()
    │   ├── HAND_ROUND_NO_OVER → 通知下一个玩家 (UserBetNoticeTask +1000ms)
    │   ├── HAND_ROUND_OVER   → 发布 HandRoundOverEvent
    │   └── HAND_OVER         → 发布 HandOverEvent

HandRoundOverListener (轮次结束):
  ├── PotCalculatorUtils.handlePot() — 重算底池
  ├── initUserOfRoundOver()          — 重置玩家轮次状态
  ├── collectionChipOfPotNotice      — 收集筹码动画 (maxCallAmount>0 ? 800ms : 0)
  └── SendPublicCardTask (+500ms)    — 发下一街公牌

HandOverListener (整手结束):
  ├── gameHandStatus = GAME_SETTLEMENT
  ├── decryptBodyAndSetCardInfo()    — 解密所有手牌
  ├── PotCalculatorUtils.handlePot() — 最终底池计算
  ├── collectionChipOfPotNotice +500ms — 收集动画
  ├── handleInvalidPot()             — 退还无效边池 (BET_RB)
  ├── Branch:
  │   ├── settlementType==1 → oneValidUserSettlement()
  │   │     填充公牌 + ProvisionalSettlementTask (SHOW_CARD_WIN_ALONE)
  │   └── 正常结算 → normalDelaySettlement()
  │         ├── 需补牌 → 计算outs/胜率 → fillPublicCardTask → 动画
  │         └── 不需补牌 → ProvisionalSettlementTask (SHOW_CARD_FORCE)
  └── 清理: currentChairId/callScore/mushroomNextButton

结算 (ClubSettleInvokeAccountService):
  ├── assembleChanges():
  │   ├── RB退还记录（排最前）
  │   ├── 所有下注操作记录
  │   ├── 赢家结算记录 (winScore > 0)
  │   ├── 保险盈亏记录 (insureNetChip != 0)
  │   └── 蘑菇结算记录
  ├── dxGameAccountRpcServiceInter.settlement() — RPC
  │   ├── SETTLE_SUCCESS → 完成
  │   └── SETTLE_ERROR → 强制解散

下一手 (BaseTableHandNextService):
  ├── 递减 RotateDealerWaitCount
  ├── 清理 handUsers
  └── 发布 NextHandStartTask (带延迟) → 回到开局流程
```

### 3.2 结算类型枚举
```
SHOW_CARD_WIN_ALONE(0) — 其余全弃牌独赢, 动画 0ms
SHOW_CARD_FORCE(1)     — 正常河牌后摊牌, 动画 1000ms
SHOW_CARD_OUTS(2)      — 河牌前全下, 动画 1500ms + 1000ms
```

## 四、底池计算

### 4.1 PotCalculatorUtils.handlePot(tableHand)
- 遍历所有轮次的 `TableHandRoundDetail`
- 计算每个玩家的总投入
- 有 ALLIN 玩家时拆分边池：
  - 按 ALLIN 金额从小到大排序
  - 每个 ALLIN 金额切一刀，产生一个边池
  - 剩余部分进入下一个边池
- 无效边池：仅有 1 人的边池 → `invalidSidePot`，退还给该玩家

### 4.2 底池退还流程 (handleInvalidPot)
```
找到 userIds.size()==1 的边池
  → BET_RB 操作记录
  → TableUserWalletCache.addChipScore(ADD_BET_RB, returnScore)
  → 减少 callTotalScore
  → 移除该边池
  → 减少 potScore
```

## 五、鱿鱼游戏 (Squid Game)

### 5.1 数据模型 TableSquid
```
├── status           — TableSquidStatusEnum:
│                      NO_START_UP(0) → START_UP(1) → WAIT_SETTLEMENT(2) → SETTLEMENT(3/4)
├── squidRounds      — 当前鱿鱼轮数
├── defaultSquidChipScore — 初始鱿鱼筹码（按模式计算）
├── userMap          — Map<Long, UserItemSquid> 参与者
├── giveUpUserMap    — 放弃的玩家
├── headSquid        — 首尾鱼
└── differenceSquidCounts — 差异鱿鱼数
```

### 5.2 初始鱿鱼筹码计算
```
GENERAL模式:     (seats-1) × rewardPrice
INFINITE有上限:  (seats-2+cap) × rewardPrice
INFINITE无上限:  seats × 2 × rewardPrice
CRAZE模式:       复杂公式
```

### 5.3 参与模式
```
SquidTriggerModeEnum:
  INFINITE(0) — 无限鱿鱼（持续进行）
  NORMAL(1)   — 普通鱿鱼（手数触发）

SquidJoinTypeEnum:
  SIT_DOWN(0)       — 坐下即参与
  JOIN_NOW(1)       — 立即加入
  JOIN_ROUNDS_NEXT(2) — 下轮加入
```

### 5.4 结算检查触发源
```
SquidCheckSourceEnum:
  HAND_SETTLEMENT_AFTER(1) — 手牌结算后
  TRANSFER_SUCCESS(2)      — 补码成功
  TRANSFER_GIVE_UP(3)      — 放弃补码
  TRANSFER_TIMEOUT(4)      — 补码超时
  GAMESTART_BEFORE(5)      — 开局前检查
```

### 5.5 结算类型
```
HAND_OVER_SQUID_SETTLEMENT(1) — 手牌+鱿鱼一起结算, 动画1500ms
ONLY_SQUID_SETTLEMENT(2)      — 纯鱿鱼结算, 动画1500ms
```

### 5.6 开关配置（TableInfo 中）
```
squidGameSwitch            — 总开关
rewardModel                — 奖励模式
triggerMode                — 触发模式（普通/无限）
handTriggerNumber          — 手数触发阈值
maximumRound               — 最大轮数
squidRewardPrice           — 鱿鱼奖励价格
squidUpperLimit            — 上限
noActionWinsNoSquidSwitch  — 无动作获胜无鱿鱼
chipLowNoSquidSwitch       — 低筹码无鱿鱼
firstSquidForceFlashSwitch — 首次获得鱿鱼强制亮牌
mostSquidWinsSwitch        — 最多者独赢
multipleRewardSwitch       — 多重奖励
htFishDoubleRewardSwitch   — 首尾鱼双倍奖励
```

### 5.7 玩家鱿鱼钱包操作
```
ActionTypeEnum:
  UP_SCORE(1, "带入")    ← SQUID_BRING_IN
  TRANSFER(2, "划入")    ← SQUID_TRANS_IN
  PAY(3, "支付/赔付")    ← SQUID_PAY

## 六、蘑菇堆叠 (Mushroom Stacking)

### 6.1 数据模型 TableMushroom
```
├── status                    — NO_START_UP → START_UP → WAIT_SETTLEMENT → SETTLEMENT_FINISH
├── mushroomRound             — 当前蘑菇轮数
├── oneMushroomChipScore      — 单个蘑菇价值 = BB × stackMushroomPerHandBbMulti
├── stackMushroomCapValuesMulti — 封顶倍数
├── curMushroomCount          — 当前蘑菇数
├── allMushroomChipScore      — 全部蘑菇总价值
├── userMushroomMap           — Map<Long, UserItemMushroom>
└── init()                    — 结算后重置（mushroomRound++）
```

### 6.2 工作流
```
开局 → mushRoomGameService.addMushRoom() — 庄家位堆叠蘑菇
每手 → 累积蘑菇数 × 单个价值
结算 → appendMushroomSettlementDetails() 追加到主结算流水
```

## 十、线程模型与并发

### 10.1 线程分配
```
@ThreadModel(TABLE)  — 所有牌桌状态变更在 tableId 线程上序列化执行
@ThreadModel(PLAYER) — 进房/退出等在 userId 线程上执行
@ThreadModel(NETTY)  — 仅心跳，不修改状态

eventManager.publish(runnable, label, tableId.toString(), delayMs)
  → 路由到 tableId 对应的事件线程组（保证串行）
```

### 10.2 业务线程池
```
BusinessThreadManager:
  mainExecutorService:          CPU*2 core, CPU*4+1 max, queue=500
  insignificantExecutorService: 1 core, CPU*2+1 max, queue=3000
```

### 10.3 锁机制
```

## 十一、牌面加密

### 11.1 加密流程
```
CardGameLib.up(cardInt, cryptoResource, handCode)
  → TigerLib.up_wei_j(native C) → float[7] 密文

CardGameLibOriginal（无native库）:
  up()         → new float[]{Float.valueOf(card)}
  getRealCard() → (int) floatArray[0]
```

## 十四、灌蛋重写参考要点

### 可复用模式
1. **Command-Context-Handler** 三层分离 + 空间类型多态路由
2. **事件驱动轮转**：下注→检查轮次/手牌结束→发牌/结算，全部通过 Spring Event 解耦
3. **延迟任务调度**：OnceEventTypeEnum 一次性任务 + LoopEventTypeEnum 循环任务
4. **底池拆分算法**：按 ALLIN 金额排序切分，无效边池退还
5. **结算流水构建**：RB退还→操作记录→赢家结算→保险盈亏→蘑菇结算，有序拼接
6. **AES 手牌加密**：每人独立密钥，按人加密发送

### 灌蛋需替换的部分
1. **牌型系统**：德州5张→灌蛋多张手牌 + 不同牌型定义
2. **出牌状态机**：下注轮替换为出牌轮（参考 dx-game-mahjong）
3. **局结算逻辑**：底池制→积分制，多手聚合
4. **鱿鱼/蘑菇**：评估是否需要，可能简化或移除
5. **保险系统**：灌蛋无保险，移除
6. **座位/庄家分配**：灌蛋的对家/队友座位规则不同
