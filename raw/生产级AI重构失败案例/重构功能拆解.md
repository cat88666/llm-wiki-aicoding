# 重构功能拆解

> 本文用于记录掼蛋重构范围内的功能拆解、模块归属与后端角色，方便排期、联调、验收和问题追踪。

## 1. 总览

| 模块 | 功能数量 |
| --- | ---: |
| 游戏准备 | 21 类 |
| 游戏过程 | 10 |
| 结算 | 13 |
| 大厅与支撑能力 | 14 |

## 2. 游戏准备(Junk)

| 序号 | 功能 | 子功能 | 备注 |
| ---: | --- | --- | --- |
| 1 | 进入房间 | 主流程<br>账号锁定校验<br>代理风控锁定校验<br>桌子权限校验<br>设备限制校验<br>房间数据组装<br>重连 / 已在房间处理 | 账号锁定包含用户游戏锁定、俱乐部掼蛋禁入；桌子权限包含俱乐部开关、直属会员、代理自建桌；设备限制包含仅 iOS 与 H5 设备信息缺失 |
| 2 | 入座 | 主流程<br>入座前状态检查<br>座位 / 椅子校验<br>入座风控校验<br>占座<br>占座超时<br>占座转入座<br>自动带入 | 入座风控覆盖 IP、GPS、设备、语音、视频、伙牌；自动带入覆盖普通筹码和彩金 |
| 3 | 带入 | 主流程<br>带入弹窗金额计算<br>普通钱包带入<br>彩金钱包带入<br>钱包类型校验<br>余额校验<br>游戏中补码<br>下一手补码<br>补码后继续状态回写 | 带入弹窗计算最小带入、最大带入、可带入、待补彩金；钱包类型仅允许掼蛋主钱包和 `GD_BONUS` |
| 4 | 站起 | 主流程<br>站起前状态检查<br>过级中途离开限制<br>强制站起<br>结算中站起<br>站起后自动带出<br>过级彩金扣除判断 | 强制站起覆盖房主、中控、风控；结算中站起需要处理继续状态和补码弹窗 |
| 5 | 带出 | 主流程<br>可带出金额查询<br>当前手立即带出<br>游戏中下一手带出<br>自动带出<br>彩金带出<br>带出结果通知 | 可带出金额包含保留筹码、最大可带出、当前 / 下一手带出类型；通知覆盖成功、失败、下一手带出成功 |
| 6 | 退出 | 主流程<br>退出前状态检查<br>站起 / 退出共用校验<br>强制踢出<br>破产 / 现金不足退出<br>退出后清理 | 共用校验覆盖过级、结算、风控、现金不足；退出后清理桌内状态、占座、任务、钱包收尾 |
| 7 | 离开状态检查 | 入座检查<br>站起检查<br>退出检查 | 入座检查只校验不改状态；站起 / 退出检查需要判断过级彩金和是否允许离开 |
| 8 | 风控 | 主流程<br>入座风控<br>实时风控轮询<br>风险提示语<br>风控站起<br>风控与过级彩金关系 | 风险类型覆盖 IP、GPS、设备、语音、视频、伙牌；风控站起需要区分立即站起和本手结束后站起 |
| 9 | 实时账单 | 主流程<br>桌内钱包余额<br>带入带出流水<br>彩金账变记录 | 覆盖普通筹码、彩金余额、手动 / 自动带入带出记录 |
| 10 | 私人房申请审核 | 主流程<br>申请人身份校验<br>牌桌状态校验<br>审核权限校验<br>申请记录生成<br>审核通知推送<br>重复申请防重 | 需要校验申请人是否在房间、是否具备申请条件、牌桌是否处于可审核状态；重复申请要幂等处理 |
| 11 | 私人房审核通过 | 主流程<br>审核结果校验<br>申请状态更新<br>用户入房 / 入座状态衔接<br>权限与风控复查<br>结果通知推送<br>失败回滚 / 补偿 | 审核通过后需要衔接进入房间、入座、带入等准备链路；失败时要保证申请状态和桌内状态一致 |
| 12 | 进入房间前校验 | 主流程<br>是否已在房间<br>是否需要密码<br>房间密码校验<br>前置响应 / 异常返回 | 对应进入牌桌前协议，发生在真正进入房间之前；用于处理带密码房间和已在房间用户 |
| 13 | 牌桌状态查询 | 主流程<br>牌桌是否存在<br>牌桌是否可进入<br>牌桌是否已解散<br>牌桌是否维护中 | 前端进入房间或刷新前需要查询桌子状态，避免进入无效、解散、维护牌桌 |
| 14 | 旁观 / 返回 / 重连 | 旁观者进入<br>用户返回桌子<br>断线重连<br>重连阶段判断<br>准备阶段 / 结算阶段数据恢复 | 区分普通进入、旁观进入、返回桌子、断线重连；重连需要恢复用户状态、补码倒计时和当前阶段数据 |
| 15 | 在线状态恢复 | 主流程<br>当前用户在线状态响应<br>桌内玩家在线状态广播<br>重连后在线状态补偿 | 掼蛋侧有独立在线状态查询，用于修复重连后在线状态未及时同步的问题 |
| 16 | 私人房固定带入审核 | 固定带入判断<br>首次入座判断<br>非房主审核判断<br>占座后自动提交审核单<br>审核通过后带入衔接 | 与普通私人房申请审核不同，这是私人房固定带入首次入座时触发的带入审核链路 |
| 17 | 放弃补码 | 主流程<br>补码状态校验<br>放弃补码记录<br>放弃补码通知<br>继续 / 站起状态衔接 | 对应放弃补码协议；需要明确放弃后用户是否继续、站起或等待结算后处理 |
| 18 | 预站起 / 留座离桌 | 预站起<br>留座离桌<br>预站起取消 / 生效<br>与正式站起 / 退出衔接 | 预站起和留座离桌不同于立即站起 / 退出，需要单独处理状态和后续触发时机 |
| 19 | 风控数据上报 | 风控数据采集<br>风控关系图<br>风控数据校验<br>风控结果与入座 / 站起联动 | 区分服务端实时风控与客户端风控数据上报，关系图用于辅助判断同 IP、GPS、设备、伙牌等风险 |
| 20 | 好友备注 | 待拆解 | |
| 21 | 用户配置查询 / 修改 | 待拆解 | |
