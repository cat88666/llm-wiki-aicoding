# 02-特征测试与 TDD 流程

> 特征测试的目标：先锁住旧系统行为，再允许 AI 重构。TDD 的目标：新行为和修复行为先有裁判，再写实现。

---

## 1. 特征测试优先级

```
先锁生产旧行为
  ├── 协议响应
  ├── 状态副作用
  ├── 缓存副作用
  ├── RPC / MQ 副作用
  ├── 通知副作用
  ├── 异常路径
  └── 并发 / 幂等路径
```

### 普通测试和特征测试

| 维度 | 普通测试 | 特征测试 |
|---|---|---|
| 问题 | 行为对不对 | 改前改后是否一致 |
| 适用 | 新功能、bugfix | 遗留代码重构 |
| 遇到旧 bug | 写正确行为 | 先锁住旧行为，bug 单独修 |
| Junk 用法 | 验收新增服务 | 锁协议、状态、通知、钱包副作用 |

---

## 2. Junk 特征测试分层

| 层级 | 测试对象 | 示例 |
|---|---|---|
| L1 纯规则 | 金额、状态、权限、风控规则 | 最小带入、可带出金额、是否允许站起 |
| L2 服务编排 | 单功能 service / handler | 入座流程、带入流程、审核通过流程 |
| L3 协议回归 | command / proto 入口 | 1000 进房、1006 入座、1042 放弃补码 |
| L4 副作用审核 | 缓存 / RPC / MQ / 通知 | 钱包扣款、账单流水、通知发送 |
| L5 并发幂等 | 重复请求 / 并发请求 | 重复入座、审核通过和退出并发 |

---

## 3. 每个功能最少测试集

| 测试集 | 必须覆盖 |
|---|---|
| 正常路径 | 成功响应、状态变化、通知 |
| 失败路径 | 错误码、错误文案、无副作用 |
| 重复请求 | 不重复写缓存、不重复扣款、不重复发通知 |
| 缓存缺失 | 不 NPE，兜底行为符合旧逻辑或 spec |
| RPC/MQ 失败 | 成功/失败语义明确，有补偿或回滚 |
| 重连/恢复 | 快照字段、用户状态、倒计时、补码状态 |
| 并发 | 同一 table/user 的状态变更不交叉 |

---

## 4. TDD 红绿循环

```
从 spec 选择一个最小行为
  └── 写失败测试 / 回归场景
        └── 让 AI 实现最小代码
              └── 跑测试 / 静态等价检查
                    └── 绿了再提交
                          └── 进入下一个 R 项
```

### R 项落地规则

| R 项类型 | 测试要求 |
|---|---|
| P0 修复 | 必须有失败用例或明确复现步骤 |
| P1 行为变化 | 必须有旧行为说明、新行为验收、灰度观察 |
| 结构重构 | 必须有特征测试证明行为等价 |
| 日志/性能 | 必须说明不影响协议和副作用 |
| 兼容新增字段 | 必须证明旧客户端不受影响 |

---

## 5. 无法跑测试时的替代闭环

生产项目可能临时没有本地编译/测试环境。此时不能假装“已验证”，只能降级为静态等价闭环。

| 替代层 | 做法 |
|---|---|
| 静态 diff | 对比 dev 基线和重构分支的协议、DTO、状态写入 |
| 调用链审核 | 入口、handler、service、RPC/MQ/通知路径逐项核对 |
| 场景表 | 写清输入、前置状态、期望响应、副作用 |
| IDE 冒烟 | 在可用环境补跑协议级场景 |
| 风险标注 | 文档写明“未编译/未自动化测试”，不得写成已通过 |

### 静态等价检查表

| 检查项 | 结论 | 证据 |
|---|---|---|
| 协议号是否变化 | 待填 | 待填 |
| 请求 DTO 是否变化 | 待填 | 待填 |
| 响应 DTO 是否变化 | 待填 | 待填 |
| 通知 DTO 是否变化 | 待填 | 待填 |
| 成功路径副作用是否一致 | 待填 | 待填 |
| 失败路径副作用是否一致 | 待填 | 待填 |
| 错误码是否一致 | 待填 | 待填 |
| 缓存写入是否一致 | 待填 | 待填 |
| RPC/MQ 调用是否一致 | 待填 | 待填 |
| 回滚方式是否明确 | 待填 | 待填 |

## 7. 停止条件

出现以下任一情况，停止当前 R 项，不继续堆代码：

| 停止条件 | 处理 |
|---|---|
| 特征测试不稳定 | 先隔离时间、随机数、缓存、数据库状态 |
| 旧行为无法确认 | 回到扫描和 spec，不让 AI 猜 |
| diff 超出当前 R 项 | 拆 PR 或回滚多余改动 |
| AI 顺手修 bug | 撤回，bug 单独建任务 |
| 协议字段变化 | 停止，走协议兼容评审 |
| 资金副作用变化 | 停止，人工确认 |
