# 测试治理 SPEC（governance · 示例）

- **状态**：governance（持续生效，不随 planned/implemented 迁移）
- **更新于**：<YYYY-MM-DD>

## 规则

- 改老代码前先有**特征测试**锁住现状；没有安全网不许重构这块。
- 特征测试锁的是"现在是什么"，不是"应该是什么"；现状里的可疑行为原样锁住，要不要改单独走 issue + PR。
- "行为不变"不只是返回值，也包括：错误语义、事件、日志、public import 路径、警告来源。
- 新代码走 TDD（先写失败测试再实现）；老代码走特征测试（先补测试再改）。
- CI 门禁：类型检查 + 测试，覆盖源码 / 测试 / 示例；设为 required check，红就不许合并。
- 一个 PR 只做一件事：重构（行为不变）和改 bug（行为变）分开提。

## 验收

- [ ] 受影响范围都有测试覆盖
- [ ] 重构 PR：既有测试全绿（行为不变）
- [ ] 改行为 PR：对应测试已更新，并在描述里说明改了什么
