# 安全重构执行卡

> 用在：特征测试已经全绿之后。目标是只改变结构，不改变行为。

## 输入

- 当前行为 spec。
- 已全绿的特征测试。
- 待重构入口或模块。
- 架构梳理中的 public API 和状态地图。

## AI 动作

1. 保留原入口和 public API。
2. 一次只动一个边界：一个函数、一个职责块、一个文件。
3. 把内联逻辑抽成命名清晰的小函数或小模块。
4. 每一步跑特征测试。
5. 如果测试红，把失败信息结构化分析，再修正。

## 人审动作

- 检查 diff 是否足够小。
- 检查入口、返回结构、事件、日志、import 路径是否保持不变。
- 检查 AI 有没有顺手改业务逻辑。
- 测试红时先判断：是测试写错、状态没隔离，还是行为真的变了。

## 完成标准

- 重构前后特征测试全绿。
- public API 没变。
- 行为没有变化。
- 每个 commit / PR 只做一个明确边界的重构。

## 禁止事项

- 禁止重构和修 bug 混在一起。
- 禁止一次移动大片文件。
- 禁止入口还没稳定就改调用方。
- 禁止测试红时让 AI 继续扩大修改范围。

## 可复制 Prompt

```text
现在特征测试已经全绿。请做一次小步安全重构。

约束：
1. 保留现有 public API / 入口 / 返回结构，不改变外部行为；
2. 一次只重构一个职责块，diff 控制在可 review 的范围内；
3. 每完成一步，运行全部特征测试；
4. 如果测试失败，先解释失败说明了哪条行为变化，再给最小修复；
5. 不要顺手修 bug，不要新增功能。

请先给本步计划，等我确认后再改代码。
```
