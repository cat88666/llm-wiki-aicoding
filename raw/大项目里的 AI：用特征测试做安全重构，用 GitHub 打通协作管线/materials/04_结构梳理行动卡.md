# 结构梳理行动卡

> 什么时候用：代码已经拆成多个文件，但你怀疑它们靠全局变量、import 副作用和共享状态暗中耦合——改一个文件，另一个莫名其妙坏了。

## 输入

一个多文件的模块或目录，代码量不大但文件之间关系不透明。

## AI 动作

让 AI 只读不改，输出三样东西：

1. **依赖图**：谁 import 了谁。特别标注近循环依赖（A import B，B 又回读 A）。
2. **状态地图**：哪些是全局/持久化状态（sqlite、模块级变量、配置文件、全局字典），哪些步骤在读写它们。
3. **隐藏契约清单**：public import 路径、下游绕过 public API 直接摸内部模块的地方、事件/日志/审计记录的格式。

## 人审动作

- 依赖图有没有漏掉跨文件的隐式依赖？
- 状态地图有没有把全局状态标全？
- 哪些 import 路径是下游在用的、重构时不能随便改？

## 完成标准

画出来的图能解释：改 A 时，B 为什么会跟着出问题。

## 可以直接套用的提示词

```text
请只读这个目录下的代码，不要修改任何文件。

先画两张图：
1. 依赖图：每个文件 import 了哪些其他文件，特别标出近循环依赖。
2. 状态地图：哪些是全局/持久化状态（数据库、模块级变量、配置文件、全局字典），哪些步骤在读写它们。

再列一份清单：
- public API 和 public import 路径。
- 下游可能绕过 public API 直接调用的内部函数。
- 事件、日志、审计记录的格式和触发位置。

最后根据以上信息，列出：如果重构时只改其中一个文件而不动其他文件，哪些地方最容易出问题。

不要给重构方案，先画图。
```
