# 掼蛋业务架构

> 本文用于梳理掼蛋项目的业务对象、核心流程、规则计算、结算链路、协议通知、任务调度、重连回放与非业务能力边界，作为后续代码阅读、重构拆分、联调排障和需求评审的业务地图。

---

## 2. 数据结构

### 2.1 核心对象关系

```text
Table
└── TableHand
    ├── TableHandUser
    │   ├── 手牌列表
    │   ├── 推荐牌型
    │   ├── 推荐方案
    │   ├── 选中方案
    │   ├── 手牌状态
    │   ├── 理牌总时间
    │   ├── 延迟时间
    │   └── 增值服务收费金额
    ├── UserItem
    │   ├── 入座玩家列表
    │   ├── 占座玩家列表
    │   ├── 旁观者列表
    │   ├── 退出玩家列表
    │   └── 镜像玩家列表
    ├── TableUserWallet
    └── SettleInfo
```

| 对象 | 职责 | 关键内容 |
| --- | --- | --- |
| `Table` | 牌桌级上下文 | 房间配置、桌子生命周期、桌内玩家集合、当前手引用 |
| `TableHand` | 一手牌局过程 | 发牌、级牌、转蛋、贡还贡、出牌轮次、排名、结算临时状态 |
| `TableHandUser` | 单个用户在当前手的手牌状态 | 手牌列表、推荐牌型、理牌方案、托管/理牌状态、延迟与收费 |
| `UserItem` | 玩家在桌内的身份与列表归属 | 入座、占座、旁观、退出、镜像用户 |
| `TableUserWallet` | 玩家桌内钱包 | 带入、带出、余额、手动带入带出累计 |
| `SettleInfo` | 当前手结算上下文 | 用户输赢、比牌结果、校验结果、结算快照 |

### 2.2 玩家手牌 TableHandUser

| 字段域 | 说明 |
| --- | --- |
| 手牌列表 | 当前用户真实持牌，是出牌校验、托管、报牌、回放的基础数据 |
| 用户当前手牌列表对应的推荐牌型 | 算法根据手牌生成的推荐牌型结果 |
| 推荐方案 | 可展示给用户的推荐理牌方案 |
| 选中方案 | 用户确认理牌后的最终方案 |
| 手牌状态 | 标识待理牌、已理牌、托管、已出完等状态 |
| 理牌总时间 | 用户理牌耗时，用于统计、收费或体验分析 |
| 延迟时间 | 延长操作、超时等待或补偿处理相关时间 |
| 增值服务收费金额 | 推荐、延时或其他增值服务产生的收费金额 |

### 2.3 玩家 UserItem

| 玩家列表 | 说明 |
| --- | --- |
| 入座玩家列表 | 当前实际参与游戏的玩家 |
| 占座玩家列表 | 已占座但可能未进入当前手流程的玩家 |
| 旁观者列表 | 不参与当前手，仅接收观战或桌面信息 |
| 退出玩家列表 | 离桌后仍需要参与结算、回放、账务收尾的玩家 |
| 镜像玩家列表 | 用于同步、测试、风控或特殊展示场景的镜像用户 |

### 2.4 玩家钱包 TableUserWallet

| 字段 | 说明 |
| --- | --- |
| 钱包类型 | 区分现金、彩金、体验币或其他账户类型 |
| 余额 | 当前桌内可用余额 |
| 上次带出金额 | 最近一次带出金额 |
| 总带入金额 | 当前桌生命周期内累计带入 |
| 总带出金额 | 当前桌生命周期内累计带出 |
| 玩家手动带入 | 用户主动带入金额 |
| 玩家手动带出 | 用户主动带出金额 |

### 2.5 结算信息 SettleInfo

```text
SettleInfo
├── SettleUserResult
│   └── CardsGroup
├── SettleUserCheckResult
└── SettleBetRoundCheckResult
    └── Card
```

| 对象 | 说明 |
| --- | --- |
| `SettleUserResult` | 每个用户的输赢情况，包含名次、输赢金额、升级结果、费用等 |
| `CardsGroup` | 每次比牌的牌型数据，可用于结算展示、回放和审计 |
| `SettleUserCheckResult` | 针对每个人输赢情况的校验结果 |
| `SettleBetRoundCheckResult` | 单道比牌结果，记录本轮牌面比较详情 |
| `Card` | 单张牌数据，是牌型识别、比较、回放展示的最小单位 |

### 2.6 算法与推荐牌型

```text
算法（推荐牌型计算）
├── 发手牌设置推荐牌型列表 IntelligentResultResp
├── 游戏开始初始化牌
├── 游戏开始初始化推荐牌型 IntelligentOneResp
├── 确认理牌设置用户选择的牌型
└── 开始理牌通知组装玩家理牌卡片组数据
    └── GroupCardResult
        └── CardsGroup
```

| 阶段 | 输入 | 输出 | 说明 |
| --- | --- | --- | --- |
| 发手牌 | 玩家手牌、级牌、玩法配置 | `IntelligentResultResp` | 批量生成推荐牌型列表 |
| 游戏开始初始化牌 | 桌子配置、玩家列表、发牌结果 | 初始牌列表 | 形成当前手的玩家持牌 |
| 游戏开始初始化推荐牌型 | 单个玩家手牌 | `IntelligentOneResp` | 初始化单个玩家的推荐牌型 |
| 确认理牌 | 用户选择的方案 | 选中牌型 | 固化用户本手选择 |
| 开始理牌通知 | 推荐结果、手牌列表 | `GroupCardResult` / `CardsGroup` | 组装前端展示用的卡片组数据 |

### 2.7 牌局回放、结算与注单

| 模块 | 记录内容 | 用途 |
| --- | --- | --- |
| 牌局回放 | 开局、发牌、出牌、过牌、贡还贡、转蛋、结算 | 断线恢复、历史回看、客服排障、风控审计 |
| 结算 | 输赢、加钱、减钱、费用、升级、排名 | 钱包落地、账单展示、下一手状态初始化 |
| 注单 | 牌局过程、用户行为、结算结果 | 对账、查询、稽核、异常补偿 |

## 4. 规则计算层

### 4.1 分层目标

规则计算层用于把业务动作与牌型规则隔离，避免出牌 Action、比较工具类和结算逻辑互相穿透。

```text
Action 输入
└── RuleCalc
    ├── 牌型识别
    ├── 合法性校验
    └── 牌型比较
        └── RoundCalc
            ├── 本轮结果快照
            ├── 可执行动作集
            ├── 最终动作裁决
            └── 结算所需快照
```

### 4.2 对齐 Texas 计算层

| Texas 思路 | 掼蛋对应职责 |
| --- | --- |
| `CardCalcService` | 作为规则计算入口，统一承接出牌判定、牌型识别、合法性校验和比较裁决 |
| `CardCalcSettleService` | 按阶段生成可用于结算的牌面结果快照，避免结算重新推导牌局过程 |

### 4.3 掼蛋落地范围

| 当前散落点 | 收敛方向 |
| --- | --- |
| `CompareChecker` | 收敛到 Compare 层，负责同牌型、炸弹优先级、特殊牌规则比较 |
| `SortCardUtils` | 收敛到 RuleCalc 的牌型识别和排序辅助能力 |
| `BaseCardOutAction` | 只保留动作编排，把规则判断下沉到 RuleCalc |

### 4.4 Compare 层

| 比较类型 | 说明 |
| --- | --- |
| 同牌型比较 | 单牌、对子、三张、顺子、连对、钢板、三带二等普通牌型比较 |
| 炸弹优先级比较 | 天王炸、超级炸弹、同花顺、5 炸、4 炸、普通牌型之间的优先级裁决 |
| 特殊牌规则比较 | 级牌、逢人配、大小王、玩法开关下的特殊比较规则 |

### 4.5 策略层

| 策略 | 说明 |
| --- | --- |
| 炸弹倍数策略 | 如 `BombDouble`、`CrazyDouble`，负责炸弹对倍率或结算的影响 |
| 玩法开关策略 | 根据房间配置控制转蛋、贡还贡、抗贡、三把不过回 2、必打等规则 |

### 4.6 输出层

| 输出 | 说明 |
| --- | --- |
| 可执行动作集 | 当前玩家可出、可过、可贡、可还、可继续等动作 |
| 最终动作裁决 | 本次输入动作是否成立，以及成立后的状态变更 |
| 结算所需快照 | 为结算保存本轮或本手的牌面结果，减少结算阶段二次推导 |

## 5. 三段式结算层

### 5.1 分层结构

```text
三段式结算
├── ProvisionalSettlement
│   └── 预结算
├── Settlement
│   └── 正式结算
└── SettlementAfter
    └── 后处理
```

| 阶段 | 责任边界 | 是否落钱包 |
| --- | --- | --- |
| `ProvisionalSettlement` | 只算名次、升级、贡还贡、费用分摊、风控标识、结算展示数据 | 否 |
| `Settlement` | 账务、状态、重试、风控落地、注单落地 | 是 |
| `SettlementAfter` | 通知、缓存刷新、下一手触发、任务清理、回放收尾 | 否 |

### 5.2 掼蛋现状映射

| 现状模块 | 当前职责 |
| --- | --- |
| `SettlementService` | 主结算入口，承接当前手结束后的结算编排 |
| `SettleStatisticService` | 统计计算，生成输赢、费用、升级、倍率等结果 |

### 5.3 结算核心关注点

| 关注点 | 说明 |
| --- | --- |
| 输赢计算 | 根据名次、底分、倍率、玩法开关生成用户输赢 |
| 加减钱 | 钱包余额变更、带入带出影响、服务费扣除 |
| 状态收尾 | 标记本手结束、清理临时状态、推进下一手 |
| 重试与补偿 | 钱包、注单、通知、缓存刷新失败时需要可重试 |
| 风控与审计 | 保留结算快照、用户行为、资金变更和规则版本 |

## 9. 重连与回放

### 9.1 重连恢复

```text
重连恢复
├── 转蛋阶段恢复
│   ├── 随机牌
│   └── 换座
├── 贡牌阶段恢复
│   ├── 上贡进度
│   └── 还贡进度
├── 结算倒计时恢复

```

| 恢复点 | 说明 |
| --- | --- |
| 转蛋阶段恢复 | 恢复随机牌、持牌人、换座、首出等阶段数据 |

| 托管状态恢复 | 断线后按托管处理，重连后取消或刷新托管状态 |

### 9.2 回放数据

| 回放类型 | 说明 |

## 11. 业务链路速查

| 场景 | 主路径 | 关键数据 |
| --- | --- | --- |
| 开始一手 | `3002` -> 开局校验 -> 发牌 -> 推荐牌型 -> 转蛋 / 贡牌 -> 首出通知 | `Table`、`TableHand`、`TableHandUser`、`IntelligentOneResp` |
