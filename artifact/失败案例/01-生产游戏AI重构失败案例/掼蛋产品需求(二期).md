## 一、需求背景与目标

### 1.1 背景

掼蛋游戏一期上线后，存在以下不足：

1. 缺少转蛋玩法入口（原仅有过级玩法）
2. 炸弹倍数无法自定义（用户反馈强烈）
3. 用户体验问题若干（见六、APP端体验优化）
4. 后台管理功能缺失（代理后台、商户后台对掼蛋牌桌的数据查看能力不完整）

### 1.2 目标

- 新增转蛋玩法，并在 APP 代理中心、中控后台、代理后台、商户后台全面补充支持
- 新增炸弹加倍/翻倍玩法，允许任意炸弹自定义倍数
- 新增三把不过回2玩法
- 新增弹幕/短语/记牌器/机器人等增值功能
- 优化 APP 端多处体验细节

### 1.3 版本范围

| 场别 | 涉及端别 |
|------|---------|
| 私人房、俱乐部 | APP（游戏端）、中控后台、代理后台、商户后台、代理中心（APP内）、商户SDK |

## 三、后台管理需求

### 3.1 中控后台

#### 3.1.1 俱乐部开桌规则配置

**入口路径**：游戏管理 > 俱乐部开桌规则

中控后台为不同玩法类型（四人掼蛋-转蛋玩法、四人掼蛋-过级玩法、二人掼蛋）分别配置开桌规则，对应字段详见第四章。

二期新增/变更项目如下：

**a. 转蛋玩法新增（Tab 页组织结构）**

配置页分为以下 Tab：
- 基础配置
- 玩法配置
- 收费配置
- **增值配置**（新增 Tab，放在收费配置之后）
- 防作弊

**b. 增值配置 Tab（新增）**

| 字段 | 说明 |
|------|------|
| 操作行动延时开关 | 功能开关，默认开启 |
| 单次行动延时次数 | 多选项，初始默认值：1、2、3；前端默认选 1；数据类型正整数，最多 30 个选项 |
| 延时时间（秒） | 多选项，初始默认值：15、20、30；前端默认选 15 |
| 延时收费方式 | 固定收费 / 底分倍数收费 / 单次行动指数递增，三选一 |
| 首次免费 | 是否首次免费 |
| 收费起始（USDT） | 仅在非首次免费时有效，示例值 100 USDT |
| 增长倍数（倍） | 指数递增时使用 |
| 延时收费上限（倍底分） | 示例值 1000 |

> **重要**：原收费配置 Tab 中的"操作行动延时"字段迁移到增值配置 Tab，收费配置 Tab 不再包含此字段。

**d. 防作弊配置新增**

新增【无障碍服务限制】配置项：

| 配置值 | 行为描述 |
|--------|---------|
| 开启 | 安卓端检测无障碍权限；若玩家已开启无障碍相关权限（屏幕读取/辅助操作类软件），则拦截进入牌桌/入座 |
| 关闭 | 安卓端不检测，即使开启无障碍服务也不拦截 |

- iOS 端不执行此检测，不受此配置影响
- **此开关默认为开启状态**
- 检测场景（牌桌内）：断线重连时、发手牌时、发翻牌时、发转牌时、发河牌时
- 检测到违规时弹出提示框，内容为："检测到您的设备可能存在以下违规风险，暂无法使用本软件：1. 投屏/录屏  2. 其它环境风险，请排除风险后重新连接"，并展示 10 秒倒计时退出游戏按钮

### 4.4 字段说明

#### 炸弹倍数字段详解

**三种模式（互斥单选）**：

| 模式 | 含义 | 前端默认 |
|------|------|---------|
| 无倍数 | 炸弹不产生倍数 | 前端默认 |
| 炸弹加倍 | 炸弹按固定值叠加倍数（加法） | — |
| 炸弹翻倍 | 炸弹按固定值翻倍（乘法） | — |

**后台配置方式**：多选（允许同时提供多个选项供前端房主选择）；**前端创建房间时单选**。

**选炸弹加倍时显示的子字段**（也是多选，前端单选）：

| 选项 | 默认 |
|------|------|

### 5.2 三把不过回2玩法

**涉及端**：APP（游戏逻辑 + 界面展示）、中控/私人房开桌规则配置

**仅四人掼蛋过级玩法支持，二人掼蛋不支持。**

#### 5.2.1 规则描述

1. 一轮牌局中，每个队伍在打必打级牌时，共有 **3次机会**
2. **记录条件**：正在过级牌的队伍打必打级牌时，才记录次数
3. **通过条件**：正在过级的队伍打必打级牌时，**必须**：
   - 胜利方一位玩家为头游
   - 且另一位玩家**不能**为末游
   - 即：以头游+末游方式胜利时，**级牌没有打过**，需继续打该级牌
4. 累计 3 次没通过：该队伍级牌**回到初始值 2**
5. **另一个队伍**如果也在打必打级牌，**即使头游二游胜利也不能过级**
6. 只有任意队伍通过必打级牌，一轮牌局才结束

#### 5.2.2 后端逻辑

- 后端需记录一轮游戏中两个队伍打目标级牌时的次数
- 累计 3 次未通过时，该队伍级牌重置为 2
- 队伍过级次数在界面上展示，所有玩家可见

#### 5.2.3 界面展示

**级牌展示框变化**：
- 队伍首次开始打目标级时，级牌展示框下方右上角新增数字，代表过目标级次数（所有人可见）
- 首局显示 1，第二次显示 2，第三次显示 3

- 对方视角：`对方第x手打A，三把不过回2`
- x = 当前是第几次打目标级牌
- A = 目标级牌的点数

### 5.3 炸弹加倍/翻倍玩法

**涉及端**：APP（游戏逻辑 + 界面展示）、后台开桌规则配置

#### 5.3.1 炸弹加倍规则

**倍数计算方式（加法）**：
- 炸弹基础倍数为 1
- 每出一个炸弹，炸弹倍数在基础上**增加**对应值
- 最终倍数 = 1 + 各炸弹加倍数之和

**各炸弹加倍数**：

| 打出牌型 | 加倍数 |
|---------|-------|
| 四炸 | +1 |
| 五炸 | +1 |
| 同花顺 | +2 |
| 六炸 | +3 |
| 七炸 | +3 |
| 八炸 | +3 |
| 九炸 | +3 |
| 十炸 | +3 |
| 天王炸 | +4 |

**示例**：头游产生前，炸出了两个四炸（+1+1）、一个同花顺（+2）、一个天王（+4），则倍数 = 1 + (1+1+2+4) = 9

**选项规则**：
- 【四炸起加倍】：从四炸开始加倍
- 【五炸起加倍】：从五炸开始加倍（同花顺算五炸），四炸不加倍
- 【六炸起加倍】：从六炸开始加倍，六炸以下不加倍

**头游产生后**：
- 头游方（头游+其队友）打出的炸弹**继续**加倍
- 对手打出的炸弹**不再**加倍，对手视角总倍数数字前增加【锁定】图标

**封顶规则**：结算时，最终倍数（炸弹总倍数 × 排名倍数）不超过配置的封顶倍数。

#### 5.3.2 炸弹翻倍规则

**倍数计算方式（乘法）**：
- 每出一个炸弹，炸弹倍数**乘以**对应翻倍数

**各炸弹翻倍数**：

| 打出牌型 | 翻倍数 |
|---------|-------|
| 四炸 | ×2 |
| 五炸 | ×2 |
| 同花顺 | ×2 |
| 六炸 | ×2（待确认） |
| 七炸 | ×2（待确认） |
| 八炸 | ×（待定） |
| 九炸 | ×（待定） |
| 十炸 | ×（待定） |
| 天王炸 | ×（待定） |

**示例**：炸出两个四炸（×2×2）、一个同花顺（×2）、一个天王（×2），则炸弹倍数 = 2×2×2×2 = 16

**排名倍数（头游二游对应的翻倍数）**：

| 胜方游次组合 | 排名翻倍数 |
|------------|----------|
| 头游+二游 | ×3 |
| 头游+三游 | ×（待定） |
| 头游+末游 | ×1 |

**封顶规则**：结算时，炸弹倍数 × 排名倍数不超过配置的封顶倍数。

#### 5.3.3 结算计算规则

**炸弹加倍结算**：
- 正常情况（筹码足够且没有封顶）：输赢分 = 底分 × (1 + 炸弹倍数) × 排名倍数
- 筹码足够已封顶：输赢分 = 底分 × 封顶倍数
- 筹码不足：输赢分 = min(剩余筹码, min(原本输赢分, 封顶分数))

**炸弹翻倍结算**：
- 正常情况（筹码足够且没有封顶）：输赢分 = 底分 × 炸弹倍数 × 排名倍数
- 筹码足够已封顶：输赢分 = 底分 × 封顶倍数
- 筹码不足：输赢分 = min(剩余筹码, min(原本输赢分, 封顶分数))

**防以小博大策略**：
- 本局带入多少筹码，本局最多输赢多少筹码
- 结算时，如果玩家输赢超过牌桌筹码则将输赢分改为筹码值
- 输赢金额平衡处理：分别算出输分玩家的总输赢金额和赢分玩家的总输赢金额，比较两者绝对值大小：
  - 取绝对值较小(X)一方：玩家输赢直接按修改后分数结算
  - 取绝对值较大(Y)一方：该玩家输赢 = 该玩家修改后输赢 / Y × X（保留2位小数，无条件舍去，小数尾差放到最大输家或赢家）

**筹码不足时队伍分配规则**：
1. 计算队伍输赢分 = 同队两个玩家结算3之和
2. 两队输赢分相同：玩家实际输赢分 = 结算3
3. 两队输赢分不同：取更小的一队输赢分作为最终队伍输赢分
   - 另一队中筹码更少玩家的输赢分 = min(自己剩余筹码, 最终队伍输赢分 / 2)
   - 另一玩家输赢分 = 最终队伍输赢分 - 同队筹码更少玩家的输赢分

**破产补码规则**：
- 破产定义：玩家剩余筹码为 0 时为破产，才弹出补码弹窗
- 补码最小值 = 设置的最小带入 - 身上剩余筹码

#### 5.3.4 界面展示

**打出炸弹时**（头游产生前）：
- 在桌面打出的牌下方显示该炸弹的倍数（加倍：`+N倍`；翻倍：`×N倍`）
- 动画展示 0.8~1 秒后，飞向倍数显示位置，飞行过程中逐渐变小，到达后消失
- 同时倍数显示框更新为新数值

**倍数显示框**：

**操作流程**：
1. 点击"添加机器人"按钮 → 弹出添加数量弹窗
2. 输入数量（默认 0，只能输入整数），输入方式与带入补码时一致
3. 点击确认 → 机器人随机入座到空位
4. 若输入数量超过空位数 → 多余机器人成为旁观者，旁观者在一手游戏结束后自动离开
5. 机器人离开后 **10 秒内**无法再将该机器人添加进该牌桌

**机器人账号条件**：
- 账户余额 ≥ 牌桌最小带入
- 若无满足条件的机器人，弱提示 `机器人不足`（持续 2 秒）

**开桌时开启【添加机器人】后，压测模式的机器人不会自动进入该牌桌。**

#### 5.7.2 机器人带入配置

- 机器人带入筹码 = 该房间设置的最小带入值
- 携带彩金 = 该房间设置的金额

#### 5.7.3 机器人游戏规则

- 出牌必须严格遵守牌局和牌型规则
- 一手游戏结束后，自动点击继续进入下一手
- **破产自动补码**：补码金额 = 该房间最小带入；彩金不足时同样自动补充
- 破产后账户余额不足以补码时，自动离开牌桌
- **陪玩桌台**：一轮游戏结束后自动点继续，进入坐下状态

#### 5.7.4 机器人理牌规则（无逢人配）

优先级（从高到低）：天王炸/大小王 > 同花顺 > 顺子 > 4~8炸 > 连对 > 钢板/3带2

详细规则：
- **天王炸/大小王**：最优先理出
- **同花顺**：花色优先级 黑桃>红桃>梅花>方块；按 2→3→...→K→A 顺序扫描；优先找**单牌数最多**的段；点数相同则取最小同花顺；最后检查 A-5 最小同花顺
- **顺子**：同花顺理完后处理；从连续点数 ≥5 张的段中，优先找单牌数最多的段；点数相同则取最小顺子；最后检查 A-5 最小顺子
- **4~8炸**：顺子理完后，剩余散牌中的炸弹与天王炸/同花顺一起按炸弹从大到小排在最左侧
- **连对**：3对连续点数直接组连对；4对连续点数按最小连对组合，保留最大对子
- **钢板/3带2**：剩余散牌中的3张和对子组合；3张和对子数量相等则全部组3带2；3张比对子少则尽量组3带2；对子比3张少则检查是否有钢板机会

#### 5.7.5 机器人理牌规则（有逢人配）

优先级不变，逢人配使用原则：
- 优先用1张逢人配补全同花顺（长度 ≥5）
- 同花顺用完逢人配后，再考虑顺子使用逢人配
- 炸弹：优先从3张+1张逢人配组炸弹（优先最小非钢板的3张）；若无3张则从最大非同花顺炸弹+逢人配组合
- 逢人配**不参与**3带2和钢板的组合

#### 5.7.6 机器人出牌逻辑

**首出优先级**（从低到高打，小的先出）：
单牌（除大小王和级牌）> 对子 > 3张（除级牌3张）> 顺子 > 连对 > 3带2 > 钢板 > 级牌和大小王 > 炸弹（从小到大）

**压牌规则**（上家为队友时默认不出）：

花色优先级：方块 > 梅花 > 红桃（红桃级牌不含）> 黑桃

| 上家出牌 | 压牌策略 |
|---------|---------|
| 单张 | 有大过上家的单张→从小到大压；无→炸弹（从小到大）；无炸→拆对子/3张；再无→拆组合牌型（钢板>3带2>连对>顺子）；均无→不出 |
| 对子 | 有大过上家对子→从小到大压；无→炸弹；无炸→拆3张；无→拆组合（3带2>连对>钢板>顺子）；均无→不出 |
| 3张 | 有大过→压；无→炸弹；无炸→拆组合（钢板>3带2>连对>顺子）；均无→不出 |
| 顺子/3带2/连对/钢板 | 有大过→压；无→炸弹；无炸→看散牌能否组成大过的牌型→均无→不出 |
| 炸弹 | 有大过的炸弹→压（从小到大）；无→不出 |

#### 5.7.7 机器人进/还贡逻辑

**进贡**：
- 最大点数手牌只有1张 → 直接取最大手牌
- 最大点数有多张 → 按花色顺序：方块>梅花>红桃（红桃级牌除外）>黑桃

**还贡**（接贡后先重新理牌）：

还贡点数10及以下时优先级：
1. 散牌区符合还贡规则的最小单张
2. 拆散牌区最小对子（按花色顺序取）
3. 拆散牌区最小3张（按花色顺序取）
4. 拆钢板中最小3张
5. 拆3带2中最小3张
6. 拆3带2中最小对子
7. 拆连对中最小对子
8. 拆顺子中点数最小的手牌
9. 拆点数最小炸弹
10. 拆同花顺中点数最小的手牌
- 逢人配手牌除外

还贡任意牌时：不考虑点数限制，按上述逻辑执行，逢人配除外。

#### 5.7.8 机器人账号配置

| 配置项 | 值 |
|--------|---|
| 数量（陪玩） | 50个 |
| 数量（压测） | 更多，按需 |
| 头像 | 注册默认头像 |
| 昵称 | 注册默认昵称 |
| 初始账户余额 | 100000 |
| 绑定关系 | 所有机器人必须与开发服/测试服代理账号关联，加入代理俱乐部 |

#### 5.7.9 压测模式

- 执行「开始测试」：机器人自动填满牌桌空位并开始游戏
- 执行「终止测试」：强制解散俱乐部所有牌桌并踢出所有玩家
- **机器人不会进入**已开启【添加机器人】功能的牌桌（陪玩桌）
- 按牌桌创建时间顺序，机器人从最先创建的牌桌开始自动进入
- 机器人按座位 id（1/2/3/4）顺序入座
- 机器人从一个牌桌离开后，该牌桌短时间内不会再抓取该机器人
- **压测桌台**：一轮游戏结束后有 **50%** 概率站起并离开（每个机器人单独走概率）；不离开则自动点继续

## 六、APP端体验优化

### 6.1 结算界面优化（增加局服务费展示）

**涉及端**：APP

在手牌结算界面，当开桌规则配置了"每局收取服务费"时：
- 结算界面增加**局服务费**展示
- 展示收取金额及收费方式说明

### 6.2 自动补码可手动输入

**涉及端**：APP

当前自动补码仅能选择预设金额，二期优化为：
- 补码弹窗支持**手动输入金额**
- 输入规则与带入筹码时的输入方式一致
- 补码最小值 = 设置的最小带入 - 身上剩余筹码（已破产玩家：剩余筹码为 0，则补码最小值 = 最小带入）

### 6.3 牌局设置-增加风格设置

**涉及端**：APP

牌局设置中新增「风格设置」选项，允许玩家自定义游戏界面风格（如扑克牌背面、桌布颜色等，具体由美术设计提供选项）。

### 6.4 展示上次打出的牌

**涉及端**：APP

**优化前**：点击桌面空白处可查看上次打出的牌。

**优化后**：
- 明确「上次打出的牌」的展示逻辑，避免与其他交互冲突
- 点击桌面空白处时，不与出牌确认/取消等核心操作冲突
- 显示当前手牌中上家最后一次打出的完整牌组

### 6.5 手牌与打出牌遮挡优化

**涉及端**：APP

手牌区域与打出牌区域在某些屏幕尺寸下存在遮挡，优化方案：
- 调整打出牌的显示位置，确保不与玩家手牌重叠
- 适配 iPad 等大屏设备（界面适配 iPad）

### 6.6 聊天记录消息数优化

**涉及端**：APP

- 优化聊天记录翻阅时的加载性能，超过分页数据量时显示 loading 菊花动画
- 调整单次加载消息数，避免卡顿

### 6.7 长按回到结束界面

**涉及端**：APP

在牌局进行中，**长按桌面**可快速回到（查看）最近一手的结算/结束界面，方便玩家回顾结算信息。

### 6.8 动效优化

**涉及端**：APP

更换一版与竞品（腾讯大掼蛋、边锋掼蛋等）风格一致的动效，主要包括：
- 出牌动效
- 结算动效
- 弹幕飞行动效（已在 §5.4.2 详细描述）
- 炸弹倍数飞向倍数显示位置的飞行动效（已在 §5.3.4 详细描述）

## 七、私人房管理

私人房（自建房）与俱乐部牌桌的主要区别在配置字段上，详见 §4 各玩法配置和 §3.1.1g 私人房额外字段。

二期私人房管理新增/变更概览：

| 变更点 | 说明 |
|--------|------|
| 新增转蛋玩法 | 私人房支持创建转蛋玩法牌桌 |
| 新增换座配置 | 开桌规则中新增换座开关 |
| 新增增值配置 Tab | 操作行动延时移入此 Tab，新增弹幕/记牌器等配置 |
| 新增防作弊-无障碍服务限制 | 默认开启，可关闭 |
| 额度审核 | 私人房独有，可设置审核时效 |
| 炸弹倍数自定义 | 支持加倍/翻倍/无倍数三种模式及各炸弹倍数单独配置 |
| 三把不过回2 | 过级玩法中可开启 |
| 弹幕/短语/记牌器 | 开桌时可配置开启/关闭 |

私人房开桌规则中，「仅限直属会员可见」开关：
- 开启后，非创建者直属会员无法看到此牌桌；其他平级代理可见此牌桌，但不允许坐下
