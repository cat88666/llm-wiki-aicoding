# 00-生产重构总控

> 本文是 Junk 生产级重构的唯一总控入口。旧资料只作为参考证据，不作为执行入口。

---

## 1. 总控目标

```
把 Junk 游戏准备模块重构成：
├── 边界清楚
├── 协议兼容
├── 行为可验证
├── 状态可追踪
├── 变更可回退
└── 生产可灰度
```

### 硬约束

| 约束 | 说明 |
|---|---|
| 协议 0 破坏 | 协议号、请求 DTO、响应 DTO、通知 DTO 默认不删、不改、不重命名 |
| 行为先等价 | 重构 PR 不顺手修 bug；bugfix 必须单独 PR |
| 先锁现状 | 没有已运行通过的特征测试，不做结构性重构 |
| 小步提交 | 一个 commit 只对应一个 R 项或一个测试点 |
| 不跨边界 | Junk 只重构游戏准备；Torsen、Duik、Ceres 只记录调用契约 |
| 可回滚 | 每个 PR 必须写清楚回滚方式 |

---

## 2. 唯一状态表

> 实际执行时复制 [`自主执行状态表模板`](templates/自主执行状态表模板.md)。AI 每完成一个原子任务必须立即更新状态表；其他文档不得维护重复完成度。

状态只能使用：

| 状态 | 含义 |
|---|---|
| `PENDING` | 前置任务未完成 |
| `READY` | 前置条件齐全，可自动执行 |
| `RUNNING` | 当前正在执行 |
| `BLOCKED` | 缺业务决策、环境或外部权限 |
| `FAILED` | 验证失败，等待修复任务 |
| `VERIFIED` | 实现和验证完成 |
| `APPROVED` | 审核通过 |
| `DONE` | 所有完成定义满足 |

## 3. 准入门禁

### 进入编码前

```
编码准入
├── spec 已审核
├── 特征测试已建立并运行通过
├── 协议契约已冻结
├── 状态副作用已列清
├── 回滚方式已写清
└── 阻塞项已关闭或保守默认已批准
```

### 进入 PR 前

| 检查项 | 要求 |
|---|---|
| diff 范围 | 只包含当前功能和当前 R 项 |
| 协议 | 请求、响应、通知 DTO 无破坏 |
| 测试 | 新增/修改测试及运行结果 |
| 副作用 | 缓存、RPC、MQ、通知、钱包、账单已核对 |
| 回滚 | 能 revert 或配置切回旧 flow |
| 文档 | spec、plan、review、变更记录同步 |

---

## 4. 待确认项处理

| 类型 | 处理规则 |
|---|---|
| 阻塞项 | 不确认不编码，例如协议废弃还是适配、资金口径变化 |
| 保守默认项 | 选择最兼容旧行为的实现，写入 PR 和待确认清单 |
| 观察项 | 允许灰度上线，但必须有日志、指标、回滚方式 |
| 跨团队项 | 只写调用契约，不替其他负责人重构 |

---

## 5. 生产风险分级

| 等级 | 特征 | 处理 |
|---|---|---|
| P0 | 资金、入座、退出、重连、协议破坏、并发状态错乱 | 必须有特征测试和人工审核 |
| P1 | 业务行为变化、异常兜底变化、风控策略变化 | 必须灰度观察 |
| P2 | 日志、代码结构、可读性、重复代码 | 可以随重构小步处理 |
| P3 | 命名、注释、文档补充 | 不和 P0/P1 混 commit |

## 6. 总控循环

```text
while 存在 READY 或 FAILED 任务:
  1. 选择风险最低、依赖最少的 READY 任务
  2. 将任务标记 RUNNING
  3. 执行一个原子 R 项
  4. 运行该任务要求的全部验证命令
  5. 通过：提交并标记 VERIFIED
  6. 失败：创建最小修复任务并标记 FAILED
  7. 业务或环境阻塞：生成阻塞决策单并标记 BLOCKED
  8. 更新状态表、证据链接和下游 READY 状态

无 READY/FAILED 任务时:
  生成最终验收报告
  结论为 COMPLETE 或 BLOCKED
```

总控不得因为单个功能 `BLOCKED` 而停止全部工作；应继续推进不依赖该功能的 READY 任务。
