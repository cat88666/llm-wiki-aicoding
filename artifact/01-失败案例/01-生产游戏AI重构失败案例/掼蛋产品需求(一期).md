# 掼蛋游戏 - 一期产品需求文档

> **文档版本**：一期
> **文档定位**：架构师视角，面向后端 / 前端开发者
> **目标**：开发者读完本文档后，无需再查阅原始设计稿，即可完整实现符合产品需求的掼蛋游戏系统。

---

## 一、产品目标

一期版本重点实现掼蛋**经典玩法**，确保用户体验流畅，避免过于复杂的功能开发；后续可逐步增加其他玩法和扩展功能。

### 一期核心功能框架

| 模块 | 内容 |
|------|------|
| 游戏用牌 | 所有玩法均使用 2 副扑克牌，大小王各 2 张，A\~K 各 8 张，共计 108 张 |
| 经典玩法 | 俱乐部开房、随机级牌、上贡玩法、红桃逢人配、接风、过级 |
| 对战模式 | 1v1（二人掼蛋）、2v2（四人掼蛋） |
| 牌桌配置 | 掼蛋核心玩法配置项（由管理后台维护） |
| 基础界面 | 简洁的游戏界面，重点突出牌桌、玩家信息和操作按钮 |

## 二、游戏概述

### 2.1 基本信息

| 项目 | 说明 |
|------|------|
| 游戏名称 | 掼蛋 |
| 游戏背景 | 流行于华东（淮安及周边）地区的扑克游戏，由"跑得快"和"八十分"演化而来 |
| 四人玩法 | 分两队，每队 2 人对战 |
| 二人玩法 | 1v1，剩余未发牌作废 |

### 2.2 游戏用牌

- 所有玩法均使用 **2 副扑克牌**
- 大王 2 张、小王 2 张、A\~K 各 8 张，**共计 108 张**
- 四人掼蛋：每人发 **27 张**手牌
- 二人掼蛋：每人发 **27 张**手牌，剩余 54 张作废（需保存用于牌局回放）

### 2.3 术语定义

| 术语 | 定义 |
|------|------|
| **级牌** | 每轮游戏的目标等级牌（从 2 升到 A）。级牌的点数优先级高于 A，低于小王。级牌可正常参与顺子、钢板、三连对、同花顺等牌型组合（按其原本点数参与）。 |
| **逢人配（万能牌）** | 当前过级队伍的**红桃级牌**即为逢人配。例如当前打 2，则红桃 2 为逢人配；其他花色的 2 为普通级牌。逢人配可与王牌以外的任意牌组成牌型；**单独打出逢人配按普通级牌计算**。 |
| **天王炸** | 2 张大王 + 2 张小王，最大的炸弹牌型。 |
| **超级炸弹** | 6 张及以上同点数牌（或加逢人配）组成的炸弹，统称超级炸弹。 |
| **同花顺** | 5 张点数相邻、花色相同的牌（含逢人配，可多张）。 |
| **接风** | 任意玩家出完手牌后，若其他玩家都要不起，则接下来一轮的出牌权归属于其队友。 |
| **头游 / 二游 / 三游 / 末游** | 四人游戏中，按出完牌的顺序依次为头游、二游、三游、末游。二人玩法中只有头游（胜利方）和二游（失败方）。 |
| **双上** | 一队玩家取得头游和二游。 |
| **单上（一头一三）** | 一队玩家一人头游、一人三游。 |
| **单上（一头一末）** | 一队玩家一人头游、一人末游。 |
| **双下** | 一队玩家两人分别为三游和末游（输方）。 |
| **单下** | 一队玩家一人末游、另一人二游。 |
| **进贡** | 上一手失败玩家须向胜利玩家进贡一张最大的牌（非红桃级牌）。 |
| **还贡** | 获得贡牌的一方从手牌中选一张点数 10 及以下的牌还给进贡方（不能是级牌）。 |
| **抗贡** | 进贡玩家手中拿到 2 张大王，可自动抗贡，无需进贡还贡。 |
| **必打** | 房主设定的最高目标级牌。胜利方必须满足：一位玩家为头游，另一位玩家**不能为末游**；否则继续打该级牌。累计三次失败则回 2 重新开始（"三把不过回 2"规则）。 |
| **过场** | 打必打级牌时，必须胜利方非末游才视为过，与必打规则一致。 |
| **升级（过级）** | 只有头游一方可以升级。双上升 3 级（或 4 级，可配置）；头游三游搭档升 2 级；头游末游搭档升 1 级。过 A 时，A 必打，不能通过升级跳过 A。 |
| **三把不过回 2** | 连续三次打必打级牌失败，则本队伍下一手从 2 开始升级，另一队等级不受影响。 |
| **转蛋 / 团团转** | 无升级，固定级牌 2 或随机级牌；抓明牌换队友。 |

## 三、牌型规则

### 3.1 牌型一览表

| 牌型 | 张数 | 描述 | 示例 |
|------|------|------|------|
| 天王炸 | 4 | 2 张大王 + 2 张小王，最大炸弹 | JOKER×2 + joker×2 |
| 超级炸弹 | ≥6 | 6 张及以上相同点数（含逢人配），相同张数越多越大，同张数比点数 | 六个 7、七个 8… |
| 同花顺 | ≥5 | 5 张（含逢人配可多张）点数相邻且花色相同的牌 | 7♠-8♠-9♠-10♠-J♠ |
| 五炸 | 5 | 5 张相同点数 | 55555 |
| 四炸 | 4 | 4 张相同点数 | 4444 |
| 钢板 | 6 | 2 组点数相邻的三张（只能 2 组，不能带其他牌） | 333-444 |
| 连对 | ≥6 | 至少 3 对连续点数对子 | 33-44-55 |
| 三带二 | 5 | 3 张同点数 + 1 对 | 888+99 |
| 顺子 | 5 | 5 张点数连续、花色不限（大小王不可加入） | 3-4-5-6-7 |
| 对子 | 2 | 2 张同点数 | 77 |
| 单牌 | 1 | 任意单张（大王 > 小王 > 级牌 > A > K…） | — |

### 3.2 牌型大小规则

#### 炸弹排序（从大到小）

```
天王炸 > 超级炸弹（张数越多越大，同张数比点数） > 同花顺 > 五炸 > 四炸 > 其他牌型
```

**炸弹可以压制任何非炸弹牌型。**

#### 单牌大小

```
大王 > 小王 > 级牌 > A > K > Q > J > 10 > 9 > 8 > 7 > 6 > 5 > 4 > 3 > 2
```

#### 对子大小

同单牌大小规则。

#### 顺子大小

- 最小顺子：A-2-3-4-5
- 最大顺子：10-J-Q-K-A
- **A 只能放在顺子的最前面或最末尾**，不能放中间

#### 连对大小

- 最小连对：AA-22-33
- 最大连对：QQ-KK-AA
- **AA 只能在连对首尾，不能在中间**

#### 钢板大小

- 最小钢板：AAA-222
- 最大钢板：KKK-AAA
- **AAA 只能在钢板首位**

#### 三带二大小

只比三带部分的大小，规则同单牌大小。

#### 炸弹大小细则

| 炸弹类型 | 最小 | 最大 |
|----------|------|------|
| 四炸 | 2222（2 为非级牌） | 4 个级牌 |
| 五炸 | 5 个 2（2 为非级牌） | 5 个级牌 |
| 同花顺 | A-2-3-4-5（同花） | 10-J-Q-K-A（同花） |
| 超级炸弹 | 6 个 2（2 为非级牌） | 10 个 A 级牌（8 张 A + 2 张逢人配） |
| 天王炸 | — | 2 大王 + 2 小王（唯一） |

**注意**：大小王不参与普通炸弹点数比较；级牌参与顺子 / 连对 / 钢板 / 同花顺时按其原本点数计算。

#### 非炸弹牌型

除炸弹和同花顺外，其他牌型**只能同牌型比较大小**（不能跨牌型压牌）。

### 3.3 特殊牌型（逢人配）

逢人配（红桃级牌）规则如下：

1. **逢人配可与王牌以外任意牌组合**，可组成：对子 / 三张 / 三带二 / 顺子 / 连对 / 钢板 / 炸弹 / 同花顺 / 超级炸弹
2. 逢人配组成的炸弹与同点数硬炸一样大（**不区分软炸 / 硬炸**）
3. 逢人配单张打出按**级牌**计算
4. **逢人配 + 两对组成三带二时，默认按最大的一对来组三带**
5. **逢人配 + 四张连牌组成顺子或同花顺时，默认组成可能的最大同花顺**
6. 在两张逢人配情况下，可能既能组成连对也能组成钢板（例如打 10 时：33441010）：
   - **非优先出牌**：根据上家出的牌型系统自动判断（上家出连对则匹配连对）
   - **优先出牌**：玩家自主选择牌型（弹出牌型选择框）；**出牌时间超时时，自动作为钢板打出**
7. 使用逢人配可组成不同大小的相同牌型时，**默认按最大牌型打出**。例如：
   - 对 A + 对 5 + 逢人配 → 优先组成三 A 带对 5
   - 5678 + 逢人配 → 优先组成 56789
   - 567 + 两逢人配 → 优先组成 56789

## 四、游戏玩法规则

### 4.1 四人掼蛋（过级玩法）

#### 4.1.1 基本规则

- 队伍分配：坐对家的为同一队（1 号位与 3 号位一队，2 号位与 4 号位一队）
- 座位号定义：正下方为 **1 号位**，逆时针依次为 2 号、3 号、4 号位
- 满 4 人自动开始；新牌桌默认从 **2** 开始打
- 任意一队两名玩家全部出完手牌，本手游戏结束，进行单手结算
- **时间限制**（均可后台配置）：
  - 进贡选牌时间：**10 秒**
  - 还贡选牌时间：**20 秒**
  - 出牌时间：**30 秒**（可配置）
  - 补码时长：**30 秒**（可配置）
- 准备超时：超时默认继续，直到本局结束

#### 4.1.2 牌桌创建配置项

房主创建牌桌时需设定以下参数（均由管理后台提供选项）：

| 配置项 | 说明 |
|--------|------|
| 超时自动解散（min） | 默认 30 分钟 |
| 牌桌名称 | 创建者自定义 |
| 底分 | 后台配置可选项 |
| 最小带入（底分倍数） | 后台配置可选倍数 |
| 出牌时间（秒） | 后台配置多个选项，创建时选定 |
| 时长（h） | 牌桌游戏时长 |
| 过级中途离开 | 开启则允许过级中途离开，关闭则不允许 |
| 过级数（必打） | 本局过几级（2\~A 可配置）。例如过 6 算一局，从 2 打到成功过 6 后本局结束 |
| 双上升级级数 | 默认升 3 级（后期可增加升 4 级选项） |
| 三把不过回 2 | 累计三次过目标级失败，本队下一手从 2 重新升级 |
| 还贡规则 | 选择点数 10 及以下 或 任意牌 |
| 报牌张数 | 剩余多少张触发报牌（默认 10 张，可配置） |
| 每手服务费方式 | 盈利比例 或 底分比例（由管理后台配置，前端选择） |
| 盈利比例（%） | 后台配置 |
| 底分比例（%） | 后台配置，参与游戏的玩家均需支付 |
| 每手比例上限（倍底分） | 后台设置 |
| 局服务费比例（%） | 后台配置 |

#### 4.1.3 出牌规则

**首出判定：**
- 新牌桌：系统随机分配首出玩家，该玩家所在队伍为打 2 的一方
- 后续手：根据本局进贡情况确定首出玩家（详见进贡规则）

**出牌顺序**：逆时针方向轮流出牌

**首出超时处理**：
- 自动出最小点数的非炸弹牌型
- 只有炸弹时，自动打出最小的炸弹
- 手牌中有逢人配时，优先级：有炸弹则和最大的炸弹组合；无炸弹有 3 张则组成炸弹；无炸弹无 3 张则和最大的非大小王点数组合成对子 / 三张

**要牌超时处理**：默认 3 秒后不出

**接风规则**：
- 首个玩家出完最后一手牌后，其他三家无人要牌，则由**该玩家队友**出牌
- 如果二游与头游不是同一队，二游出完手牌后另外两家无人压牌，则由**二游队友**出牌

**游戏结束条件**：同一队玩家全部出完手牌

#### 4.1.4 游次判定

1. 按出完牌的顺序依次为：头游 → 二游 → 三游 → 末游
2. **如果一队玩家同时获得头游和二游，游戏直接结束**；此时二游的下家获得三游，上家为末游

#### 4.1.5 升级（过级）规则

| 胜利游次组合 | 升级级数 |
|-------------|---------|
| 双上（头游 + 二游） | 升 3 级（可配置升 4 级） |
| 单上（头游 + 三游） | 升 2 级 |
| 单上（头游 + 末游） | 升 1 级 |

- **必打规则**：房主设定的最高目标级（必打级）不能通过升级跳过。遇到必打级时，必须胜利方一位玩家为头游，另一位玩家**不能为末游**，才算过级成功
- **头游末游胜利情况**：级牌视为未过，需继续打该级牌
- **三把不过回 2**：累计三次过目标级失败，本队伍下一手从 2 重新升级；另一队等级不受影响

#### 4.1.6 进贡与抗贡规则

**双贡（上一手双下）：**
- 双下两位玩家分别进贡手牌中最大的牌（除逢人配）给对手头游和二游
- 头游将手牌中点数 10 及以下的牌还贡给进贡大牌的玩家；二游同理还贡给另一位
- 首出规则：进贡牌不同大时，由进贡大牌的玩家首出；进贡牌相同大时，由上一手头游的下家首出

**单贡（上一手末游进贡）：**
- 末游将手牌中最大的牌（除逢人配）进贡给头游
- 头游将手牌中点数 10 及以下的牌还贡给末游
- **首出规则**：末游玩家首出（抗贡失败情况）；同队伍也需进贡

**抗贡：**
- **双下抗贡**：双下队伍在本局中拿到 2 张大王（两人各持 1 张或一人持 2 张均算）则抗贡成功
- **单贡抗贡**：末游玩家拿到 2 张大王才算抗贡成功
- 抗贡成功：不论双上还是单上，统一由上一手头游玩家首出

**进贡选牌规则（系统默认逻辑）：**
- 有大王：只能选大王
- 无大王有小王：只能选小王
- 无大小王：选除逢人配（红桃级牌）以外的最大手牌
- 除大小王外其他牌，玩家可选不同花色的相同点数牌

**进贡超时处理：**
- 系统默认选一张除逢人配外最大的牌进贡；有多张相同时取最下方那张

**还贡选牌规则：**
- 要求选点数 10 及以下手牌；若无 10 及以下手牌，可选任意手牌（逢人配除外）
- 还贡时无需玩家选中，系统有默认选牌

**还贡超时处理：**

有点数 10 及以下手牌时：
- 优先级：单张 > （拆）对子 > （拆）三张 > （拆）炸弹
- 按点数从小到大选择（花色优先级：方块 > 梅花 > 红桃 > 黑桃）

无点数 10 及以下手牌时（还贡任意牌）：
- 优先级：单张（非逢人配）> （拆）对子（非逢人配）> （拆）三张 > （拆）炸弹
- 按点数从小到大选择，花色优先级同上

#### 4.1.7 比牌规则

详见 § 3.2 牌型大小规则。

**级牌参与顺子 / 连对 / 钢板 / 同花顺时，按原本点数位置计算。**

#### 4.1.8 初始摆牌规则（发牌后手牌排序）

**牌值排序（从左到右从大到小）：**

```
大王 > 小王 > 红心级牌 > 级牌 > A > K > Q > J > 10 > 9 > 8 > 7 > 6 > 5 > 4 > 3 > 2
```

**花色排序（同点数牌从上到下）：**

```
黑桃 > 红桃 > 梅花 > 方块
```

**理牌区域说明：**
- 炸弹：显示在手牌**左侧**，按炸弹大小从左往右排列
- 组合牌（钢板 / 连对 / 三带二 / 顺子）：显示在手牌**右侧**，从右往左依次为顺子 / 三带二 / 连对 / 钢板

#### 4.1.9 重新开始规则

- 当前牌桌有玩家站起：下一局从 2 重新开始打
- 当前牌桌有队伍过级成功：下一手两队重新从 2 开始打

#### 4.1.10 组队规则

- 玩家进入牌桌后自由选择座位，坐对家的为同一队
- 玩家入坐后不得直接更换位置，需先站起再选择其他空位入坐（限 4 人房）

### 4.2 二人掼蛋

#### 4.2.1 基本规则

- 满 2 人自动开始，两人自动成为对手，**不能同队**
- 发牌：每人 27 张，剩余牌作废（**保存用于牌局回放**）
- 任意一方出完手牌，本手游戏结束，进行单手结算
- 结算后破产玩家须补码，否则被强制站起

#### 4.2.2 牌桌创建配置项

在四人掼蛋基础上额外配置：
- **进贡**：可选开启进贡或不进贡
  - 选择进贡：还贡默认选择点数 10 及以下

#### 4.2.3 出牌规则

- 新牌桌首出玩家系统随机分配；谁首出，该轮就是谁打 2
- 出牌顺序：两方轮流出牌

**首出规则（非首手）：**
- 不进贡玩法：上一手胜利方首出
- 进贡玩法（抗贡失败）：本手进贡玩家首出
- 进贡玩法（抗贡成功）：上一手胜利方首出

**结束条件**：一个玩家全部出完手牌

**特殊情况（逢人配牌型选择）**：规则与四人掼蛋相同（见 § 4.1.3）

#### 4.2.4 胜利判定

- 先出完手牌的一方获胜，固定升 **1 级**
- 游次：头游（胜利） / 二游（失败）

#### 4.2.5 进贡与抗贡规则

- 上一手失败玩家给胜利玩家进贡；胜利玩家给失败玩家还贡；本手进贡玩家首出
- 上一手失败玩家抓到**一对大王**，抗贡成功，由上一手胜利玩家首出
- **还贡规则**：同四人掼蛋（10 及以下；无则可还任意牌，逢人配除外）

#### 4.2.6 其他规则

逢人配规则、重新开始规则、牌桌解散规则、比牌规则、初始摆牌规则均与四人掼蛋一致。

- **无升级**，固定使用级牌 2 或随机级牌
- 抓明牌换队友
- 其他规则待二期补充

## 五、结算规则

### 5.1 基础结算公式

**基础公式：**

```
本手输赢金额 = 底分 × 排名倍数
```

| 胜利游次 | 排名倍数 |
|----------|---------|
| 双上（头游 + 二游） | 3 倍 |
| 单上（头游 + 三游） | 2 倍 |
| 单上（头游 + 末游） | 1 倍 |
| 二人掼蛋固定 | 1 倍 |

**防以小博大策略：**
- 规则：本局带入多少筹码，本局最多输赢多少筹码
- **玩家最大输赢不能超过玩家牌桌当前筹码量**

防以小博大计算：
1. 结算时，如果玩家输赢超过牌桌筹码则将输赢金额改为筹码值
2. **输赢金额平衡处理**：
   - 分别计算手牌输分玩家总输赢金额（绝对值 X）和赢分玩家总输赢金额（绝对值 Y）
   - 较小绝对值（X）一方玩家输赢按修改后分数直接结算
   - 较大绝对值（Y）一方玩家输赢 = 该玩家修改后输赢 ÷ Y × X
   - 保留两位小数（无条件舍去），**小数尾差全放到最大输家或最大赢家**（有多个时任选一个）

### 5.2 炸弹加倍 / 翻倍规则

#### 炸弹加倍模式

**结算公式：**
```
输赢金额 = 底分 × (1 + 炸弹倍数) × 排名倍数
```

| 打出牌型 | 炸弹加倍数 |
|----------|-----------|
| 四炸 / 五炸 | +1 |
| 同花顺 | +2 |
| 六 / 七 / 八 / 九 / 十炸（超级炸弹） | +3 |
| 天王炸 | +4 |

#### 炸弹翻倍模式

**结算公式：**
```
输赢金额 = 底分 × 炸弹倍数 × 排名倍数
```

| 打出牌型 | 炸弹翻倍数 |
|----------|-----------|
| 四炸 / 五炸 / 同花顺 | ×2 |
| 六炸 / 七炸 | ×3（具体倍数由后台配置） |
| 八炸 / 九炸 / 十炸 | ×4（具体倍数由后台配置） |

**倍数说明：**
- 每当有炸弹打出时，倍数在当前基础上叠加或翻倍
- 游戏结束时展示计算排名倍数后的总倍数
- **达到封顶倍数时，倍数不再增加**
- 封顶结算公式：`底分 × 封顶倍数`

### 5.3 服务费计算

#### 手牌服务费（每手收取）

**按盈利比例：**
```
服务费 = 盈利筹码 × 盈利比例（%）
```
- 只有**赢家**扣除
- 有上限：单手服务费不超过后台配置的上限值（上限为所有玩家合计上限）
- 例：底分 100，盈利比例 5%，上限底分 0.1 倍（= 10），A 赢 1000，服务费 = min(1000×5%, 10) = 10

**按底分比例：**
```
服务费 = 底分 × 底分比例（%）
```
- **输赢玩家均需支付**

**特殊情况：**
- 结算后牌桌筹码不足以抵扣服务费时，有多少扣多少，剩余筹码为 0 则不扣服务费
- **触发防以小博大时**，将收取的手牌服务费按各玩家计算值占比分摊；先无条件舍去保留 2 位小数，尾差分给最大赢家

**服务费贡献计算（盈利比例模式）：**
- 总手牌服务费 × 各玩家贡献计算值占比
- 贡献计算值：
  - 输家：`-1 × 盈亏金额 × 膨胀系数（目前为 10）`
  - 赢家：`盈亏金额`（不乘膨胀系数）

#### 局服务费（每局结束时收取）

- 在对局结算时进行局服务费扣除
- **按底分比例**：收取每人 `底分 × 收费比例`
- **按盈利比例**：收取一局游戏中 `(总打牌盈利 + 彩金盈利) × 收费比例`（无上限）
- **特殊情况**：如果一局游戏中第一手未结束即被强制立即解散，**不收取局服务费**

### 5.4 数字格式规范

#### 后端精度

- 精度：取小数点后两位
- **服务费特殊处理：**
  - 计算出的服务费 > 0.01 且出现三位及以上小数：将三位以上小数部分**向上取整**后保留两位
  - 计算出的服务费 ≤ 0.01：收取 **0.01**（最低收费保护）

#### 前端展示

- 没有小数时，只显示整数（例：结算 -10.00 展示为 **-10**）
- 有小数时只取有效小数（例：结算 +8.10 展示为 **+8.1**）

**需遵守此规范的数字展示场景：**
- 带入、带出、补充筹码
- 桌面筹码数
- 底分展示
- 单手结算飘分
- 彩金飘分
- 彩金钱包余额、账变记录
- 账户余额展示
- 单轮结算界面数据展示
- 实时盈亏数据展示
- 牌局回顾数据展示
- 弱提示内容中的数据展示

## 六、APP 功能需求

### 6.1 牌桌创建与座位

#### 牌桌人数与座位布局

**四人玩法：**
- 4 个座位，正下方为 1 号位，顺时针依次为 2、3、4 号位
- 1 号位与 3 号位（对家）为一队，2 号位与 4 号位（对家）为一队
- 玩家坐下后，界面自动旋转使本家座位位于正下方

**二人玩法：**
- 2 个座位，两人坐对家位置
- 隐藏原队友手牌和位置，两人强制成为对手

#### 入座流程

1. **旁观状态（有空位）**：玩家可自由选择空位坐下
2. **坐下后未达开局人数**：显示"等待其他玩家入座"
3. **满员自动开始**：四人满 4 人、二人满 2 人自动开始
4. **游戏已开始时进入牌桌**：玩家处于旁观者视角（"观战中"），随机视角，不能看当前玩家手牌

**旁观者说明：**
- 不显示队友 / 对手标记
- 默认隐藏聊天按钮（牌桌开启允许旁观者发弹幕，或总代 / 房主为旁观者时不隐藏）
- 可观看出牌及操作倒计时、单手结算展示

**队友 / 对手标记规则：**
- 入座后才显示队友 / 对手标记
- 玩家站起后，该玩家位置所有信息和标记隐藏，切换到旁观者视角

**特殊情况：**
- 设备强制竖屏进桌时：提示"请设置为横屏，进行游戏"；点击"离开"返回牌桌列表

### 6.2 带入 / 补码逻辑

#### 首次带入

- 玩家点击【坐下】弹出带入弹框
- **带入上限**：1000 倍底分（可配置），不含彩金金额
- 滑块默认在最左侧（最小带入位置）
- 若玩家现金钱包 > 带入上限：右侧滑至最大不超过带入上限
- 若玩家现金钱包 ≤ 带入上限：右侧滑至最大不超过玩家余额
- 若玩家余额 < 最小带入：弹 toast 提示"最低带入 XX，您的现金钱包余额不足"，不弹带入弹框

**输入规则：** 带入金额不能为负数且非小数第一位不能是 0

#### 非首次带入

- 若上次离开牌桌筹码 ≥ 最小带入：**自动带入上次离开时的筹码量**
- 若现金钱包余额 < 上次离开筹码：弹带入弹框，全部带入余额
- 若上次离开筹码 < 最小带入：弹带入弹框，最小值为最小带入（默认值为最小带入，不自动走一步）

#### 滑块步长规则

- 每步长 = 1/4 最小带入
- 步数计算：`(最大带入上限 - 最小值) ÷ (最小带入 / 4)`，向上取整
- 最后一步的滑动条长度按小数比例缩短
- **每步数值低于 0.01 时，向上取至 0.01**（带入带出规则一致）
- 最小值等于最大值时，滑动条直接拉满；最小和最大按钮置灰不可点
- 玩家输入金额小于最小值时，自动换成最小值

#### 带出筹码

- 游戏状态下可手动带出牌桌筹码，**必要保留最小带入量**
- 带出最小值：0；带出最大值：牌桌筹码 - 最小带入（计算为负数统一为 0）
- 带出成功条件：游戏结算后，牌桌剩余筹码 ≥ 最小带入 + 带出金额
- 若牌桌筹码 < 带出筹码：不允许带出，toast 提示"筹码不足，带出筹码失败"
- 同时进行带入和带出操作时：**先执行带入，再执行带出**
- 牌桌若开启局服务费：带出筹码只能在一局游戏结束、计算完服务费后才带出成功
- 带出筹码**默认不勾选"默认带出最大值"**，玩家勾选状态只影响相同游戏类型

#### 补码规则

**手动补码：**
- 当剩余筹码 < 最小带入时：最小补码 = 最小带入 - 剩余筹码
- 当剩余筹码 ≥ 最小带入时：最小补码 = 0，默认补码 = 1/4 最小带入
  - 若钱包余额 < 默认补码：默认补码 = 钱包余额
- 若钱包余额 = 0：不弹补码弹窗，弹 toast 提示"您的现金钱包余额不足"
- 当牌桌筹码已达带入上限：点击补码弹提示"带入已达上限，无法补码"
- 玩家钱包余额 < 1/4 最小带入时：补码弹窗默认拉满，默认补码余额全部

**自动补码：**
- 玩家在带入或补码界面勾选"自动补码"后，筹码低于指定值时自动补充
- 自动补码数值由后台管理配置
- **默认勾选**自动补码

**破产 / 强制补码：**
- 规定时间内未完成补码会被**强制站起**

### 6.3 定级

- **定级规则**：所有新一轮游戏默认从 2 开始，双方初始级牌统一为 2
- 已在游戏中的牌桌：按上一队过级级牌 + 过级后升级级数确认新级牌
- **定级牌动画**：旋转翻滚动画（翻滚 0.5 秒后缩小飞向左上角级牌展示区）
- 定完级牌后主界面左上方显示我方级牌和对方级牌
- **当前局哪方过级，该队伍级牌展示区高亮**

### 6.4 发牌

- 发牌规则：108 张牌打乱后随机给每位玩家各发 27 张
- 二人掼蛋：每人各发 27 张，剩余牌作废
- **发牌动画**：定级牌过程中同步发牌，手牌从牌桌中央往下飞落（半透明 → 实体），从中间往两侧依次落下

**手牌初始摆牌规则：**
- 牌值从左到右排列（大 → 小），相同点数按花色从上至下排列
- 牌值排序：大王 > 小王 > 红心级牌 > 级牌 > A > K > Q > J > 10 > 9 > 8 > 7 > 6 > 5 > 4 > 3 > 2
- 花色排序：黑桃 > 红桃 > 梅花 > 方块（从上到下）

**同花顺快速找牌：**
- 发牌后若玩家手牌可组成同花顺，对应花色的同花顺按钮高亮且可点击
- 点击后选中对应花色的同花顺手牌
- 有多种同花顺时点击后从小到大切换

**底部按钮**：发牌后同时展示同花顺 / 理牌 / 一键理按钮

### 6.5 出牌逻辑

#### 出牌倒计时显示位置

- 首出：倒计时显示在【提示按钮】和【出牌按钮】之间
- 要牌：倒计时显示在【不出按钮】和【提示按钮】之间

#### 出牌按钮规则

- 出牌按钮：常驻高亮状态
- 有选中手牌时点击出牌但不符合规则：提示"您选择的牌不符合规则"，选中状态保持

#### 出牌展示规则

- 已出手牌：从左往右从小到大排序展示（三带二：三张显示左边）
- **出牌显示周期**：轮到自己出牌前，上一轮出的牌隐藏（转完一圈后本家的牌从桌面消失）
- 所有人不出后新一轮：轮到出牌时清空桌面不出信息

#### 提示规则

**首出或无人要牌时点击提示：**
- 无理牌：从右到左从小到大按牌型顺序提示（炸弹除外）
- 有理牌：先从散牌区域从右到左从小到大提示，无散牌时直接从右到左提示
- 多次点击：按优先级从小到大循环

**接牌时点击提示：**
- 无理牌：优先同上家牌型，按从小到大轮询，只提示比上家大的牌型
- 有理牌：同上逻辑，最后才考虑拆组合牌型
- 玩家无牌大过上家：点击提示后显示"没有牌大过上家"

#### 不出按钮

- 仅在要牌时出现
- 有手牌选中时，点击不出后取消选中状态

#### 接风动画与展示

- 接风触发时：在队友头像旁播放接风动画
- **队友手牌展示**：出完牌的玩家在自己手牌区域展示队友手牌，并显示"队友手牌"

#### 记牌器

- 每出一张牌，记牌器对应牌型减少一张
- 记牌器中的牌须与实际玩家手牌保持对应

#### 报牌规则

- 剩余手牌 ≤ 10 张（可后台配置）时，在头像旁显示剩余手牌数量
- **首出动画播放时，所有玩家头像旁实时显示剩余手牌数量**
- 显示位置：本家在头像及加倍上方；对家在头像左侧；上家在头像右侧；下家在头像左侧

#### 炸弹动画规则

| 场景 | 动画 |
|------|------|
| 炸弹打对家或上下家的牌 | 炸弹飞向对应玩家头像处炸开 |
| 炸弹空炸 | 直接在桌面中间炸开 |
| 天王炸 | 满屏炸开，不指向特定玩家 |

炸弹越大，爆炸效果越强烈：天王炸 > 6\~10 炸 / 同花顺

#### 特殊牌型选择（逢人配优先出牌）

玩家在优先出牌时选择可组成不同牌型的手牌（如一对逢人配 + 两连对），弹出牌型选择框让玩家选钢板或连对；**出牌时间超时自动作为钢板打出**。

### 6.6 单手结算

#### 结算流程

```
游戏结束 → 显示玩家剩余手牌 → 显示所有玩家名次 → 结算 → 检查玩家余额 → 破产玩家处理
```

#### 剩余手牌展示

- 任意一队玩家全部出完手牌，本手游戏结束
- 剩余玩家手牌全部以平铺方式展示在牌桌上（最多 14 张一行，超出换行）
- 玩家点击继续后桌面展示内容消失

#### 游次展示

- 头游 / 二游：一种颜色；三游 / 末游：另一种颜色
- 玩家游次显示在头像右上角
- 出完牌后游戏未结束：游次信息出现在桌面，展示 1 秒后飞到对应头像
- 出完牌后游戏立即结束：所有玩家游次信息直接展示在头像和桌面

#### 倍数区

- 显示本局结束后的总倍数（含炸弹倍数 × 排名倍数）
- 游戏结束时展示计算排名倍数后的**总倍数**
- 达到封顶倍数时不再变化

#### 结算数字展示

- 结算输赢使用两种不同颜色区分
- 显示位置：本家在头像上方，对家在头像下方，上家在右侧，下家在左侧
- 服务费超 2 位小数：按无条件进位保留 2 位小数
- **扣除服务费后赢分为 0 时，飘分"+0"显示为金色**

#### 破产处理

- 结算时检测剩余筹码 = 0 时显示破产标识并弹出补码弹窗
- 破产玩家未补码则在结算完毕后**强制站起**，位置空出来给其他玩家

#### 等待开始（可中途离场）

- 倒计时结束（默认 5 秒，有人破产则以补码时间为准）：未操作则强制站起
- 玩家可手动点击站起离开座位
- 掉线或退出超时：自动强制站起
- 有玩家离场出现空位时：倒计时隐藏，只显示"等待开始中..."（本家仍显示倒计时和继续按钮）

#### 等待开始（不可中途离场）

- 倒计时结束：未点击继续的玩家自动准备开始
- 玩家不能退出或站起；掉线或退出默认系统托管

#### 牌桌时长超时处理

- 牌桌时长结束时，如牌局未结束：在**本手结束后解散牌桌**
- 若开启彩金：本手是本轮最后一手则正常结算彩金；非最后一手则所有人不扣彩金

### 6.7 对局结算（过级结算）

#### 触发条件

- **只有队伍打过目标级后才弹出单局结算弹窗**
- 过级中途有玩家主动离开、破产离开等任何原因导致过级中断：**不走单局结算，只走单手结算**

#### 单局结算展示内容

| 字段 | 说明 |
|------|------|
| 牌桌名 | 该牌桌的牌桌名 |
| 总手数 | 本局总手数（从开局到结算计一手） |
| 总带入 | 所有玩家本局累计带入（不含彩金），含补码 |
| 总流水 | 所有盈利的总和（未抽水之前） |
| 底分 | 房主设定的底分值 |
| 过级数 | 房主设定的过级目标 |
| 玩家带入 | 本局游戏累计带入（不含彩金） |
| 玩家胜利次数 | 本局胜利的累计次数 |
| 玩家头游次数 | 本局取得头游的累计次数 |
| 玩家总盈亏 | 包含彩金结算和服务费 |
| 彩金盈亏 | 开启彩金时展示（未开启不显示） |

- 排序：本家、对家、上家、下家
- **本家玩家信息用不同颜色标记**

#### 单局结算倒计时

- 无人需要补码：**15 秒**；超时未操作则从坐位站起
- 有人需要补码：倒计时与补码时间一致
- 站起 / 等待下一局 / 超时自动站起时关闭单局结算弹窗

#### 破产处理（单局结算）

- 一局游戏结束后，破产玩家不展示单轮结算界面，**直接弹出补码界面**
- 玩家头像展示"补码中 + 倒计时"状态
- 不补码或补码超时：强制站起，清空桌面
- 单局游戏结束补彩金与补码流程一致

#### 操作按钮

- **站起**：从座位离开，位置空出来供其他玩家入坐
- **继续下一局**：继续在当前牌桌进行新一局游戏（新局从 2 重新开始打）

### 6.8 整局结束

整局结束后所有玩家全部站起，页面跳转至**战绩界面**。

#### 战绩界面字段

| 字段 | 类型 | 说明 |
|------|------|------|
| 牌桌 ID | 字符串 | 系统自动生成，点击查看详情时使用 |
| 牌桌名称 | 字符串 | 房主创建时填写 |
| 币种 | — | 系统自动生成 |
| 总手数 | 正整数 | 本牌桌总手牌数 |
| 总带入 | 数值 | 所有参与玩家总带入（含补码） |
| 总流水 | 数值 | 所有盈利总和（未抽水之前） |
| 底分 | — | 读取创建数据 |
| 游戏名称 | — | 四人掼蛋 / 二人掼蛋 |
| 牌桌开始时间 | 时间 | 游戏正式开始时间 |
| 牌桌时长 | 时间 | 创建时设定，含延长时间 |

**手牌详情**：点击可跳转到该牌桌所有牌局回顾详情界面

#### 玩家标签

| 标签 | 判定规则 |
|------|---------|
| **MVP** | 盈利最多；相同则比头游次数；再相同则取最近结束的 |
| **土豪** | 带入最多；相同则取最近结束的 |
| **大鱼** | 亏损最多；相同则取最近结束的 |

**战绩列表排序**：按盈利排名；盈利相同则按时间倒序

**注意**：战绩详情里不展示异常牌局

### 6.9 托管功能

#### 进入托管条件

- 玩家**连续三次超时未操作**，自动进入超时托管状态

#### 托管时操作逻辑

| 场景 | 托管操作 |
|------|---------|
| 首出超时 | 出最小点数的非炸弹牌型；只有炸弹时打最小炸弹 |
| 要牌超时 | 默认不出 |
| 进贡超时 | 默认选最大手牌进贡（相同点数按花色：方块 > 梅花 > 红桃 > 黑桃） |
| 还贡超时 | 有 10 及以下：单张 > 对子 > 三张 > 炸弹，按点数从小到大，花色优先级同上；无 10 及以下：规则同上 |

托管状态下手牌中有逢人配时：
- 有炸弹：和最大的炸弹组合
- 无炸弹有 3 张：和最大的 3 张组成炸弹
- 无炸弹无 3 张：和最大的非大小王点数组合成对子 / 三张

**托管操作时间：统一 3 秒**

#### 托管状态标识与界面

- 托管中的玩家头像处显示机器人托管标识
- 托管状态下玩家手牌上方显示【取消托管】按钮
- 托管状态下除取消托管外，不可进行任何其他牌桌功能操作
- **断线重连弹窗 / 牌桌解散通知弹窗**：在托管状态上层展示，玩家可操作
- 已打开的弹窗在托管上层；托管后游戏自动弹出的弹窗也在托管上层

**托管层级**：在游戏内所有弹窗的下层

#### 取消托管条件

- 玩家手动点击【取消托管】
- **可中途离场牌桌**：每手游戏结束后自动取消托管；重新超时 3 次后再进入托管
- **不可中途离场牌桌**：破产强制站起时取消；本轮游戏结束后自动取消；其他情况一直保持托管
- 取消托管后，再次出现超时**直接进入托管状态**（即重新触发需连续超时 3 次的规则被重置）

**取消托管后重新计数**：需再次连续超时 3 次才进入托管

#### 断线处理

- 玩家断线后超时按托管处理
- 断线重连后取消托管状态

### 6.11 牌桌解散

#### 解散权限

1. 总代可见所有牌桌的解散功能
2. 其他代理只可见自己创建牌桌的解散功能

#### 解散流程

```
发起解散 → 确认弹窗 → 检查是否有进行中手牌
  → 有：本手结束后强制解散
  → 无：立即解散
```

**框架处理说明**：必须等下一手开局时才判断牌桌是否已解散，因此在所有玩家点继续或倒计时结束时才弹出已解散弹窗。

#### 延长牌桌时长

- 房主可延长 1 小时：点击确定后，toast 提示"牌桌时长已延长1小时"

#### 解散后提示语

| 解散类型 | 提示语 | 跳转页面 |
|----------|--------|---------|
| 手数 = 0（未打牌即解散） | 当前牌桌已解散，您将在3s后返回大厅 | 俱乐部牌桌列表 |
| 手数 > 0 且当前无进行中手牌（非彩金） | 当前牌桌已解散，您将在3s后返回大厅 | 战绩界面 |
| 手数 > 0 且当前无进行中手牌（彩金玩家） | 当前牌桌已解散，您将在3s后返回大厅，过级彩金将原路返回 | 战绩界面 |
| 本手后解散且本手即本轮最后一手（彩金正常结算） | 当前牌局已解散，您将在3s后返回大厅 | — |
| 本手后解散且本轮未结束（彩金原路返回） | 当前牌桌已解散，您将在3s后返回大厅，过级彩金将原路返回 | — |

### 6.12 牌局回顾

#### 功能说明

提供对每一手牌局的完整回顾，支持查看每手的手牌详情、盈亏情况、游次等信息。

#### 页面信息

| 元素 | 说明 |
|------|------|
| 当前手数 / 总手数 | 如 1/14，打开时默认显示最新一手 |
| 翻页 | 支持跳转至：上一手 / 下一手 / 第一手 / 最后一手 |
| 玩家头像区 | 头像、昵称、余额、与本家位置关系、队友 / 对手标记、房主标识 |

#### 玩家列表

| 字段 | 说明 |
|------|------|
| 名次 | 本手游次：头游 / 二游 / 三游 / 末游 |
| 倍数 | 本局结算倍数（炸弹倍数 × 排名倍数） |

**列表显示排序：**

- 旁观者视角：首位为首家出牌玩家，依次为下家、对家、上家；排序以 0 号位为首

**本家信息需高亮凸显；盈亏数字颜色需与其他字体颜色区分。**

#### 防以小博大提示

- 触发防以小博大机制时，在"查看手牌"按钮右上角显示 **？图标**
- 触发类型与提示文字：

|----------|---------|

| 被动触发（不够输） | 初始筹码 xxx，不够输触发"防以小博大" |
| 他人本金不足（被动调整） | 有玩家本金不足，触发"防以小博大"，您的结算金额有受到调整 |

#### 已选规则展示

- 炸弹翻倍 / 加倍、封顶倍数、过级彩金：**已选的规则才展示，未选规则不展示**
- 例：炸弹翻倍 → 展示；封顶 50 倍 → 展示；过级彩金 50 → 展示

## 七、特殊情况处理

### 超时处理汇总

| 场景 | 处理方式 |
|------|---------|
| 进贡超时 | 系统默认选最大牌（除逢人配），有大王选大王，多张时取最下方那张 |
| 还贡超时（有 10 及以下手牌） | 优先级：单张 > （拆）对子 > （拆）三张 > （拆）炸弹；按点数从小到大；花色优先级：方块 > 梅花 > 红桃 > 黑桃 |
| 还贡超时（无 10 及以下手牌） | 同上，但可选任意牌（非逢人配） |
| 首出超时 | 出最小点数非炸弹牌型；只有炸弹时打最小炸弹 |
| 要牌超时 | 3 秒后默认不出 |
| 带入超时 | 强制站起 |
| 补码超时 | 强制站起 |
| 准备超时 | 默认继续，直到本局结束 |

### 托管处理

- 玩家**连续三次超时**进入托管
- 断线后超时按托管处理，断线重连后取消托管

### Toast 提示层级规则

- Toast 出现在最上层
- 短时间内多条 Toast：后出现的将前一条向上顶，**最多同时显示三条**；第四条出现时最上面的消失
- 展示时间内过滤掉相同内容的 Toast

## 八、玩法比较

| 对比项 | 四人掼蛋 | 二人掼蛋 |
|--------|---------|---------|

| 组队方式 | 自由选座，坐对家为同队 | 两人强制为对手 |
