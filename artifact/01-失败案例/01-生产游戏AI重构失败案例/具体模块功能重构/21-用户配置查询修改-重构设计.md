# 用户配置查询/修改 重构设计文档

> 对应分析文档：`docs/Junk-游戏准备重构/21-用户配置查询修改.md`
> 设计日期：2026-06-04
> 工程：dx-game-frame + dx-game-guandan
> 分支：20260615/junk

## 1. 背景与目标

用户配置覆盖 4 个协议：

- 框架 1028 `USER_CONFIG_QUERY` / 1029 `USER_CONFIG_UPDATE`
- 掼蛋 3018 `GD_USER_CONFIG_QUERY` / 3019 `GD_USER_CONFIG_UPDATE`

当前实现存在：

- P0：1029 和 3019 各自实现 `saveAutoBringChipScoreIfNecessary`，口径不同（1029 只要求正数、3019 要求正整数、3019 兼容 `configMap` 透传、3019 关闭时跳过校验并写 0），同一目标值在两条路径上行为不一致。
- P0：3019 成功后用 `JSONUtil.parseObj(req.getConfigContent())` 拼响应，`configContent` 为空时响应阶段可能异常，玩家配置服务已更新但客户端拿到失败，并触发 `refreshUserConfigCacheAfterException` 多刷一次。
- P0：3018 直接 `BeanUtils.copyProperties(dxPlayerConfigResp, respDto)`，玩家服务返回 null 时抛 NPE，最终客户端只能看到系统异常。
- P1：`UserConfigCache.refreshUserCfgCache` invalidate 后立即 `getUnchecked`，RPC 失败会从刷新路径继续抛异常，3019 异常分支没有显式的成功/失败语义，调用方无法判断当前缓存是不是已经回到权威值。
- P2：1028/1029/3018/3019 全链路用 `log.info GsonUtils.toJson(req/resp)` 打印整份 DTO，配置内容可能包含客户端扩展字段，CPU 和日志体积放大。

目标：消除 P0 自动补码口径不一致 + 响应阶段空值异常 + 3018 NPE，并把 `UserConfigCache` 刷新结果显式化，使 3019 异常分支可判断是否成功还原；保持 4 个协议请求/响应字段完全兼容、14050 通知字段不变。

## 2. 决策记录

| 决策项 | 结论 |
| --- | --- |
| AutoComplement 服务归属 | 放 frame（`com.dx.game.frame.service.biz.userconfig`），1029 / 3019 共用；guandan 不重复封装 |
| 自动补码目标值是否必须整数 | **统一要求正整数**（3019 当前行为）。1029 路径同步收紧，避免“桌面通用”和“掼蛋专用”出现非整数差异。`待确认项`：见第 13 节 |
| 关闭自动补码的目标值清理 | 统一在 `AutoComplementConfigService.clear()` 写 0（无 delete 语义），由调用方决定何时调（3019 在 RPC 成功后调；1029 当前不写关闭时清 0，保持现状） |
| 3019 响应 `configContent` 空值 | 直接兜底为空 `JSONObject`，不抛异常 |
| 3018 RPC 空值响应 | 玩家服务返回 null 时返回空 `GdUserConfigQueryRespDto`（带 `userId`），打 warn 日志，不抛系统异常 |

| 灰度策略 | 不做开关，直接替换，依赖测试覆盖。原有私有方法保留为 `@Deprecated`，新代码不再调用，方便回滚 grep |
| 日志收敛 | 1028/1029/3018/3019 入口/出口 `GsonUtils.toJson(req/resp)` 由 `info` → `debug`；保留关键状态变更 info（开关变化、目标值保存/清理） |

## 4. 安全改造（R-01 ~ R-05）

### R-01：抽 `AutoComplementConfigService`（P0）

`AutoComplementConfigService`（frame）

```java
package com.dx.game.frame.service.biz.userconfig;

@Slf4j
@Component
public class AutoComplementConfigService {

    @Resource
    private TableUserRedisService tableUserRedisService;

    /**
     * 保存自动补码目标值。
     * 入参 null 直接返回 null（视为未携带）；
     * 入参不合法（非正整数 / 桌不存在 / 桌范围非法）抛 PARAM_EXCEPTION。
     */
    public BigDecimal saveIfPresent(Long tableId, Long userId, BigDecimal requestValue, boolean requireInteger) {
        if (requestValue == null) {
            return null;
        }
        BaseTableInfo tableInfo = BaseTableInfoCache.getTableInfo(tableId);
        if (tableInfo == null) {
            log.warn("自动补码配置-牌桌不存在 tableId:{} userId:{} requestValue:{}", tableId, userId, requestValue);
            throw BusinessException.createException(BusinessExceptionTypeEnums.PARAM_EXCEPTION);
        }
        BigDecimal min = tableInfo.getRealMinBringChipScore();
        BigDecimal max = tableInfo.getRealMaxBringChipScore();
        if (!isValidRange(min, max)) {
            log.warn("自动补码配置-牌桌范围非法 tableId:{} userId:{} min:{} max:{}", tableId, userId, min, max);
            throw BusinessException.createException(BusinessExceptionTypeEnums.PARAM_EXCEPTION);
        }
        if (!isValidRequest(requestValue, requireInteger)) {
            log.warn("自动补码配置-目标值非法 tableId:{} userId:{} requestValue:{} requireInteger:{}",
                    tableId, userId, requestValue, requireInteger);
            throw BusinessException.createException(BusinessExceptionTypeEnums.PARAM_EXCEPTION);
        }
        BigDecimal saveValue = requestValue.max(min).min(max);
        tableUserRedisService.setAutoBringChipScore(tableId, userId, saveValue);
        log.info("自动补码配置-保存 tableId:{} userId:{} requestValue:{} saveValue:{} min:{} max:{}",
                tableId, userId, requestValue, saveValue, min, max);
        return saveValue;
    }

    /** 关闭自动补码后清空目标值（写 0；Redis 无 delete 语义）。 */
    public void clear(Long tableId, Long userId) {
        if (tableId == null || userId == null) {
            return;
        }
        tableUserRedisService.setAutoBringChipScore(tableId, userId, BigDecimal.ZERO);
        log.info("自动补码配置-清空目标值 tableId:{} userId:{}", tableId, userId);
    }

    /** 兼容 3019 旧客户端：configMap.autoBringChipScore / configMap.autoBringChip */
    public BigDecimal resolveFromConfigMap(Map<String, Object> configMap) {
        if (configMap == null || configMap.isEmpty()) {
            return null;
        }
        Object raw = Optional.ofNullable(configMap.get("autoBringChipScore"))
                .orElse(configMap.get("autoBringChip"));
        if (raw == null) {
            return null;
        }
        try {
            return new BigDecimal(String.valueOf(raw));
        } catch (NumberFormatException e) {
            log.warn("自动补码配置-configMap 格式错误 raw:{}", raw, e);
            throw BusinessException.createException(BusinessExceptionTypeEnums.PARAM_EXCEPTION);
        }
    }

    private boolean isValidRange(BigDecimal min, BigDecimal max) {
        return min != null && max != null
                && min.compareTo(BigDecimal.ZERO) > 0
                && max.compareTo(min) >= 0;
    }

    private boolean isValidRequest(BigDecimal value, boolean requireInteger) {
        if (value == null || value.compareTo(BigDecimal.ZERO) <= 0) {
            return false;
        }
        if (requireInteger && value.stripTrailingZeros().scale() > 0) {
            return false;
        }
        return true;
    }
}
```

**1029 改造**：删除私有 `saveAutoBringChipScoreIfNecessary` / `isValidAutoBringRange` / `isPositive`，改为：

```java
BigDecimal savedValue = autoComplementConfigService.saveIfPresent(tableId, userId, reqDto.getAutoBringChipScore(), true);
if (savedValue != null) {
    reqDto.setAutoBringChipScore(savedValue);
}
boolean autoBringChipScoreUpdated = savedValue != null;
```

**3019 改造**：删除私有 `saveAutoBringChipScoreIfNecessary` / `clearAutoBringChipScoreIfClosed` / `resolveAutoBringChipScore` / `toBigDecimal` / `isValidAutoBringRange` / `isPositiveInteger`，改为：

```java
private void saveAutoBringChipScoreIfNecessary(GdUserConfigUpdateReq req) {
    if (Integer.valueOf(0).equals(req.getAutoComplement())) {
        log.info("掼蛋自动补码配置-关闭请求跳过目标值保存 tableId:{} userId:{} autoComplement:{}",
                req.getTableId(), req.getUserId(), req.getAutoComplement());
        return;
    }
    BigDecimal requestValue = req.getAutoBringChipScore();
    if (requestValue == null) {
        requestValue = autoComplementConfigService.resolveFromConfigMap(req.getConfigMap());
    }
    if (requestValue == null) {
        return;
    }
    autoComplementConfigService.saveIfPresent(req.getTableId(), req.getUserId(), requestValue, true);
}

private void clearAutoBringChipScoreIfClosed(GdUserConfigUpdateReq req, boolean updateFlag) {
    if (!updateFlag || !Integer.valueOf(0).equals(req.getAutoComplement())) {
        return;
    }
    autoComplementConfigService.clear(req.getTableId(), req.getUserId());
}
```

### R-04：`UserConfigCacheService` 刷新结果显式化（P1）

`UserConfigCacheService`（frame）

```java
package com.dx.game.frame.service.biz.userconfig;

@Slf4j
@Component
public class UserConfigCacheService {

    public UserConfig get(Long userId) {
        try {
            return UserConfigCache.getUserCfgCache(userId);
        } catch (Exception e) {
            log.warn("UserConfigCacheService-get 异常 userId:{}", userId, e);
            return null;
        }
    }

    /** 刷新缓存，返回是否成功。失败仅记录日志，不向上抛。 */
    public boolean refresh(Long userId) {
        if (userId == null) {
            return false;
        }
        try {
            UserConfigCache.refreshUserCfgCache(userId);
            return true;
        } catch (Exception e) {
            log.warn("UserConfigCacheService-refresh 异常 userId:{}", userId, e);
            return false;
        }
    }

    /** RPC 前预写本地开关，仅用于收窄同节点并发窗口。 */
    public void prewriteAutoComplement(Long userId, Integer autoComplement) {
        if (userId == null || autoComplement == null) {
            return;
        }
        UserConfig userConfig = get(userId);
        if (userConfig == null) {
            log.warn("UserConfigCacheService-prewrite 跳过 缓存为空 userId:{} autoComplement:{}", userId, autoComplement);
            return;
        }
        Integer before = userConfig.getAutoComplement();
        userConfig.setAutoComplement(autoComplement);
        log.info("UserConfigCacheService-RPC 前已预写 autoComplement userId:{} before:{} after:{}",
                userId, before, autoComplement);
    }
}
```

**3019 改造**：

- `prewriteUserConfigCacheBeforeRpc(req)` → `userConfigCacheService.prewriteAutoComplement(req.getUserId(), req.getAutoComplement())`
- `refreshUserConfigCacheIfNecessary(req, updateFlag)` → `userConfigCacheService.refresh(req.getUserId())`（仅当 `req.getAutoComplement() != null && req.getUserId() != null` 时调，保持原触发条件）
- `refreshUserConfigCacheAfterException(req)` → `userConfigCacheService.refresh(req.getUserId())`（异常路径同样按 `autoComplement != null` 触发）

**1029 改造**：`UserConfigCache.refreshUserCfgCache(userId)` 调用替换为 `userConfigCacheService.refresh(userId)`，RPC 成功且刷新成功才走匿名变化比较。

`UserConfigCache` 静态方法继续保留，供其它非本次范围的调用方使用（如结算、退出、补码链路），后续可单独迁移。

## 7. 验收关键场景

1. **1029 / 3019 自动补码口径一致**：同一 `tableId/userId`，分别通过 1029 和 3019 提交 `autoBringChipScore=150.5`，两条路径都因非整数被拒（PARAM_EXCEPTION）。
2. **1029 / 3019 写入结果一致**：同一桌设置 `autoBringChipScore=200`，两路径都按 [min, max] 夹取并写 Redis；后续读取一致。
3. **3019 关闭自动补码**：`autoComplement=0`，RPC 成功后 Redis 目标值被写为 0；关闭失败时 Redis 目标值不变。
4. **3019 configMap 兼容**：仅传 `configMap.autoBringChipScore=300`，AutoComplementConfigService 仍能解析并保存。
5. **3019 `configContent=null`**：3019 请求只携带 `autoComplement/autoBringChipScore`，配置更新成功后响应仍返回成功（空 JSON），不出现异常分支。
6. **3018 RPC 返回 null**：玩家服务故障返回 null 时，客户端拿到 `userId` 非空、`configContent=null` 的响应，日志有 warn，不抛系统异常。
7. **3019 RPC 异常后缓存还原**：RPC 抛异常后 `UserConfigCacheService.refresh(userId)` 被调一次；刷新失败时日志有 warn，但不会再次抛异常导致丢响应。
8. **日志体积**：1028/1029/3018/3019 正常路径不再出现 `GsonUtils.toJson(req/resp)` 的 info 输出；info 行只保留协议 + 关键字段。
