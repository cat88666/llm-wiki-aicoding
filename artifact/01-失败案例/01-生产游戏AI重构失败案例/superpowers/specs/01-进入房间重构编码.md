## 二. 安全重构

### P0：执行链路简化

```java
/**
 * 进入房间核心执行链路简化，单一入口，清晰职责边界
 * 原来逻辑：Handler 里 initContext/check/handleData/after 混在一起，职责不清，难以维护和扩展
 * 原来问题：
 * 改造原则：
 * - 单一入口：handleData 只做一件事，调用 EnterRoomFlowService.execute(context)，后续改进都在这个 service 里做，不再修改 Handler 主流程
 * - 职责边界：execute 内部分为校验、状态管理、快照组装、风控初始化四个步骤，每个步骤由独立组件负责，互不干扰，易于维护和扩展
 */ 
```

#### `EnterRoomFlowService`（frame）

```java
@Component
public class EnterRoomFlowService {
    @Resource EnterRoomValidatorChain    validatorChain;
    @Resource EnterRoomStateService      stateService;
    @Resource EnterRoomSnapshotAssembler snapshotAssembler;

    public void execute(EnterRoomContextV2 context) {
        validatorChain.validate(context);
        stateService.applyState(context);
        snapshotAssembler.assemble(context);
        UserRiskCache.initRiskDataEnterRoom(context.getRequest());
    }
}
```

### P1：Handler 工厂收敛

```java
/**
 * Handler 工厂收敛
 * 原来逻辑：HandlerSpaceService / InitializingBeanConfig 分散注册空间 Handler
 * 原来问题：Handler 查找和注册入口不统一，空间类型扩展容易散落
 * 改造原则：
 * - 单一入口：统一由 HandlerFactory 按 spaceType + protoId 获取 Handler
 * - 兼容现有空间：保留 Hall / Club / Private / Race 空间 Handler
 */
```

### P2：责任链校验
```java
/**
 * 责任链校验
 * 原来逻辑：原有8个校验，然后按序调用所有
 * 原来问题：
 * 改造原则：
 */
@Component
public class EnterRoomValidatorChain {
    @Resource List<IEnterRoomValidationExtension> contributors;

    public void validate(EnterRoomContextV2 context) {
        Long tableId = context.getTableId();
        Long userId  = context.getRequest().getUserId();
        Boolean alreadyInRoom = context.isAlreadyInRoom();

        BaseTableCheckService.checkDevice(tableId, context.getRequest().getDeviceType());
        BaseTableCheckService.checkTable(tableId);
        BaseTableCheckService.checkCurrency(tableId, userId);
        BaseTableCheckService.checkTableWatchUserLimit(tableId, alreadyInRoom);
        BaseUserCheckAuthService.checkUserPwd(tableId, userId, context.getRequest().getTableEntryPassword());
        BaseUserCheckAuthService.checkUserKick(tableId, userId);
        BaseUserCheckAuthService.checkUserLockClub(userId, alreadyInRoom);
        BaseUserCheckAuthService.checkUserClub(tableId, userId, alreadyInRoom);

        SpaceTypeEnum spaceType = context.getSpaceType();
        contributors.stream().filter(c -> c.supports(spaceType)).forEach(c -> c.validate(context));
    }
}

/**
 * 空间类型校验扩展点，提供默认实现，游戏侧按需覆盖
 */
public interface IEnterRoomValidationExtension {
    void validate(EnterRoomContextV2 context);
    default boolean supports(SpaceTypeEnum spaceType) { return true; }
}
```

### P4：响应快照组装

#### 1、`EnterRoomSnapshotAssembler`（frame）
```java
/**
    * 响应快照组装，单一入口，分段组装
    * 原来逻辑：handleData 里直接组装响应，涉及用户配置、用户列表、手牌快照、桌信息等多个领域，代码耦合，难以维护和扩展
    * 原来问题：
    * 改造原则：
    * - 单一入口：handleData 只调用 snapshotAssembler.assemble(context)，后续改进都在 snapshotAssembler 里做，不再修改 Handler 主流程
    * - 分段组装：snapshotAssembler 内部分为多个 Section，每个 Section 负责组装响应的一部分，互不干扰，易于维护和扩展
    */
@Component
public class EnterRoomSnapshotAssembler {
    private final List<IEnterRoomSnapshotSectionAssembler> sectionAssemblers;

    public EnterRoomSnapshotAssembler(List<IEnterRoomSnapshotSectionAssembler> sectionAssemblers) {
        this.sectionAssemblers = sectionAssemblers;
    }

    public void assemble(EnterRoomContextV2 context) {
        EnterRoomRespDtoV2 response = context.getEnterRoomRespDtoV2();
        for (IEnterRoomSnapshotSectionAssembler sectionAssembler : sectionAssemblers) {
            sectionAssembler.assemble(context, response);
        }
    }
}

/**
* 快照组装 Section 接口，单一职责，按需实现
* 每个 Section 负责组装响应的一部分，输入 context 和当前 response，输出修改后的 response
* frame 内置多个 Section 负责不同领域的组装，游戏侧也可以新增 Section 承接特定领域的组装逻辑
*/
public interface IEnterRoomSnapshotSectionAssembler {
    void assemble(EnterRoomContextV2 context, EnterRoomRespDtoV2 response);
}


@Component
public class GdEnterRoomHandSnapshotProvider implements IEnterRoomHandSnapshotProvider {
    private final TableUserHandBootstrap tableUserHandBootstrap;

    public EnterRoomHandSnapshot build(EnterRoomContextV2 context) {
        Long tableId = context.getTableId();
        Long userId  = context.getRequest().getUserId();

        BaseTableHand tableHand = BaseTableHandCache.getTableHand(tableId);
        if (tableHand == null) {
            log.warn("进入房间-TableHand 缓存为空，返回空手牌快照 tableId:{} userId:{}", tableId, userId);
            return EnterRoomHandSnapshot.empty();
        }

        BaseTableHandDto handDto = tableUserHandBootstrap.buildTableHandInfo(tableId, userId);
        Map<Integer, BaseTableUserHandDto> userHandMap = tableUserHandBootstrap.buildTableUserHand(tableId, userId);
        return new EnterRoomHandSnapshot(handDto, userHandMap);
    }
}

public interface IEnterRoomHandSnapshotProvider {
    EnterRoomHandSnapshot build(EnterRoomContextV2 context);
}

public class EnterRoomHandSnapshot {
    private BaseTableHandDto tableHandDto;
    private Map<Integer, BaseTableUserHandDto> tableUserHandMap;

    public static EnterRoomHandSnapshot empty() {
        return new EnterRoomHandSnapshot(new BaseTableHandDto(), Collections.emptyMap());
    }
}

@Component
@Order(30)
public class EnterRoomHandSnapshotAssembler implements IEnterRoomSnapshotSectionAssembler {
    private final Optional<IEnterRoomHandSnapshotProvider> handSnapshotProvider;

    public void assemble(EnterRoomContextV2 context, EnterRoomRespDtoV2 response) {
        EnterRoomHandSnapshot snapshot = handSnapshotProvider
                .map(provider -> provider.build(context))
                .orElseGet(() -> buildDefaultSnapshot(context));

        response.setTableHandDto(snapshot.getTableHandDto());
        response.setTableUserHandMap(snapshot.getTableUserHandMap());
    }

    private EnterRoomHandSnapshot buildDefaultSnapshot(EnterRoomContextV2 context) {
        Long tableId = context.getTableId();
        Long userId = context.getRequest().getUserId();
        return new EnterRoomHandSnapshot(
                GameUserHandBootstrapService.buildTableHandInfo(tableId, userId),
                GameUserHandBootstrapService.buildTableUserHand(tableId, userId));
    }
}
```

### P5：后置动作拆分

```java
protected void after(EnterRoomContextV2 context) {
    Long tableId = context.getTableId();
    Long userId  = context.getUserBaseInfo().getUserId();
    BaseTableHand tableHand = BaseTableHandCache.getTableHand(tableId);

    // 2. 轨迹/活动事件（可失败，监听器内 catch 打日志）
    applicationEventPublisher.publishEvent(new UserEnterRoomFinishEventV2(context, tableHand != null ? tableHand.getHandCode() : null));

    // 3. 发送响应
    EnterRoomRespDtoV2 enterRoomResp = context.getEnterRoomRespDtoV2();
    log.info("进入房间成功 tableId:{} userId:{}", tableId, userId);
    NetResponseBody netResponseBody = NetResponseBuilder.buildResponse(CommonProto.ENTER_ROOM, NoticeEncryptUtils.encrypt(enterRoomResp, tableId, userId), context.getRequest());
    SendUtil.send(netResponseBody);

    // 4. 清理、私人房、广播（保持不变）
    baseUserOnlineService.clearOfflineStartTime(tableId, userId);
    BaseTableInfo tableInfo = BaseTableInfoCache.getTableInfo(tableId);
    if (tableInfo.isPrivateTable()) {
        retryPrivateTableRedisService.joinPrivateTable(userId, tableId);
    }
    enterRoomNotice.sendNotice(tableId, userId, context.isAlreadyInRoom());
}
```

`BaseUserEnterRoomFinishListener` 移除 `异常带出` 分支，只保留轨迹和活动逻辑。

#### 私人房人数计数修复

改法：去掉 `incrementAndGet`，改用 WATCH_USER + PLAYER_USER 实际集合大小判断：

```java
public static void checkPrivateTableUserUpperLimit(Long tableId, Boolean alreadyInRoom) {
    if (alreadyInRoom) return;
    BaseTableInfo tableInfo = BaseTableInfoCache.getTableInfo(tableId);
    int currentCount = BaseTableUserCache.fetchTableUserIds(tableId, TableUserCacheEnum.WATCH_USER).size()
                     + BaseTableUserCache.fetchTableUserIds(tableId, TableUserCacheEnum.PLAYER_USER).size();
    if (currentCount >= tableInfo.getPrivateUpperLimit()) {
        log.info("房间{}人数已达上限{}，禁止进入，当前:{}", tableId, tableInfo.getPrivateUpperLimit(), currentCount);
        throw BusinessException.createException(BusinessExceptionTypeEnums.PRIVATE_ENT_ROOM_EXCEEDS_LIMIT);
    }
}
```
